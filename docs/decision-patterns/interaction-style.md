# Шаблон выбора стиля взаимодействия

Шаблон помогает выбрать между синхронной моделью, асинхронной моделью и гибридом.

## Матрица выбора

| Условие | Синхронный вызов | Асинхронные события | Гибрид |
| --- | --- | --- | --- |
| Требуется мгновенный ответ пользователю | да | нет | да |
| Высокий риск временной недоступности downstream | риск | устойчиво через очередь | устойчиво |
| Гарантия порядка обработки | сложнее | зависит от брокера и ключа | управляемо |
| Сложность трассировки | ниже | выше | средняя |
| Подходит для long-running процессов | нет | да | да |

## Рекомендация по умолчанию

- Клиентский запрос с требованием быстрого ответа: синхронный [API](../glossary.md#abbr-005).
- Межсервисная обработка с очередями и ретраями: асинхронная модель.
- Бизнес-процесс с подтверждением и последующей обработкой: гибрид.

## Шаблон решения

```markdown
Сценарий:
- Операция:
- SLA:
- Критичность консистентности:

Решение:
- Front-door: sync/async
- Back-stage: sync/async
- Идемпотентность:
- Retry/DLQ:
```

## Пример (финтех-платеж)

- Front-door: синхронный запрос на создание платежа (подтверждение приема).
- Back-stage: асинхронные шаги антифрода, клиринга и уведомлений.
- Управление сбоями: idempotency key, retry с backoff, DLQ.

## Риски и компенсации

| Риск | Компенсация |
| --- | --- |
| Дубли операций | обязательная идемпотентность и deduplication |
| Каскадный timeout | timeout budget + circuit breaker |
| Потеря сообщения | durable queue, ack-policy, мониторинг lag |
