# Решения по шаблону

Раздел содержит готовые decision records для типовых развилок, с которыми аналитик и архитектор сталкиваются в большинстве проектов.

## Уровни сложности

### Базовый уровень

- выбрать подход по контексту задачи;
- зафиксировать решение в кратком формате;
- согласовать последствия с ключевыми стейкхолдерами.

### Средний уровень

- оценивать альтернативы по метрикам (latency, стоимость, time-to-market);
- учитывать риски совместимости и масштабирования;
- привязывать решение к требованиям и ограничениям.

### Продвинутый уровень

- выстраивать portfolio-стандарты решений для нескольких команд;
- управлять эволюцией решений через [ADR](../glossary.md#abbr-003);
- измерять эффект решений через delivery и эксплуатационные метрики.

## Как использовать раздел

1. Выберите нужную развилку из карты ниже.
2. Скопируйте шаблон decision record.
3. Заполните входные параметры по фактам проекта.
4. Сравните альтернативы по таблице критериев.
5. Зафиксируйте выбранный вариант и план отката.
6. Сошлитесь на запись в [требованиях](../requirements/index.md), [архитектуре](../architecture/index.md) и [интеграциях](../integrations/index.md).

## Стандарты принятия решений

- Любое решение должно содержать контекст, альтернативы и последствия.
- Для критичных решений обязателен измеримый критерий успеха.
- Для решений с высокой стоимостью изменения нужен plan B.
