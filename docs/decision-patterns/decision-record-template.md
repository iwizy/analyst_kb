# Унифицированный Decision Record

Страница задает единый формат, в котором фиксируются технические и процессные решения.

## Структура записи

| Поле | Что заполнять | Пример |
| --- | --- | --- |
| ID | Уникальный идентификатор | `DR-2026-03-001` |
| Статус | Draft, Accepted, Superseded | `Accepted` |
| Контекст | Факты, ограничения, [SLA](../glossary.md#abbr-079), зависимости | Public API, p95 <= 250 ms |
| Альтернативы | 2-4 реально рассматриваемых варианта | REST, gRPC, GraphQL |
| Критерии сравнения | latency, стоимость, сложность, риск | 4 критерия с весами |
| Выбор | Принятое решение | REST для внешнего API |
| Последствия | Выгоды, риски, техдолг | +быстрый старт, -overfetch |
| План отката | Как вернуться к безопасному состоянию | версионирование + dual-run |
| Связанные артефакты | ссылки на [ADR](../glossary.md#abbr-003), [RTM](../glossary.md#abbr-107), схемы | ADR-014, C4-L2 |

## Шаблон (Markdown)

```markdown
# DR-YYYY-MM-XXX: <Название решения>

## Статус
Accepted

## Контекст
- Бизнес-цель:
- Ограничения:
- NFR/SLA:
- Зависимости:

## Альтернативы
1. Вариант A
2. Вариант B
3. Вариант C

## Критерии и оценка
| Критерий | Вес | A | B | C |
| --- | --- | --- | --- | --- |
| ... | ... | ... | ... | ... |

## Решение
<Выбранный вариант и границы применения>

## Последствия
- Плюсы:
- Риски:
- Ограничения:

## План отката/миграции
- Trigger:
- Action:
- Deadline:

## Связанные материалы
- Требования:
- Диаграммы:
- Тест-план:
```

## Правила оценки альтернатив

| Вес | Интерпретация |
| --- | --- |
| 1 | малый вклад в успех |
| 3 | значимый вклад |
| 5 | критичный вклад |

Итоговый балл варианта: `Σ(оценка * вес)`.

## Мини-пример

| Критерий | Вес | REST | gRPC |
| --- | --- | --- | --- |
| Внешняя совместимость | 5 | 5 | 2 |
| Latency | 3 | 3 | 5 |
| Простота входа для партнеров | 4 | 5 | 2 |
| Итого |  | 46 | 36 |

Решение: внешний API строится на REST, внутренний межсервисный канал допускает gRPC.
