# Agile-процессы и практики команды

Раздел описывает как организовать поток работ в Agile-среде: Scrum, Kanban, SAFe, Lean.

## Scrum: роли, церемонии, артефакты

### Роли

- Product Owner;
- Scrum Master;
- Developers.

### Церемонии

- Sprint Planning;
- Daily Scrum;
- Sprint Review;
- Sprint Retrospective.

### Артефакты

- Product Backlog;
- Sprint Backlog;
- Increment.

## Kanban: поток и WIP-лимиты

- визуализация доски;
- лимиты WIP по колонкам;
- управление cycle time и throughput;
- регулярные flow-review.

## SAFe и Lean

- SAFe: координация нескольких Agile-команд через PI Planning и ART.
- Lean: устранение потерь, оптимизация value stream.

## Практический пример Sprint Planning

1. Сформулировать sprint goal.
1. Выбрать backlog items, которые ведут к цели.
1. Разбить на задачи и проверить capacity.
1. Зафиксировать риски и зависимости.

## Пример Definition of Done

- код в main ветке;
- unit/integration тесты зелёные;
- обновлена документация;
- нет блокирующих дефектов;
- инкремент готов к демонстрации.

## Пример ретроспективы

Формат: `Start / Stop / Continue`.

- Start: добавляем early API review.
- Stop: берем в спринт незрелые задачи.
- Continue: ежедневная синхронизация по блокерам.
