# Основы SRE

SRE (Site Reliability Engineering) - инженерный подход к надежности сервиса,
где эксплуатация управляется метриками и automation-first практиками.

## Ключевые понятия

- SLI: измеримый индикатор качества сервиса.
- SLO: целевой уровень сервиса по выбранному SLI.
- Error Budget: допустимый объем отклонений от SLO.

## Как это применять

1. Выберите 2-3 SLI для критичных пользовательских сценариев.
1. Зафиксируйте SLO и период измерения.
1. Определите политику действий при исчерпании error budget.
1. Автоматизируйте повторяемые операционные задачи.

## Пример логики по error budget

```kroki-plantuml
@startuml
start
:Проверить текущий error budget;
if (Budget в норме?) then (Да)
  :Плановые релизы и развитие;
else (Нет)
  :Ограничить релизы;
  :Приоритет на reliability-задачи;
endif
stop
@enduml
```

## Типичные ошибки

- установка SLO без договоренности с бизнесом;
- избыточное число метрик без операционной пользы;
- ручные операции без автоматизации в критичном контуре.

## Чек-лист качества

- SLO привязаны к пользовательской ценности.
- Команда использует error budget в релизных решениях.
- Reliability-улучшения имеют отдельный приоритет в backlog.
