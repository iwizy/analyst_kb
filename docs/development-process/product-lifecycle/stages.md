# Этапы жизненного цикла ПО

Этапы жизненного цикла помогают определить, какие решения принимаются в проекте
и какие артефакты считаются результатом каждого шага.

## Базовые этапы

- Инициация: цели, границы, бизнес-кейс.
- Анализ и проектирование: требования, архитектурные решения, план релизов.
- Реализация: разработка, код-ревью, сборка.
- Тестирование: функциональная и нефункциональная проверка.
- Внедрение: релиз, миграции, контроль запуска.
- Эксплуатация и развитие: мониторинг, инциденты, улучшения.

## Пример артефактов по этапам

| Этап | Ключевые артефакты |
| --- | --- |
| Инициация | vision, scope, список стейкхолдеров |
| Анализ и проектирование | спецификация требований, схема интеграций |
| Реализация | pull request, changelog, техническая документация |
| Тестирование | тест-кейсы, отчеты дефектов, протокол приемки |
| Внедрение | release notes, план отката, runbook |
| Эксплуатация | dashboard метрик, postmortem, backlog улучшений |

## Типичные ошибки

- переход к разработке до стабилизации критичных требований;
- отсутствие критериев готовности этапа;
- формальный handover между командами без контекста.

## Чек-лист качества

- Для каждого этапа определены входы, выходы и ответственные.
- Есть критерии готовности и критерии перехода на следующий этап.
- Решения этапа фиксируются в доступных команде артефактах.
