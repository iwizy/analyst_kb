# Жизненный цикл программного продукта

Жизненный цикл ПО (SDLC) задает последовательность инженерных и управленческих решений от идеи до устойчивой эксплуатации. Для системного аналитика это основной каркас трассировки: бизнес-цель -> требования -> архитектура -> реализация -> тесты -> релиз -> обратная связь.

## Уровни сложности

### Базовый уровень

- понимать фазы SDLC и их назначение;
- знать обязательные артефакты каждой фазы;
- участвовать в проверке готовности перехода между фазами.

### Средний уровень

- формализовать входы/выходы этапов;
- настраивать DoR/DoD и quality gates;
- увязывать SDLC с Agile/DevOps-практиками.

### Продвинутый уровень

- адаптировать SDLC под домен (fintech, e-commerce, gov, embedded);
- управлять метриками эффективности и надежности;
- строить масштабируемый процесс для multi-team программ.

```kroki-plantuml

:Инициация;
:Анализ;
:Дизайн;
:Разработка;
:Тестирование;
:Внедрение;
:Эксплуатация;
:Улучшения и новая итерация;
stop
```

## Что важно системному аналитику

- фиксировать артефакты в формате, пригодном для проверки и реализации;
- заранее учитывать эксплуатационные сценарии и ограничения;
- не допускать разрыва между требованиями, тестами и операционной поддержкой.

## Чек-лист самопроверки

- для каждой фазы определены входы, выходы, роли и критерии завершения;
- определены quality gates и правила эскалации рисков;
- pipeline поставки связан с мониторингом и обратной связью;
- процесс допускает управляемые изменения требований.
