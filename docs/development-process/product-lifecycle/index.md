# Жизненный цикл программного продукта

[Жизненный цикл ПО](../../glossary.md#term-026) (SDLC) задает последовательность инженерных и управленческих решений от идеи до устойчивой эксплуатации. Для системного аналитика это основной каркас трассировки: бизнес-цель -> требования -> архитектура -> реализация -> тесты -> релиз -> обратная связь.

## Уровни сложности

### Базовый уровень

- понимать фазы SDLC и их назначение;
- знать обязательные артефакты каждой фазы;
- участвовать в проверке готовности перехода между фазами.

### Средний уровень

- формализовать входы/выходы этапов;
- настраивать [DoR](../../glossary.md#abbr-027)/[DoD](../../glossary.md#abbr-026) и quality gates;
- увязывать SDLC с Agile/DevOps-практиками.

### Продвинутый уровень

- адаптировать SDLC под домен (fintech, e-commerce, gov, embedded);
- управлять метриками эффективности и надежности;
- строить масштабируемый процесс для multi-team программ.

## SDLC как непрерывный цикл

```kroki-plantuml
skinparam monochrome true
start
:Инициация;
:Анализ;
:Дизайн;
:Разработка;
:Тестирование;
:Внедрение;
:Эксплуатация;
:Улучшения и новая итерация;
stop
```

## Что важно системному аналитику

- фиксировать артефакты в формате, пригодном для проверки и реализации;
- заранее учитывать эксплуатационные сценарии и ограничения;
- не допускать разрыва между требованиями, тестами и операционной поддержкой.
