# Git

Git - система контроля версий, которая позволяет безопасно вести изменения,
прослеживать историю решений и организовывать командную разработку.

## Базовые сущности

- commit: зафиксированное изменение;
- branch: изолированная линия разработки;
- merge / rebase: интеграция изменений;
- tag: отметка версии релиза.

## Типовой рабочий процесс

1. Создайте ветку от актуальной основной ветки.
1. Делайте небольшие логические коммиты с понятными сообщениями.
1. Откройте pull request и пройдите код-ревью.
1. После merge убедитесь, что изменения попали в релизный поток.

## Типичные ошибки

- крупные коммиты без связки с задачей;
- долгоживущие ветки без регулярной синхронизации;
- merge без ревью и автоматических проверок.

## Чек-лист качества

- Стратегия ветвления зафиксирована и понятна всей команде.
- История изменений позволяет восстановить контекст решений.
- Для merge в основную ветку обязательны проверки качества.
