# Тестирование в процессе разработки

Тестирование подтверждает, что продукт реализует требования, соответствует NFR и безопасно работает в целевой среде.

## Уровни сложности

### Базовый уровень

- различать уровни тестирования: unit, integration, system, acceptance;
- понимать типы тестов: functional, performance, security, usability;
- знать роль аналитика в приемке и анализе дефектов.

### Средний уровень

- строить тестовую стратегию с приоритизацией по рискам;
- связывать test cases с требованиями и трассировкой;
- использовать shift-left подход и CI-интеграцию тестов.

### Продвинутый уровень

- проектировать автоматизированный quality-gate контур;
- применять BDD/TDD и contract testing;
- управлять тестовыми метриками и качеством релизного решения.

## Уровни тестирования

| Уровень | Что проверяет | Кто обычно владелец |
| --- | --- | --- |
| Unit | функция/класс | разработчик |
| Integration | взаимодействие модулей/сервисов | разработчик + QA |
| System | поведение системы end-to-end | QA |
| Acceptance (UAT) | бизнес-ценность и критерии приемки | бизнес + BA/SA + QA |

## Типы тестирования

| Тип | Цель | Примеры инструментов |
| --- | --- | --- |
| Functional | корректность сценариев | pytest, JUnit, Postman |
| Performance | отклик и устойчивость под нагрузкой | JMeter, k6, Gatling |
| Security | уязвимости и неправильные конфигурации | OWASP ZAP, Snyk, Trivy |
| Usability | удобство и понятность для пользователей | UX тест-сессии, прототипы |

## Стратегии тестирования

- Shift-left: раннее включение тестирования в анализ и дизайн;
- BDD: спецификация поведения через примеры (Given/When/Then);
- TDD: тест до кода для критичных правил;
- Risk-based testing: глубина тестов по стоимости ошибки.

## Роль аналитика в тестировании

- формулирует тестируемые AC;
- помогает QA уточнять бизнес-сценарии;
- участвует в triage дефектов;
- подтверждает бизнес-приемку инкремента.

## Чек-лист самопроверки

- тесты покрывают критичные пользовательские сценарии и NFR;
- есть связь требований с тест-кейсами;
- дефекты классифицируются по влиянию на бизнес;
- quality gates встроены в pipeline.
