# Тестирование

Тестирование подтверждает соответствие решения требованиям и снижает риск дефектов в production.

## Уровни тестирования

| Уровень | Что проверяет | Когда критичен |
| --- | --- | --- |
| Unit | отдельные функции/модули | быстрый feedback при разработке |
| Integration | взаимодействие модулей и сервисов | интеграционные риски |
| Acceptance | бизнес-сценарии и критерии приемки | перед релизом |
| Load/Performance | поведение под нагрузкой | NFR и SLA |

## Практика

- связывайте тесты с требованиями и рисками;
- проектируйте негативные и граничные сценарии;
- для критичных потоков добавляйте performance-проверки.
