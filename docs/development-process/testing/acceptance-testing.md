# Приемка функциональности

Приемка функциональности подтверждает, что поставка соответствует согласованным
требованиям и критериям готовности.

## Что проверяется на приемке

- выполнение бизнес-сценариев end-to-end;
- соответствие функциональным и ключевым нефункциональным требованиям;
- отсутствие блокирующих дефектов для релиза.

## Типовой процесс приемки

```kroki-plantuml
@startuml
start
:Подготовить критерии приемки;
:Провести демонстрацию и проверку сценариев;
if (Критерии выполнены?) then (Да)
  :Подписать протокол приемки;
  :Разрешить релиз/внедрение;
else (Нет)
  :Зафиксировать замечания;
  :Согласовать доработки;
endif
stop
@enduml
```

## Артефакты приемки

- критерии приемки;
- протокол проверки сценариев;
- список замечаний и решение по релизу.

## Типичные ошибки

- приемка без заранее согласованных критериев;
- замена приемки демонстрацией без фактической проверки;
- отсутствие формального решения при частичном прохождении критериев.

## Чек-лист качества

- Критерии приемки измеримы и согласованы до теста.
- Решение по приемке зафиксировано и прозрачно.
- Замечания преобразованы в задачи с приоритетами.

## Практическое дополнение

- Проведите мини-аудит текущего процесса по схеме: `требование -> реализация -> тест -> эксплуатация`.
- Зафиксируйте одно узкое место и добавьте улучшение в ближайший спринт/итерацию.
- Проверьте эффект через 2 недели по измеримой метрике (дефекты, lead time, MTTR или SLA).
