# Ручное тестирование

Ручное тестирование проверяет пользовательские сценарии и качество поведения
системы там, где автоматизация не дает достаточной уверенности.

## Когда особенно полезно

- exploratory-проверка новых сценариев;
- проверка UX и текстов интерфейса;
- валидация интеграций с нестабильными внешними системами.

## Формат работы

1. Подготовьте тестовые данные и окружение.
1. Выполните сценарии по тест-кейсам и exploratory-чек-листу.
1. Зафиксируйте дефекты: шаги, фактический результат, severity.
1. Проведите ретест после исправлений.

## Типичные ошибки

- отсутствие критериев завершения тестирования;
- проверка только happy path;
- слабая детализация дефектов, из-за чего их сложно воспроизвести.

## Чек-лист качества

- Покрыты критичные пользовательские и негативные сценарии.
- Дефекты описаны воспроизводимо и с понятным влиянием.
- Результаты тестирования прозрачны для команды и бизнеса.
