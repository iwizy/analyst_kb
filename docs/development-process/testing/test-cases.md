# Тест-кейсы

Тест-кейс - формализованный сценарий проверки, который определяет условия,
шаги и ожидаемый результат.

## Структура тест-кейса

- идентификатор и цель проверки;
- предусловия и тестовые данные;
- шаги выполнения;
- ожидаемый результат;
- фактический результат и статус.

## Мини-пример

| Поле | Значение |
| --- | --- |
| ID | TC-ORDER-001 |
| Цель | Проверка успешного создания заказа |
| Предусловия | Пользователь авторизован, товар в наличии |
| Шаги | 1) Добавить товар в корзину 2) Оформить заказ |
| Ожидаемый результат | Заказ создан, статус `NEW` |

## Практические рекомендации

1. Один тест-кейс проверяет один главный ожидаемый результат.
1. Пишите шаги так, чтобы их мог выполнить другой участник команды.
1. Добавляйте негативные и граничные сценарии отдельно.

## Типичные ошибки

- слишком общие шаги без проверяемого результата;
- объединение нескольких сценариев в одном кейсе;
- отсутствие актуализации кейсов после изменений требований.

## Чек-лист качества

- Тест-кейс однозначен и воспроизводим.
- Ожидаемый результат измерим и проверяем.
- Кейсы связаны с требованиями и критериями приемки.

## Практическое дополнение

- Проведите мини-аудит текущего процесса по схеме: `требование -> реализация -> тест -> эксплуатация`.
- Зафиксируйте одно узкое место и добавьте улучшение в ближайший спринт/итерацию.
- Проверьте эффект через 2 недели по измеримой метрике (дефекты, lead time, MTTR или SLA).
