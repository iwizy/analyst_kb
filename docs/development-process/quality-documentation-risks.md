# Качество, документация и риски

Этот раздел фиксирует минимальный управленческий контур, без которого delivery становится непредсказуемым.

## Управление качеством

- quality gates в CI/CD;
- test strategy по уровням (unit/integration/e2e/load);
- критерии приемки и DoD;
- регулярный анализ дефектов и причин.

## Управление документацией

| Артефакт | Что должно быть |
| --- | --- |
| Требование | цель, сценарии, критерии приемки, NFR |
| Архитектурное решение | контекст, выбор, trade-off, последствия |
| API-контракт | схемы, версии, ошибки, примеры |
| Runbook | шаги реагирования, эскалация, rollback |

## Управление рисками

1. Вести единый реестр рисков.
1. Для high-risk элементов заводить превентивные задачи в backlog.
1. Контролировать риск-статус на планировании и review.

## Практический чек-лист перед релизом

- критерии приемки выполнены;
- критичные дефекты закрыты;
- документация актуальна;
- есть rollback и коммуникационный план;
- риски релиза подтверждены ответственными.
