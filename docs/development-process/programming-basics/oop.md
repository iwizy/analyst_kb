# ООП

Объектно-ориентированное программирование (ООП) организует код вокруг объектов
и их поведения, что упрощает поддержку и развитие сложной бизнес-логики.

## Базовые принципы

- инкапсуляция: данные и поведение управляются внутри объекта;
- абстракция: наружу показывается только нужный интерфейс;
- наследование: переиспользование общего поведения;
- полиморфизм: единый интерфейс для разных реализаций.

## Где полезно аналитику

- понимать влияние изменений на связанные классы/модули;
- корректно описывать ответственность компонентов;
- закладывать расширяемость в требованиях.

## Пример структуры

```kroki-plantuml
@startuml
class Order {
  +id
  +status
  +pay()
}
class PaymentService {
  +authorize(order)
}
class NotificationService {
  +send(order)
}

Order --> PaymentService
Order --> NotificationService
@enduml
```

## Типичные ошибки

- чрезмерная иерархия наследования;
- бизнес-логика распределена по сервисам без явного доменного центра;
- нарушение инкапсуляции через прямой доступ к внутреннему состоянию.

## Чек-лист качества

- Ответственность классов четко разделена.
- Зависимости между модулями контролируемы.
- Бизнес-правила тестируются на уровне доменной логики.

## Практическое дополнение

- Проведите мини-аудит текущего процесса по схеме: `требование -> реализация -> тест -> эксплуатация`.
- Зафиксируйте одно узкое место и добавьте улучшение в ближайший спринт/итерацию.
- Проверьте эффект через 2 недели по измеримой метрике (дефекты, lead time, MTTR или SLA).
