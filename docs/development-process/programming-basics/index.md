# Основы программирования для аналитика

Подраздел дает системному аналитику прикладочную техническую базу: как читать реализацию, формулировать реализуемые требования и снижать число дефектов на стыке анализа и разработки.

## Уровни сложности

### Базовый уровень

- понимать базовый синтаксис Python, Java, JavaScript и [SQL](../../glossary.md#abbr-084);
- различать переменные, функции, условия, циклы, коллекции;
- читать простой код и соотносить его с требованиями.

### Средний уровень

- понимать структуру приложения: слои, модули, [API](../../glossary.md#abbr-005), репозитории;
- распознавать типовые паттерны и anti-patterns;
- готовить технически корректные user stories и [AC](../../glossary.md#abbr-002).

### Продвинутый уровень

- оценивать архитектурные последствия требований;
- обсуждать trade-off по производительности и поддерживаемости;
- участвовать в проектировании контрактов и data-моделей.

## Мини-курс: что изучить

| Тема | Зачем аналитику |
| --- | --- |
| Python/Java/JavaScript основы | понимать реализацию бизнес-логики |
| SQL | проверять данные, писать диагностические запросы |
| Структуры данных | оценивать сложность и ограничения алгоритмов |
| Паттерны проектирования | формулировать расширяемые требования |
| Чтение кода и PR | быстрее выявлять расхождения с требованиями |

## Пример связи требований и кода

[Требование](../../glossary.md#term-071): "Система должна отклонять платежи с истекшим токеном".

```python
from datetime import datetime, timezone

def validate_token(expire_at_utc: datetime) -> bool:
    return datetime.now(timezone.utc) <= expire_at_utc
```

Аналитику важно проверить:

- явно ли указан timezone;
- что происходит при граничном значении времени;
- есть ли сообщение об ошибке, понятное пользователю/поддержке.
