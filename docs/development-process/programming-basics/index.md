# Основы программирования

Подраздел дает аналитику базовое понимание того, как устроен код и почему
некоторые требования реализуются проще или сложнее.

## Зачем это системному аналитику

- формулировать требования с учетом технической реализуемости;
- быстрее обсуждать решения с разработкой;
- видеть риски в логике, структуре данных и интеграциях.

## Практическое дополнение

- Проведите мини-аудит текущего процесса по схеме: `требование -> реализация -> тест -> эксплуатация`.
- Зафиксируйте одно узкое место и добавьте улучшение в ближайший спринт/итерацию.
- Проверьте эффект через 2 недели по измеримой метрике (дефекты, lead time, MTTR или SLA).
