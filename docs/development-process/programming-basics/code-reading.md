# Чтение кода

Чтение кода помогает аналитику быстро проверять реализованное поведение,
находить расхождения с требованиями и точнее формулировать изменения.

## Что смотреть в первую очередь

- точка входа сценария (контроллер, handler, use case);
- доменные проверки и бизнес-правила;
- интеграции с внешними сервисами и БД;
- обработка ошибок и граничных условий.

## Подход к разбору

1. Начните с пользовательского сценария и найдите его entry point в коде.
1. Пройдите основной поток данных до точки сохранения/ответа.
1. Отдельно проверьте негативные ветки и fallback-логику.
1. Зафиксируйте наблюдения в формате "факт -> риск -> предложение".

## Мини-чек-лист для аналитика

- Реализация совпадает с формулировкой требования.
- Исключения и ошибки обрабатываются явным образом.
- Логика интеграций отражает согласованные контракты.

## Типичные ошибки

- чтение только happy path без негативных сценариев;
- игнорирование конфигурации и feature toggles;
- выводы по коду без проверки runtime-логов и тестов.
