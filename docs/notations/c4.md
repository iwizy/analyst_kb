# C4

Диаграммы C4 (Context, Container, Component, Code) — это модель визуализации архитектуры программных систем, разработанная Саймоном Брауном. Она предназначена для упрощения архитектурных описаний с разными уровнями детализации.

C4 включает 4 уровня:

- Context (Контекст) — обзор системы и её окружения.
- Container (Контейнер) — основные элементы системы (серверы, базы данных, сервисы).
- Component (Компонент) — структура контейнера, основные компоненты внутри.
- Code (Код) — детали реализации отдельных классов или модулей.

## Уровень 1: Диаграмма Контекста

На этом уровне показывается система в окружении пользователей и других систем.

Объяснение:

- Пользователь взаимодействует с системой.
- Система взаимодействует с внешней системой.

## Уровень 2: Диаграмма Контейнеров

Показывает основные компоненты внутри системы: веб-приложение, базу данных, API и т. д.

Объяснение:

- Веб-приложение взаимодействует с базой данных.
- Веб-приложение также взаимодействует с внешним API.

## Уровень 3: Диаграмма Компонентов

Показывает ключевые части контейнера (например, внутри веб-приложения).

Объяснение:

- Контроллер принимает запросы.
- Сервис выполняет бизнес-логику.
- Репозиторий работает с базой данных.

## Уровень 4: Диаграмма Кода

Этот уровень детализирует конкретную реализацию (например, классы в репозитории).

Объяснение:

- UserRepository использует ORM (например, Hibernate).
- ORM взаимодействует с базой данных.

## Заключение

Диаграммы C4 позволяют структурированно описывать архитектуру системы, начиная с общего контекста и углубляясь до уровня кода.
