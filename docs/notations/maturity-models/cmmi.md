# CMMI

CMMI (Capability Maturity Model Integration) - модель зрелости процессов,
которая помогает организациям повышать предсказуемость качества, сроков и
стоимости поставки.

## Где применяется

- разработка и сопровождение ПО;
- сервисные ИТ-процессы;
- инженерные организации с высокой регуляторной нагрузкой.

## Уровни зрелости CMMI

| Уровень | Смысл |
| --- | --- |
| 1. Initial | Процессы нестабильны и зависят от индивидуальных усилий |
| 2. Managed | Процессы планируются и выполняются управляемо |
| 3. Defined | Процессы стандартизованы на уровне организации |
| 4. Quantitatively Managed | Управление на основе статистики и метрик |
| 5. Optimizing | Непрерывное улучшение и предотвращение дефектов |

## Пример дорожной карты перехода с 2 на 3 уровень

- унифицировать шаблоны требований, дизайна и тест-планов;
- ввести регулярные peer review и контроль отклонений;
- закрепить процесс управления изменениями требований;
- обучить команды стандартам и критериям "Definition of Done".

## Как аналитик применяет CMMI

1. Использует модель как основу диагностики процесса требований.
1. Фиксирует доказательства зрелости: артефакты, метрики, регламенты.
1. Формирует backlog улучшений с приоритетом по риску.
1. Контролирует эффект через качество требований и predictability релизов.

## Типичные ошибки

- формальное "собрали документы" без изменения практик;
- перегрузка команды бюрократией вместо полезных стандартов;
- отсутствие метрик, подтверждающих рост зрелости.

## Чек-лист качества

- Есть единые обязательные артефакты по жизненному циклу.
- Процесс review работает и оставляет след в инструментах.
- Метрики дефектов/переработок демонстрируют устойчивое улучшение.
