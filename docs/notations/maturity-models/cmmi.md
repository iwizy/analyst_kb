# [CMMI](../../glossary.md#abbr-018)

CMMI (Capability Maturity Model Integration) оценивает зрелость процессов разработки и поставки ПО.

## Уровни сложности

### Базовый уровень

- понимать 5 уровней зрелости CMMI;
- оценивать текущий процесс на уровне команды;
- выявлять ключевые gap между уровнями.

### Средний уровень

- планировать переход уровнями (например, 2 -> 3);
- внедрять процессные области с метриками;
- использовать CMMI в quality improvement цикле.

### Продвинутый уровень

- связывать CMMI с бизнес-целями и [SLO](../../glossary.md#abbr-081);
- управлять зрелостью в масштабе программы/портфеля;
- интегрировать CMMI с Agile/DevOps практиками.

## Уровни зрелости

| Уровень | Характеристика |
| --- | --- |
| 1 Initial | процесс непредсказуем и реактивен |
| 2 Managed | процесс управляем на уровне проекта |
| 3 Defined | стандартизированные организационные процессы |
| 4 Quantitatively Managed | управление на основе количественных метрик |
| 5 Optimizing | непрерывное улучшение на основе данных |

## Пример практического применения

Контекст: продуктовая команда в финтехе на уровне 2 хочет выйти на уровень 3.

- стандартизируют требования и ревью-процедуры;
- вводят общий [DoR](../../glossary.md#abbr-027)/[DoD](../../glossary.md#abbr-026);
- внедряют шаблоны дефект-анализа;
- фиксируют процесс в внутреннем стандарте и обучают команды.

## Плюсы и минусы

| Плюсы | Минусы |
| --- | --- |
| системный подход к процессному качеству | может восприниматься как бюрократия |
| помогает стабилизировать delivery | внедрение требует ресурса |
| поддерживает масштабирование команд | риск "галочного" соответствия |

## Как строить roadmap улучшений

1. Определите целевой уровень зрелости и почему он нужен.
2. Выберите 3-5 process areas с максимальным эффектом.
3. Установите метрики успеха (cycle time, defect leakage, predictability).
4. Запланируйте инициативы и owners.
5. Проводите регулярную переоценку.

## Типичные ошибки и как избежать

| Ошибка | Как избежать |
| --- | --- |
| фокус на документах вместо результата | связывайте инициативы с [KPI](../../glossary.md#abbr-047) |
| слишком широкий scope | начинайте с приоритетных областей |
| нет поддержки руководства | закрепите sponsorship и owners |
| отсутствие обратной связи | проводите регулярные ретро и ревью метрик |
