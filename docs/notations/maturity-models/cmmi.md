# CMMI

CMMI (Capability Maturity Model Integration) - модель зрелости процессов разработки и поставки, которая повышает предсказуемость качества, сроков и стоимости.

## Цель и задачи

- стабилизировать процесс поставки;
- стандартизировать практики требований, разработки, тестирования и релизов;
- перейти к управлению на основе метрик, а не интуиции.

## Уровни зрелости

| Уровень | Суть |
| --- | --- |
| 1. Initial | Процессы хаотичны, результат зависит от отдельных людей |
| 2. Managed | Процессы планируются и контролируются на уровне проектов |
| 3. Defined | Организация использует единые стандартизованные процессы |
| 4. Quantitatively Managed | Процессы управляются статистически и по KPI |
| 5. Optimizing | Непрерывные улучшения и предупреждение дефектов |

## Пример практического применения

Сценарий: команда застряла между уровнями 2 и 3.

| Область | Текущее состояние | Улучшение |
| --- | --- | --- |
| Требования | разные шаблоны по командам | единый шаблон + критерии качества |
| Review | нерегулярный | обязательный peer review с чек-листом |
| Изменения | фиксируются частично | formal change process + трассировка |
| Метрики | только velocity | дефекты, rework, predictability, lead time |

## Пошаговый алгоритм перехода (2 -> 3)

1. Введите единые обязательные артефакты по требованиям и дизайну.
1. Формализуйте процесс review и ответственности ролей.
1. Настройте контроль изменений требований и связь с тестами.
1. Обучите команды и закрепите практики в `Definition of Done`.
1. Ежемесячно оценивайте соблюдение и качество результатов.

## Типовые ошибки

- "бумажное" внедрение процессов без изменений поведения;
- перегрузка команды шаблонами без практической пользы;
- метрики собираются, но не используются в управлении.

## Визуальные рекомендации

- используйте таблицу разрывов `as-is/to-be` по ключевым process areas;
- показывайте прогресс по кварталам с целевыми KPI;
- разделяйте обязательные и рекомендуемые практики.

## Связь с другими нотациями

- `BPMN/UML/DFD`: стандартизированные модели как часть evidence зрелости.
- `Модели требований`: зрелость сильно зависит от качества requirements-практик.
- `Карта целей`: целевой уровень CMMI должен быть привязан к бизнес-целям.
