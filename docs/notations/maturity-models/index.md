# Модели зрелости

Модели зрелости используются для оценки текущего состояния практик и планирования целевого уровня. Для системного аналитика это инструмент диагностики процессов, аргументации улучшений и управления изменениями.

## Цель и задачи

- объективно оценить текущий уровень практик;
- определить целевое состояние и разрывы;
- сформировать реалистичный roadmap развития.

## Когда применять

- процесс нестабилен и сильно зависит от отдельных сотрудников;
- есть требования аудита, комплаенса или масштабирования;
- нужно приоритизировать улучшения по максимальному бизнес-эффекту.

## Сравнение моделей

| Модель | Фокус | Шкала | Типичный результат |
| --- | --- | --- | --- |
| [CMMI](cmmi.md) | инженерные и delivery-процессы | уровни зрелости 1-5 | стандартизация и предсказуемость поставки |
| [COBIT](cobit.md) | ИТ-governance и контроль | capability 0-5 | управляемость ИТ-процессов и рисков |
| [ACMM](acmm.md) | архитектурная функция | уровни зрелости 1-5 | системное архитектурное управление |

## Общий подход к оценке

1. Определите scope: процесс, подразделение, продуктовый поток.
1. Выберите модель и критерии оценки.
1. Соберите evidence: регламенты, артефакты, метрики, интервью.
1. Зафиксируйте текущий и целевой уровни по доменам.
1. Сформируйте backlog улучшений и контрольные метрики.

## Пример цикла развития

```kroki-plantuml
@startuml
start
:Оценка текущего уровня;
:Определение целевого уровня;
:План улучшений;
:Внедрение изменений;
:Измерение эффекта;
if (Цель достигнута?) then (Да)
  stop
else (Нет)
  :Корректировка плана;
  -> :Внедрение изменений;
endif
@enduml
```

## Типовые ошибки

- оценка "по ощущениям" без подтверждающих артефактов;
- попытка поднять все домены сразу;
- отсутствие связи между уровнем зрелости и бизнес-рисками.

## Визуальные рекомендации

- показывайте результаты по доменам (тепловая карта, radar chart, таблица разрывов);
- разделяйте "текущий" и "целевой" уровень явно;
- фиксируйте владельца и срок для каждого улучшения.

## Связь с другими нотациями

- `BPMN`, `DFD`, `UML`, `C4`: служат evidence зрелости практик моделирования.
- `Карта целей` и `Impact Map`: задают бизнес-контекст для выбора приоритетов улучшений.
