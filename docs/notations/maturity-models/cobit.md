# COBIT

COBIT (Control Objectives for Information and Related Technologies) - framework ИТ-управления и контроля. Он связывает ИТ-процессы с бизнес-целями, рисками и комплаенсом.

## Цель и задачи

- выстроить управляемость ИТ на уровне организации;
- обеспечить контроль рисков, качества и соответствия требованиям;
- улучшить прозрачность ИТ-метрик для руководства.

## Домены COBIT

| Домен | Назначение |
| --- | --- |
| EDM | Evaluate, Direct, Monitor - управление на уровне руководства |
| APO | Align, Plan, Organize - планирование и организация ИТ |
| BAI | Build, Acquire, Implement - поставка изменений |
| DSS | Deliver, Service, Support - эксплуатация и поддержка |
| MEA | Monitor, Evaluate, Assess - мониторинг и аудит |

## Capability уровни

| Уровень | Характеристика |
| --- | --- |
| 0 Incomplete | процесс отсутствует |
| 1 Performed | процесс выполняется нерегулярно |
| 2 Managed | процесс управляется на базовом уровне |
| 3 Established | процесс стандартизован |
| 4 Predictable | процесс измеряется и прогнозируется |
| 5 Optimizing | непрерывное улучшение |

## Пример: оценка процесса DSS02

| Критерий | Текущий статус | Цель |
| --- | --- | --- |
| SLA инцидентов | соблюдение 82% | >= 95% |
| RCA инцидентов | нерегулярно | обязательно для P1/P2 |
| Аналитика MTTR | частичная | полный дашборд с трендом |
| Категоризация | разнородная | единый справочник |

## Пошаговый алгоритм улучшения

1. Выберите 2-3 критичных процесса COBIT, а не весь каталог сразу.
1. Оцените capability уровень по evidence.
1. Определите целевой уровень с учетом бизнес-риска.
1. Сформируйте план улучшений с владельцами и сроками.
1. Встройте метрики процесса в регулярный управленческий цикл.

## Типовые ошибки

- внедрение COBIT как только аудиторской инициативы;
- оценка без проверяемых evidence;
- попытка одинаково быстро поднять все процессы.

## Визуальные рекомендации

- используйте матрицу `процесс x capability` с цветовой индикацией;
- отдельно показывайте метрики риска и метрики эффективности;
- визуализируйте roadmap повышения capability по кварталам.

## Связь с другими нотациями

- `Модели процессов (BPMN/eEPC)`: формализуют процессы для оценки COBIT.
- `C4/ArchiMate`: показывают архитектурный контекст управляемых процессов.
- `Модели зрелости`: CMMI и ACMM дополняют COBIT в delivery и архитектуре.
