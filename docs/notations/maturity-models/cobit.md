# COBIT

COBIT (Control Objectives for Information and Related Technologies) -
framework для управления и контроля ИТ. Он помогает выстроить governance,
связать ИТ-процессы с бизнес-целями и управлять рисками.

## Что покрывает COBIT

- EDM: Evaluate, Direct, Monitor (управление на уровне руководства).
- APO: Align, Plan, Organize (планирование и организация).
- BAI: Build, Acquire, Implement (поставка изменений).
- DSS: Deliver, Service, Support (эксплуатация и поддержка).
- MEA: Monitor, Evaluate, Assess (контроль и аудит).

## Шкала capability (0-5)

| Уровень | Характеристика |
| --- | --- |
| 0 Incomplete | Процесс фактически отсутствует |
| 1 Performed | Процесс выполняется эпизодически |
| 2 Managed | Есть планирование и базовое управление |
| 3 Established | Процесс стандартизован |
| 4 Predictable | Процесс управляется количественно |
| 5 Optimizing | Непрерывное улучшение процесса |

## Пример: оценка DSS02 (Service Requests and Incidents)

| Критерий | Наблюдение | Уровень |
| --- | --- | --- |
| SLA инцидентов | SLA есть, соблюдение 82% | 2 |
| Root cause analysis | Проводится нерегулярно | 1-2 |
| Метрики трендов | Нет стабильной аналитики MTTR | 1 |

Решение для роста до уровня 3:

- стандартизировать классификацию инцидентов;
- ввести обязательный post-incident review;
- построить дашборд MTTR/повторяемости инцидентов.

## Типичные ошибки

- внедрение COBIT как чисто аудиторской активности;
- оценка уровней без проверяемых evidence;
- попытка выровнять все процессы одновременно.

## Чек-лист качества

- Для выбранных процессов есть измеримые целевые уровни.
- Связь ИТ-метрик с бизнес-рисками формализована.
- Улучшения процессов закреплены в регулярном управленческом цикле.
