# eEPC

eEPC (extended Event-driven Process Chain) описывает процесс как цепочку событий и функций с организационными и информационными объектами.

## Уровни сложности

### Базовый уровень

- отличать события и функции;
- строить линейную цепочку процесса;
- использовать простые логические коннекторы.

### Средний уровень

- связывать функции с ролями и документами;
- применять XOR/AND/OR развилки;
- строить value chain на уровне предприятия.

### Продвинутый уровень

- интегрировать eEPC в enterprise-репозиторий процессов;
- использовать для SAP blueprint и регламентных процессов;
- связывать eEPC с [KPI](../glossary.md#abbr-047) и контрольными точками.

## Элементы eEPC

| Элемент | Назначение |
| --- | --- |
| Event | состояние/условие до и после функции |
| Function | действие процесса |
| Connector (AND/OR/XOR) | ветвление/слияние |
| Organization Unit | роль/подразделение |
| Information Object | документ/данные |

## Пример цепочки создания ценности

```kroki-plantuml
@startuml
rectangle "Событие: заказ получен" as E1
rectangle "Функция: проверить заказ" as F1
rectangle "Событие: заказ проверен" as E2
rectangle "Функция: выставить счет" as F2
rectangle "Событие: счет отправлен" as E3
rectangle "Функция: подтвердить оплату" as F3
rectangle "Событие: заказ готов к отгрузке" as E4

E1 --> F1
F1 --> E2
E2 --> F2
F2 --> E3
E3 --> F3
F3 --> E4
@enduml
```

## Плюсы и минусы

| Плюсы | Минусы |
| --- | --- |
| хорошая связка событий и операций | меньше распространен, чем [BPMN](../glossary.md#abbr-009) |
| удобен для enterprise процессного анализа | меньше modern tooling |
| подходит для value-chain описаний | ниже читаемость для внешних команд |

## Когда выбирать

- нужен событийный взгляд на процесс и организационные связи;
- используется ARIS/SAP-практика;
- важна формализация цепочки создания ценности.

## Как строить

1. Сформулируйте стартовое и конечное событие.
2. Разложите процесс на функции между событиями.
3. Добавьте развилки AND/OR/XOR.
4. Привяжите роли и информационные объекты.
5. Проверьте, что функция окружена событиями до/после.

## Типичные ошибки и как избежать

| Ошибка | Как избежать |
| --- | --- |
| функции без событий | соблюдайте правило event-function-event |
| слишком длинная цепочка в одном листе | делите на подцепочки |
| неоднозначные события | называйте событие как факт состояния |
| нет связи с владельцем процесса | добавляйте орг-единицы и ответственность |
