# UML

UML (Unified Modeling Language) - стандарт визуального моделирования для
проектирования и документирования систем. В практике системного аналитика UML
закрывает две ключевые задачи: понять поведение системы и описать ее структуру.

## Когда UML особенно полезен

- проектируются новые сценарии взаимодействия между сервисами;
- нужно формализовать бизнес-логику и переходы состояний;
- требуется передать проектное решение между командами без потери смысла.

## Как выбрать тип UML-диаграммы

| Вопрос | Диаграмма |
| --- | --- |
| Как участники обмениваются сообщениями во времени? | [Диаграмма последовательности](sequence.md) |
| Как выглядит поток шагов и условий в процессе? | [Диаграмма активности](activity.md) |
| Какие сущности есть в модели и как они связаны? | [Диаграмма классов](class.md) |
| Как объект меняет состояние по событиям? | [Диаграмма состояний](state-machine.md) |

## Практика применения UML

- Используйте одну диаграмму для одной цели коммуникации.
- Соблюдайте единый словарь терминов между UML и требованиями.
- Держите диаграммы актуальными: обновление вместе с изменением требований.
