# Диаграмма классов

Диаграмма классов (Class Diagram) — это один из ключевых видов диаграмм в языке UML (Unified Modeling Language). Она представляет структуру системы, включая классы, их атрибуты, методы и связи между классами.

Диаграмма классов используется для:

- Анализа предметной области (моделирование реальных объектов и их взаимосвязей).
- Проектирования программного обеспечения (создание архитектуры системы).
- Документирования существующих систем.
- Обмена информацией между разработчиками, аналитиками и архитекторами.
## 2. Основные элементы диаграммы классов

Диаграмма классов состоит из множества элементов, каждый из которых имеет свою роль. Рассмотрим их подробно.

### 2.1. Классы (Class)

Класс — это шаблон (абстракция), описывающий объекты с одинаковыми свойствами и поведением. На диаграмме классы изображаются в виде прямоугольников, разделенных на три секции:

```
+------------------+ | Имя класса | +------------------+ | Атрибуты | +------------------+ | Методы | +------------------+
```

Пример класса Person:

Обозначения атрибутов и методов:

- - (минус) — приватный (private): доступен только внутри класса.
- + (плюс) — публичный (public): доступен извне.
- # (решётка) — защищённый (protected): доступен внутри класса и его наследников.
- ~ (тильда) — пакетный (package-private): доступен только внутри пакета.
### Атрибуты (Attributes)

Атрибут — это переменная, хранящая состояние объекта. Записывается в формате:

```
[модификатор доступа] имя: тип
```

Примеры:

- - name: String → приватное строковое поле name
- + salary: double → публичное числовое поле salary
- # birthDate: Date → защищённое поле birthDate
### Методы (Operations)

Методы определяют поведение класса. Записываются в формате:

```
[модификатор доступа] имяМетода(параметры): возвращаемый_тип
```

Примеры:

- + getName(): String → публичный метод getName(), возвращающий String
- - calculateSalary(): double → приватный метод calculateSalary(), возвращающий double
- # setAge(newAge: int): void → защищённый метод setAge(), принимающий int
## Связи между классами

Классы могут взаимодействовать друг с другом разными способами. Рассмотрим основные виды связей.

### Ассоциация (Association)

Ассоциация описывает отношение между классами (один объект связан с другим). На диаграмме обозначается сплошной линией.

Пример: Учитель обучает Студента.

Множественность (Multiplicity):

- 1 — один объект.
- * — много объектов.
- 0..1 — ноль или один объект.
- 1..* — минимум один объект.
### Наследование (Generalization)

Наследование описывает отношение "является" (is-a), где один класс получает свойства и методы другого.

Пример: Employee — это Person.

Employee наследует Person, получая все его атрибуты и методы.

### Агрегация (Aggregation)

Агрегация описывает отношение "содержит" (has-a), но объекты могут существовать отдельно.

Пример: Машина содержит Двигатель, но двигатель может существовать без машины.

Полый ромб означает агрегацию.

### Композиция (Composition)

Композиция — это сильный тип агрегации, где объекты не могут существовать раздельно.

Пример: Дом состоит из Комнат, и без Дома комнаты не существуют.

Заполненный ромб означает композицию.

### Зависимость (Dependency)

Зависимость означает, что один класс использует другой, но не владеет им.

Пример: Сервис использует Логгер.

Пунктирная стрелка обозначает зависимость.

- Моделировать связи между объектами
- Понимать структуру проекта
- Документировать существующие системы
