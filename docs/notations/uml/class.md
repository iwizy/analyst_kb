# Диаграмма классов

Диаграмма классов (Class Diagram) — один из ключевых видов диаграмм UML
(Unified Modeling Language). Она показывает структуру системы: классы,
их атрибуты, методы и связи между ними.

Диаграмма классов используется для:

- анализа предметной области;
- проектирования программного обеспечения;
- документирования существующих систем;
- коммуникации между аналитиками, разработчиками и архитекторами.

## Основные элементы диаграммы классов

Диаграмма классов состоит из нескольких базовых элементов.

### Класс

Класс — это абстракция, описывающая объекты с одинаковыми свойствами
и поведением. На диаграмме класс обычно изображается в виде
прямоугольника, разделенного на три секции.

```text
+------------------+
| Имя класса       |
+------------------+
| Атрибуты         |
+------------------+
| Методы           |
+------------------+
```

### Обозначения атрибутов и методов

- `-` — приватный (`private`), доступен только внутри класса.
- `+` — публичный (`public`), доступен извне.
- `#` — защищенный (`protected`), доступен в классе и его наследниках.
- `~` — пакетный (`package-private`), доступен внутри пакета.

### Атрибуты

Атрибут — это переменная, которая хранит состояние объекта.

Формат записи:

```text
[модификатор доступа] имя: тип
```

Примеры:

- `- name: String`
- `+ salary: double`
- `# birthDate: Date`

### Методы

Методы определяют поведение класса.

Формат записи:

```text
[модификатор доступа] имяМетода(параметры): возвращаемый_тип
```

Примеры:

- `+ getName(): String`
- `- calculateSalary(): double`
- `# setAge(newAge: int): void`

## Связи между классами

### Ассоциация

Ассоциация описывает связь между классами. Обычно обозначается сплошной
линией.

### Наследование (Generalization)

Наследование описывает отношение «является» (`is-a`), когда один класс
получает свойства и методы другого.

### Агрегация (Aggregation)

Агрегация описывает отношение «содержит» (`has-a`), при котором объекты
могут существовать отдельно.

### Композиция (Composition)

Композиция — сильная форма агрегации, при которой части не существуют
отдельно от целого.

### Зависимость (Dependency)

Зависимость означает, что один класс использует другой, но не владеет им.

## Итог

Диаграмма классов UML помогает:

- проектировать архитектуру;
- моделировать связи между объектами;
- понимать структуру проекта;
- поддерживать актуальную документацию.
