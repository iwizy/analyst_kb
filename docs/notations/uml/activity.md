# Activity

Диаграмма активности (Activity Diagram) в Unified Modeling Language (UML) — это графическое представление динамического поведения системы. Она используется для моделирования процессов, алгоритмов и рабочих потоков.

Диаграмма активности показывает последовательность действий, которые выполняются в рамках определенного процесса, а также разветвления, условия, циклы и параллельные операции.

## Где применяется диаграмма активности?

Диаграмма активности может применяться в различных сферах:

- Моделирование бизнес-процессов: показывает этапы выполнения бизнес-операций.
- Проектирование алгоритмов: визуализация последовательности вычислений.
- Анализ сценариев использования (Use Case): показывает, как пользователи взаимодействуют с системой.
- Описания рабочих процессов (Workflow): для документирования процессов внутри организации.
- Проектирование программного обеспечения: для описания логики поведения объектов.
## Основные элементы диаграммы активности

Диаграмма активности состоит из нескольких ключевых элементов.

### Начальный узел (Initial Node)

### Действие (Action)

Действие представляет собой конкретную операцию или шаг в процессе. Обозначается прямоугольником с закругленными углами.

- Выполняется действие B (Проверка данных).
- Затем выполняется C (Сохранение информации).
### Переход (Transition)

Переход обозначает направление выполнения действий. Он связывает начальный узел, действия, решения и конечные узлы.

### Решение (Decision Node)

Решение представляет собой ветвление процесса и изображается в виде ромба. Ветвление происходит на основе логического условия (например, "да" или "нет").

- В обоих случаях процесс завершается (E).
### Объединение потоков (Merge Node)

Объединение потоков позволяет соединить несколько ветвей в одну. Оно также изображается в виде ромба.

В предыдущем примере E является узлом объединения: независимо от исхода проверки, процесс приходит в один и тот же конец.

Fork используется для параллельного выполнения нескольких действий, а Join — для их объединения обратно в один поток.

## Полный пример диаграммы активности

Рассмотрим более сложный процесс — авторизацию пользователя.

### Описание процесса

- Пользователь вводит логин и пароль.
- Проверяется корректность введенных данных.
- Если данные неверные — отображается ошибка авторизации.
- Если данные верные — система проверяет, существует ли такой пользователь в базе данных.
- Если пользователь найден, выполняется успешная авторизация.
- Если пользователя нет, отображается ошибка авторизации.

### Разбор элементов

- A — начальный узел.
- B — ввод логина и пароля.
- C — проверка корректности данных.
- D — ошибка авторизации, если данные неверные.
- E — проверка пользователя в базе данных.
- F — условие: найден ли пользователь?
- G — успешная авторизация.
- H — завершение процесса.
## Итоги

### Что мы узнали?

- Диаграмма активности помогает визуализировать процессы и алгоритмы.
- Ключевые элементы: начальный и конечный узлы, действия, решения, переходы, разветвления (fork), объединения (join).
- Применение: моделирование бизнес-процессов, описание алгоритмов, проектирование программного обеспечения.
### Когда использовать?

- Когда нужно показать последовательность действий в процессе.
- Когда есть сложные ветвления и условия.
- Когда необходимо продемонстрировать параллельные действия.
