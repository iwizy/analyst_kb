# Диаграмма активности

Диаграмма активности (Activity Diagram) показывает поток действий,
ветвления, параллельность и завершение процесса. Это рабочий инструмент для
описания бизнес-процессов и алгоритмов обработки в системе.

## Ключевые элементы

- Initial Node: старт процесса.
- Action: шаг выполнения.
- Decision/Merge: ветвление и объединение ветвей.
- Fork/Join: параллельные потоки и их синхронизация.
- Final Node: завершение процесса.

## Пример: регистрация пользователя

```kroki-plantuml
@startuml
start
:Ввести email и пароль;
:Проверить формат данных;
if (Данные корректны?) then (Да)
  :Создать учетную запись;
  fork
    :Отправить welcome email;
  fork again
    :Создать профиль по умолчанию;
  end fork
  :Показать экран успеха;
  stop
else (Нет)
  :Показать ошибку;
  stop
endif
@enduml
```

## Как строить диаграмму

1. Зафиксируйте границы процесса: вход, выход, триггер старта.
1. Опишите основной поток (happy path).
1. Добавьте альтернативные ветки и исключения.
1. Вынесите параллельные шаги через fork/join.
1. Проверьте процесс на тест-кейсах из требований.

## Где особенно полезна

- описание пользовательских и операционных сценариев;
- согласование автоматизации процесса в BPM/Workflow;
- подготовка критериев приемки и тест-кейсов.

## Типичные ошибки

- нет конечных состояний для всех ветвей;
- в одной диаграмме смешаны разные уровни детализации;
- условия в decision не покрывают все варианты.

## Чек-лист качества

- Каждый шаг имеет понятный результат.
- Для каждой развилки определены условия.
- Все ветви приводят к контролируемому завершению.
