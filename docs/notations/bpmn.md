# BPMN

BPMN (Business Process Model and Notation) - стандарт моделирования
бизнес-процессов. Нотация помогает согласовать процесс между бизнесом,
аналитиками, разработкой и автоматизацией.

## Когда использовать

- нужно описать процесс "как есть" и "как будет";
- требуется формализовать регламент или сценарий в workflow-системе;
- есть несколько ролей/систем и важны точки передачи ответственности.

## Ключевые элементы

- Events: старт, промежуточные события, завершение.
- Activities: задачи и подпроцессы.
- Gateways: развилки (XOR/AND/OR).
- Pools/Lanes: участники процесса и зоны ответственности.
- Message Flow: обмен сообщениями между участниками.

## Пример: обработка заявки клиента

```kroki-plantuml
@startuml
|Клиент|
start
:Подать заявку;
|Аналитик|
:Проверить комплектность;
if (Пакет полный?) then (Да)
  :Оценить риск;
  if (Риск допустим?) then (Да)
    :Согласовать договор;
    stop
  else (Нет)
    :Отказать;
    stop
  endif
else (Нет)
  :Запросить уточнение;
  stop
endif
@enduml
```

Минимум для практики:

- у каждой задачи указан исполнитель;
- у каждого gateway есть условие ветвления;
- у каждого сценария есть конечное событие.

## Порядок работы аналитика

1. Определите начало и ожидаемый результат процесса.
1. Разметьте роли в lanes.
1. Разложите шаги в формате "глагол + объект".
1. Добавьте точки решений и исключения.
1. Проверьте процесс на реальных кейсах (happy path + edge cases).

## Типичные ошибки

- использование gateway без взаимоисключающих условий;
- смешение действий человека и системной обработки в одной задаче;
- отсутствие ошибок/таймаутов в модели.

## Чек-лист качества

- Диаграмма отвечает на вопрос "кто, что, когда делает".
- Видны SLA-точки и возможные задержки.
- Есть базовый и альтернативные сценарии.
- Процесс можно напрямую переложить в user stories или workflow-движок.
