# IDEF0

IDEF0 - нотация функционального моделирования, которая отвечает на вопрос: что делает система, чем управляется функция, какой результат дает и какими средствами выполняется.

## Цель и задачи

- разложить сложную функцию на управляемые подфункции;
- выделить входы, управления, выходы и механизмы (ICOM);
- согласовать границы ответственности до проектирования решения.

## Основные элементы ICOM

| Компонент | Значение |
| --- | --- |
| Input | Что преобразуется функцией |
| Control | Правила и ограничения выполнения |
| Output | Результат функции |
| Mechanism | Исполнитель и инструменты |

## Когда применять

- в предпроектном анализе и функциональной декомпозиции;
- при согласовании ролей и правил исполнения функций;
- когда требуется прозрачная структура "что делаем", а не "как технически".

## Пример A0

```kroki-plantuml
@startuml
left to right direction
rectangle "A0 Обработать заявку" as A0
rectangle "Input\n- Заявка клиента\n- Документы" as I
rectangle "Control\n- Политика KYC\n- SLA\n- Лимиты риска" as C
rectangle "Output\n- Решение\n- Уведомление" as O
rectangle "Mechanism\n- Аналитик\n- CRM\n- Скоринг" as M

I --> A0
C -down-> A0
A0 --> O
M -up-> A0
@enduml
```

## Пример декомпозиции A0 -> A1..A4

```kroki-plantuml
@startuml
left to right direction
rectangle "A0 Обработать заявку" as A0
rectangle "A1 Проверить комплектность" as A1
rectangle "A2 Оценить риск" as A2
rectangle "A3 Принять решение" as A3
rectangle "A4 Уведомить клиента" as A4

A0 ..> A1
A0 ..> A2
A0 ..> A3
A0 ..> A4
@enduml
```

## Пошаговый алгоритм построения

1. Определите контекстную функцию `A-0` и ее границы.
1. Заполните ICOM для верхнего уровня `A0`.
1. Выполните декомпозицию на 3-6 функций.
1. Для каждой подфункции отдельно заполните ICOM.
1. Проверьте консистентность входов/выходов между уровнями.

## Типовые ошибки

- подмена `Control` входными данными;
- слишком глубокая декомпозиция без прикладной цели;
- отсутствие владельца функции и критериев результата.

## Визуальные рекомендации

- держите стабильную ориентацию ICOM: `Input` слева, `Control` сверху, `Output` справа, `Mechanism` снизу;
- используйте нумерацию блоков (`A0`, `A1`, `A2`) иерархически;
- ограничивайте блоки по длине текста, выносите детали в спецификацию.

## Связь с другими нотациями

- `BPMN/eEPC`: детализируют последовательность шагов, выявленных в IDEF0.
- `DFD`: добавляет структуру потоков данных между функциями.
- `C4/ArchiMate`: показывают, где эти функции реализуются в архитектуре.
