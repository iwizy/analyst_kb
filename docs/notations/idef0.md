# IDEF0

> Глоссарий терминов и сокращений: [Открыть](../glossary.md)


IDEF0 моделирует функции системы через четыре типа стрелок ICOM: Input, Control, Output, Mechanism.

## Уровни сложности

### Базовый уровень

- понимать ICOM-логику;
- строить A-0 (контекст) и A0 (верхний уровень);
- различать входы, управление, выходы и механизмы.

### Средний уровень

- декомпозировать функции до A1/A2/A3;
- связывать функции с ролями и ресурсами;
- использовать IDEF0 для анализа регламентов.

### Продвинутый уровень

- применять IDEF0 в промышленном и гос-секторе;
- использовать для аудита процесса и оптимизации ресурсов;
- связывать IDEF0 с BPMN/DFD и целевыми метриками.

## Модель ICOM

| Элемент | Расположение | Значение |
| --- | --- | --- |
| Input | слева | что преобразуется функцией |
| Control | сверху | правила и ограничения |
| Output | справа | результат функции |
| Mechanism | снизу | ресурсы/исполнители |

## Пример: верхний уровень A0

```kroki-plantuml
@startuml
left to right direction
rectangle "A0 Обработать заказ" as A0
rectangle "Вход: заявка клиента" as I
rectangle "Контроль: политика продаж" as C
rectangle "Выход: подтвержденный заказ" as O
rectangle "Механизм: CRM, оператор, склад" as M

I --> A0
C -down-> A0
A0 --> O
M -up-> A0
@enduml
```

## Пример декомпозиции A0

- A1: Проверить заказ
- A2: Проверить оплату
- A3: Резервировать товар
- A4: Подтвердить отгрузку

## Плюсы и минусы

| Плюсы | Минусы |
| --- | --- |
| очень четкая функциональная декомпозиция | менее привычна для продуктовых команд |
| удобна для регламентов и аудита | слабее показывает динамику времени |
| хорошо работает в инженерных процессах | требует обучения нотации ICOM |

## Когда выбирать

- нужно формально описать функцию и ее управление;
- важны контрольные ограничения и ресурсы выполнения;
- требуется стандартная декомпозиция в enterprise/industrial проектах.

## Как строить

1. Определите функцию A-0 и границы.
2. Опишите ICOM для A0.
3. Разбейте A0 на 3-6 функций A1..An.
4. Убедитесь в согласованности ICOM между уровнями.
5. Проведите ревью с владельцем процесса.

## Типичные ошибки и как избежать

| Ошибка | Как избежать |
| --- | --- |
| перепутаны control и input | проверяйте: control не преобразуется |
| декомпозиция без связи с A0 | валидируйте соответствие входов/выходов |
| слишком глубокая детализация | останавливайтесь на уровне управляемости |
| отсутствие механизма | всегда фиксируйте исполнителей/ресурсы |
