# Диаграмма Исикавы

Диаграмма Исикавы (fishbone, cause-and-effect) помогает структурировать причины
проблемы: от симптома к корневым факторам. Полезна при анализе инцидентов,
дефектов качества и отклонений KPI.

## Когда использовать

- есть повторяющаяся проблема, но причина неочевидна;
- команда предлагает противоречивые гипотезы;
- нужно перейти от "починили симптом" к системным улучшениям.

## Типовая структура причин

Чаще всего применяют категории 6M:

- People (люди);
- Process (процессы);
- Methods (методы);
- Machines (системы/инструменты);
- Materials (данные/входы);
- Measurement (метрики/контроль).

## Пример: рост ошибок при оформлении заказа

| Категория | Возможная причина | Проверка | Действие |
| --- | --- | --- | --- |
| People | Новые операторы не обучены | % ошибок по стажу | Ввести чек-лист и shadowing |
| Process | Нет обязательной валидации адреса | Отказы доставки | Добавить шаг верификации |
| Machines | Медленно отвечает сервис адресов | P95 API latency | Кэш + retry policy |
| Measurement | Нет метрики "ошибка оформления" | Пробелы в дашборде | Добавить KPI и алерты |

## Как проводить сессию

1. Сформулируйте эффект одной фразой ("Что болит?").
1. Соберите причины 1-го уровня по категориям.
1. Для каждой причины примените "5 Why" до проверяемой гипотезы.
1. Выделите причины с максимальным влиянием.
1. Зафиксируйте план экспериментов и владельцев.

## Типичные ошибки

- смешение фактов и мнений без данных;
- попытка сразу обсуждать решения вместо причин;
- отсутствие проверки гипотез после воркшопа.

## Чек-лист качества

- Проблема сформулирована измеримо.
- Для критичных причин есть способ верификации.
- По итогам сформирован список действий и сроков.
