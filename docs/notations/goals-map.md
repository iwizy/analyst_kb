# Карта целей

Карта целей (Goal Model) связывает стратегические цели, задачи, метрики и инициативы, чтобы показать "зачем" и "как" изменения создают ценность.

## Уровни сложности

### Базовый уровень

- формулировать цель и ожидаемый результат;
- связывать цель с измеримым KPI;
- строить иерархию целей и подцелей.

### Средний уровень

- связывать цели с требованиями и roadmap;
- проверять конфликтующие цели и зависимости;
- использовать карту для приоритизации backlog.

### Продвинутый уровень

- применять goal model в portfolio governance;
- связывать цели с архитектурными инициативами и рисками;
- выстраивать систему метрик достижения целей.

## Элементы карты целей

| Элемент | Назначение |
| --- | --- |
| Goal | желаемый бизнес-результат |
| Sub-goal | декомпозиция цели |
| KPI | измеритель достижения |
| Initiative | инициатива/эпик для достижения |
| Constraint | ограничение/допущение |

## Пример: снижение времени обработки возвратов

```kroki-plantuml
@startuml
left to right direction
rectangle "Goal\nСократить цикл возврата на 30%" as G
rectangle "Sub-goal\nАвтоматизировать проверку статуса заказа" as SG1
rectangle "Sub-goal\nУменьшить ручные проверки" as SG2
rectangle "KPI\nСредний цикл возврата <= 48ч" as KPI
rectangle "Initiative\nНовый сервис валидации возврата" as I

G --> SG1
G --> SG2
SG1 --> I
SG2 --> I
G --> KPI
@enduml
```

## Плюсы и минусы

| Плюсы | Минусы |
| --- | --- |
| фокусирует команду на ценности | требует регулярного обновления |
| дает основу для приоритизации | может быть слишком абстрактной без KPI |
| хорошо связывает бизнес и delivery | сложнее поддерживать в больших портфелях |

## Когда выбирать

- нужно связать стратегию и требования;
- команда спорит о приоритетах без общей цели;
- необходимо показать value chain от цели к реализации.

## Как строить

1. Зафиксируйте 1-2 верхнеуровневые цели.
2. Разложите их на подцели.
3. Добавьте измеримые KPI.
4. Привяжите инициативы/эпики и владельцев.
5. Сопоставьте с ограничениями и рисками.

## Типичные ошибки и как избежать

| Ошибка | Как избежать |
| --- | --- |
| цели без метрик | добавляйте KPI с формулой и сроком |
| смешение целей и задач | цель = результат, задача = действие |
| нет связи с backlog | привязывайте цели к эпикам/фичам |
| противоречивые цели | проводите конфликт-анализ целей |

## Связь с требованиями

- карту целей используйте как вход для [бизнес-требований](../requirements/levels/business.md);
- подцели декомпозируйте в [пользовательские и системные требования](../requirements/levels/index.md).

## Инструменты и форматы

- Miro, draw.io, FigJam, Archimate motivation view;
- экспорт: SVG/PNG/PDF.

## Контрольные вопросы

1. Каждая цель измерима и ограничена сроком?
2. Понятно ли, какие инициативы влияют на цель?
3. Есть ли конфликтующие цели между подразделениями?
4. Есть ли owner и период пересмотра карты?

## Чек-лист самопроверки

- цели связаны с KPI;
- подцели логично поддерживают верхний уровень;
- инициативы привязаны к целям;
- карта синхронизирована с roadmap;
- назначены владельцы и ревью-период.
