# Спиральная модель

Спиральная модель - риск-ориентированный процесс, в котором проект проходит
повторяющиеся витки: цели, анализ рисков, разработка, оценка результатов.

## Когда использовать

- проект технологически или предметно неопределен;
- высока стоимость ошибочного решения;
- нужны прототипы для снижения рисков до масштабной реализации.

## Виток спирали

```kroki-plantuml
@startuml
start
:Определить цели и варианты;
:Проанализировать и приоритизировать риски;
:Разработать прототип/инкремент;
:Оценить результат со стейкхолдерами;
if (Есть критичные риски?) then (Да)
  :План следующего витка;
  -> [повтор]
else (Нет)
  :Переход к поставке;
endif
stop
@enduml
```

## Практический пример

Сценарий: внедрение новой скоринговой модели.

- Виток 1: прототип модели + проверка на исторических данных.
- Виток 2: пилот на ограниченной группе пользователей.
- Виток 3: промышленный rollout с контролем качества.

## Сильные стороны

- раннее выявление критичных рисков;
- управляемая неопределенность;
- принятие решений на основе данных каждого витка.

## Типичные ошибки

- формальный риск-анализ без влияния на план;
- слишком длинные витки без проверяемого результата;
- отсутствие критериев выхода из спирали.

## Чек-лист качества

- На каждом витке есть измеримый результат.
- Риски пересматриваются и приоритизируются регулярно.
- Решение о переходе к следующему витку аргументировано данными.
