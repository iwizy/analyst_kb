# Waterfall

Waterfall - классическая предиктивная модель разработки, где этапы идут
последовательно: от требований к проектированию, реализации, тестированию и
вводу в эксплуатацию.

## Когда Waterfall уместен

- требования стабильны и заранее согласованы;
- проект имеет высокую регуляторную нагрузку;
- важны формальные этапные согласования и документация.

## Базовые этапы

| Этап | Основной результат |
| --- | --- |
| Анализ требований | утвержденная спецификация |
| Проектирование | архитектурное и техническое решение |
| Реализация | готовый функционал |
| Тестирование | подтверждение соответствия требованиям |
| Внедрение и поддержка | эксплуатация и сопровождение |

## Плюсы и ограничения

- Плюсы: высокая прогнозируемость, понятная ответственность по этапам.
- Ограничения: высокая стоимость поздних изменений, длинный feedback loop.
