# Incremental

Инкрементная модель делит поставку на последовательность функционально завершенных инкрементов.

## Принцип

Каждый инкремент проходит полный mini-SDLC и приносит бизнес-ценность.

## Этапы инкремента

1. Выбор scope инкремента.
1. Дизайн.
1. Реализация.
1. Тестирование.
1. Выпуск и сбор feedback.

## Артефакты

- release plan по инкрементам;
- критерии готовности каждого релиза;
- post-release feedback.

## Плюсы и минусы

Плюсы:

- ранняя поставка ценности;
- управляемый риск релиза.

Минусы:

- риск архитектурной фрагментации без общего каркаса;
- сложнее приоритизировать долгосрочные зависимости.

## Подходит для

- roadmap с поэтапным внедрением;
- продуктов, где важен ранний пользовательский feedback.
