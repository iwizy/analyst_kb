# Инкрементная модель

Инкрементная модель делит поставку на серию небольших релизов. Каждый
инкремент проходит полный цикл: анализ, разработка, тестирование, ввод.

## Когда использовать

- нужно быстрее давать бизнесу полезные части решения;
- есть возможность поэтапного ввода функциональности;
- важно снизить риск "большого релиза".

## Пример релизного плана

| Инкремент | Состав | Бизнес-эффект |
| --- | --- | --- |
| R1 | базовая регистрация и авторизация | запуск MVP |
| R2 | профиль клиента и уведомления | рост retention |
| R3 | биллинг и отчеты | монетизация и контроль |

## Схема поставки инкрементов

```kroki-plantuml
@startuml
start
:Инкремент 1 (MVP);
:Сбор обратной связи;
:Инкремент 2;
:Сбор обратной связи;
:Инкремент 3;
stop
@enduml
```

## Преимущества

- ранняя поставка ценности;
- управляемый риск на каждом релизе;
- возможность корректировки приоритетов между инкрементами.

## Типичные ошибки

- инкременты делятся по техническим слоям, а не по ценности;
- отсутствие архитектурного каркаса для будущего масштабирования;
- накопление интеграционного долга между релизами.

## Чек-лист качества

- Каждый инкремент дает законченный пользовательский результат.
- Между инкрементами есть явный feedback loop.
- Архитектура выдерживает постепенное наращивание функционала.
