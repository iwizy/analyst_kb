# V-модель

V-модель связывает каждый этап разработки с соответствующим уровнем тестирования.

## Принцип

Чем раньше зафиксированы критерии тестирования, тем ниже риск дефектов на поздних этапах.

## Соответствие этапов

- бизнес-требования -> acceptance tests;
- системные требования -> system tests;
- архитектурный дизайн -> integration tests;
- код -> unit tests.

## Артефакты

- requirement-test traceability matrix;
- тест-дизайн на раннем этапе;
- протоколы прохождения по уровням.

## Плюсы и минусы

Плюсы:

- сильная трассировка требования -> тест;
- раннее планирование качества.

Минусы:

- менее гибкая к изменяющимся требованиям;
- высокая стоимость документационного контура.

## Подходит для

- safety-critical и compliance-heavy систем;
- проектов с обязательным audit trail.
