# Agile

Agile - это семейство подходов к разработке, где приоритет отдается ценности
для пользователя, коротким итерациям и постоянной обратной связи.

## Когда Agile дает лучший результат

- требования меняются по мере проверки гипотез;
- скорость вывода инкрементов важнее "идеального плана";
- продукт развивается через learning loop с пользователями.

## Ключевые принципы

- поставка ценности небольшими шагами;
- тесное взаимодействие бизнеса и команды;
- регулярная проверка результата и пересмотр приоритетов;
- прозрачность процесса через визуализацию и метрики потока.

## Типовой цикл Agile-команды

```kroki-plantuml
@startuml
start
:Уточнить цели и приоритеты;
:Сформировать sprint/release backlog;
:Реализовать и протестировать инкремент;
:Провести review с бизнесом;
:Собрать обратную связь;
:Провести ретроспективу;
if (Нужна корректировка?) then (Да)
  :Обновить backlog и подход;
endif
stop
@enduml
```

## Артефакты и метрики

| Артефакт/метрика | Зачем нужен |
| --- | --- |
| Product backlog | единый приоритизированный список работ |
| Definition of Done | единый стандарт качества инкремента |
| Velocity/throughput | прогноз объема поставки |
| Lead time/cycle time | контроль скорости потока |
| Sprint goal | фокус итерации на бизнес-результате |

## Роль аналитика в Agile

- формулирует требования в формате, пригодном для реализации и тестирования;
- поддерживает актуальность backlog и связь задач с целями;
- помогает команде выявлять и снимать неопределенность до разработки;
- обеспечивает трассировку изменений требований между итерациями.

## Типичные ошибки

- подмена Agile хаотичным управлением без ритма и артефактов;
- отсутствие регулярного feedback от пользователя;
- фиксация всего объема заранее при декларации "гибкости".

## Чек-лист качества

- Каждая итерация заканчивается проверяемым инкрементом.
- Приоритеты привязаны к бизнес-целям и метрикам.
- Изменения требований управляются прозрачно и с причинами.
