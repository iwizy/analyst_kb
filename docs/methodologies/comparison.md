# Сравнение методологий

## Сводная таблица

| Методология | Изменчивость требований | Скорость feedback | Документационный контур | Масштаб |
| --- | --- | --- | --- | --- |
| Waterfall | низкая | низкая | высокий | средний/крупный |
| V-Model | низкая | средняя | высокий | крупный/regulatory |
| Spiral | высокая | средняя | средний | R&D/сложные |
| Incremental | средняя | средняя/высокая | средний | средний |
| Scrum | высокая | высокая | средний | команда/неск. команд |
| Kanban | высокая | высокая | низкий/средний | потоковые команды |
| XP | высокая | высокая | средний | инженерно-зрелые команды |
| RUP | средняя | средняя | высокий | крупный enterprise |
| FDD | средняя | средняя | средний | средний/крупный |
| DDD | высокая (доменно) | средняя | средний | доменно-сложные |
| SAFe | средняя/высокая | средняя | высокий | крупные программы |
| Lean | любая | средняя/высокая | средний | org-wide |

## Как связать методологию с требованиями и рисками

- Высокая неопределенность требований -> Agile + короткий цикл валидации.
- Высокая цена ошибки -> V-Model/Spiral + усиленное тестирование и риск-контур.
- Много команд и зависимостей -> SAFe/LeSS + централизованная синхронизация.
- Сложный домен -> DDD + контекстная декомпозиция требований.
