# Методологии и фреймворки

Подраздел содержит практические фреймворки для адаптивной и гибридной поставки. Все материалы оформлены по единому шаблону: принципы -> процесс -> роли/артефакты -> применение -> риски.

## Уровни сложности

### Базовый уровень

- понимать назначение каждого фреймворка;
- знать базовые роли и артефакты;
- уметь выбрать стартовый вариант для команды.

### Средний уровень

- адаптировать фреймворк под контекст продукта;
- внедрять метрики потока и качества;
- увязывать подход с требованиями и change management.

### Продвинутый уровень

- масштабировать практики на несколько команд;
- комбинировать фреймворки в гибридных контурах;
- оценивать эффективность процесса по метрикам.

## Быстрый выбор

| Контекст | Подход |
| --- | --- |
| Одна продуктовая команда | Scrum |
| Поток входящих задач/поддержка | Kanban |
| Нужен сильный инженерный контур | XP |
| Сложный домен | DDD + Scrum |
| Крупная программа 5+ команд | SAFe / LeSS / Nexus |
| Continuous improvement организации | Lean + PDCA |
| Формальный enterprise governance | RUP / Hybrid |
