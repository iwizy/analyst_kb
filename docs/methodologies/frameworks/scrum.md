# Scrum

Scrum - фреймворк для итеративной разработки продукта короткими спринтами.
Он задает роли, события и артефакты для предсказуемой поставки ценности.

## Основные роли

- Product Owner: отвечает за ценность и приоритеты backlog.
- Scrum Master: отвечает за процесс и устранение impediments.
- Developers: кросс-функциональная команда поставки.

## События Scrum

- Sprint planning;
- Daily Scrum;
- Sprint review;
- Sprint retrospective.

## Поток одного спринта

```kroki-plantuml
@startuml
start
:Уточнение Product Backlog;
:Планирование спринта;
:Daily + разработка и тестирование;
:Демонстрация инкремента (Review);
:Ретроспектива;
:Обновление backlog;
stop
@enduml
```

## Ключевые артефакты

| Артефакт | Назначение |
| --- | --- |
| Product Backlog | единый источник приоритетных задач |
| Sprint Backlog | объем работ на итерацию |
| Increment | готовая ценность по DoD |

## Метрики для контроля

- Sprint Goal success rate;
- Velocity;
- Defect leakage;
- Lead time от идеи до поставки.

## Типичные ошибки

- превращение Scrum в "набор встреч" без продуктовой цели;
- изменение Sprint backlog без прозрачной причины;
- отсутствие DoD и нестабильное качество инкремента.

## Чек-лист качества

- Sprint goal понятен всем участникам.
- Review дает бизнес-обратную связь по результату.
- Ретроспектива приводит к конкретным улучшениям процесса.
