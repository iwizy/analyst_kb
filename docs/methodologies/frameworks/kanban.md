# Kanban

Kanban - фреймворк управления потоком работ. Основная идея: визуализировать
работу, ограничить WIP и оптимизировать время прохождения задачи через систему.

## Ключевые принципы

- визуализация потока на доске;
- ограничение Work In Progress;
- управление потоком на основе метрик;
- явные правила перехода между статусами;
- непрерывное улучшение.

## Пример потока задач

```kroki-plantuml
@startuml
[*] --> Backlog
Backlog --> Ready
Ready --> InProgress : взять в работу
InProgress --> Review : готово к проверке
Review --> Done : принято
InProgress --> Blocked : препятствие
Blocked --> InProgress : устранено
Done --> [*]
@enduml
```

## Практические метрики

| Метрика | Что показывает |
| --- | --- |
| Cycle time | сколько задача в работе |
| Lead time | время от запроса до поставки |
| Throughput | сколько задач завершено за период |
| WIP age | риск старения задач в колонках |

## Когда Kanban подходит лучше всего

- продуктовая поддержка и сопровождение;
- команды с неравномерным входящим потоком;
- ситуации, где важна непрерывная поставка, а не спринтовый ритм.

## Типичные ошибки

- отсутствие WIP-лимитов при использовании Kanban-доски;
- работа "в столбик" без анализа узких мест;
- метрики собираются, но не используются для решений.

## Чек-лист качества

- У каждой колонки есть явные вход/выход критерии.
- WIP-лимиты соблюдаются и пересматриваются по данным.
- Обсуждение потока проходит регулярно (flow review).
