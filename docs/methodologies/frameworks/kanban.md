# Kanban

Kanban - потоковый фреймворк без спринтов, ориентированный на скорость и предсказуемость прохождения задач.

## Принципы

- визуализация потока;
- ограничение WIP;
- управление по метрикам;
- непрерывное улучшение.

## Этапы (типовой поток)

1. Commit (взяли в работу).
1. In Progress.
1. Review/Test.
1. Done.

## Артефакты

- Kanban board;
- explicit policies;
- WIP limits;
- flow metrics dashboard.

## Роли

- Service Request Manager (часто роль тимлида/аналитика);
- Delivery Team;
- Flow Manager (может совмещаться с Scrum Master/лидом).

## Плюсы и минусы

Плюсы:

- хорошо работает в потоке входящих задач;
- снижает multitasking;
- быстро выявляет узкие места.

Минусы:

- без дисциплины метрик превращается в «просто доску»;
- сложнее long-term прогноз без статистики.

## Пример подходящего проекта

Команда сопровождения интеграций и инцидентов, где приоритеты меняются ежедневно.

## Связь с требованиями и рисками

- требования дробятся на небольшие flow-элементы;
- high-risk задачи отмечаются class-of-service;
- риски контролируются через aging WIP и blockers.
