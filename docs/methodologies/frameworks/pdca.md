# PDCA

PDCA (Plan-Do-Check-Act) - цикл непрерывного улучшения процессов. Он полезен
как в разработке, так и в операционной деятельности продукта.

## Этапы PDCA

- Plan: сформулировать цель и гипотезу улучшения.
- Do: провести изменение в ограниченном масштабе.
- Check: измерить результат и сравнить с целевыми метриками.
- Act: стандартизировать успешное решение или скорректировать гипотезу.

## Схема PDCA

```kroki-plantuml
@startuml
start
:Plan;
:Do;
:Check;
if (Цель достигнута?) then (Да)
  :Act (закрепить стандарт);
else (Нет)
  :Act (скорректировать подход);
endif
stop
@enduml
```

## Пример: снижение lead time согласования требований

| Этап | Действие | Метрика |
| --- | --- | --- |
| Plan | Ввести шаблон требования и SLA на review | baseline lead time |
| Do | Пилот на одном продукте 2 спринта | время согласования |
| Check | Сравнить до/после | delta lead time |
| Act | Раскатить на остальные команды | доля соблюдения SLA |

## Типичные ошибки

- запуск изменений без baseline-метрик;
- слишком большой масштаб пилота;
- отсутствие стандартизации после успешного эксперимента.

## Чек-лист качества

- У гипотезы есть измеримая цель и срок.
- Check основан на данных, а не ощущениях.
- Итог цикла влияет на рабочие стандарты команды.

## Связь с требованиями и рисками

- Любой фреймворк должен иметь явный вход требований из единого backlog.
- Для критичных задач фиксируйте риск-триггеры и план реакции в рамках цикла планирования.
- Изменения процесса валидируйте не мнением, а метриками потока и качества.
