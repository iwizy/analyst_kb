# RUP

RUP (Rational Unified Process) - итеративный процесс разработки с четкими
фазами, ролями и артефактами. Подходит, когда нужна управляемость и
трассируемость решений.

## Фазы RUP

| Фаза | Цель | Ключевой результат |
| --- | --- | --- |
| Inception | сформировать видение и границы | бизнес-кейс, scope, high-level риски |
| Elaboration | снизить архитектурные риски | архитектурный baseline, уточненные требования |
| Construction | реализовать основную функциональность | рабочие инкременты системы |
| Transition | ввести решение в эксплуатацию | релиз, обучение пользователей, стабилизация |

## Пример фазного потока

```kroki-plantuml

```

## Когда RUP полезен

- крупные проекты с несколькими подрядчиками;
- высокие требования к документации и audit trail;
- сложная архитектура с критичными рисками на старте.

## Типичные ошибки

- попытка делать RUP как Waterfall без итераций;
- избыточная документация без практической ценности;
- слабая работа с рисками в фазе Elaboration.

## Чек-лист качества

- Для каждой фазы определены критерии готовности.
- Архитектурные риски закрываются до масштабной разработки.
- Артефакты поддерживаются в актуальном состоянии.
