# XP

XP (Extreme Programming) - инженерный Agile-фреймворк, ориентированный на
высокое качество кода и быструю обратную связь при частой поставке.

## Ключевые практики XP

- TDD (разработка через тесты);
- парное программирование;
- непрерывная интеграция;
- рефакторинг;
- маленькие релизы;
- коллективное владение кодом.

## Пример цикла TDD

```kroki-plantuml
@startuml
start
:Написать тест (Red);
:Запустить тесты -> тест падает;
:Написать минимальный код (Green);
:Запустить тесты -> тест проходит;
:Рефакторинг (Refactor);
:Повторить цикл;
stop
@enduml
```

## Когда XP уместен

- высокая цена дефекта в проде;
- частые изменения в бизнес-логике;
- команда готова к дисциплине инженерных практик.

## Роль аналитика в XP

- формулировать требования как testable examples;
- участвовать в уточнении acceptance criteria;
- помогать команде быстро получать обратную связь от бизнеса.

## Типичные ошибки

- выбор отдельных практик XP без системности;
- формальный TDD без meaningful тестов;
- отказ от рефакторинга при росте технического долга.

## Чек-лист качества

- Изменения покрываются автотестами на уровне поведения.
- CI дает быстрый сигнал о проблемах.
- Команда регулярно сокращает технический долг.
