# Lean

Lean - подход к организации разработки через устранение потерь и оптимизацию value stream.

## Принципы

- определить ценность для клиента;
- картировать поток создания ценности;
- обеспечить непрерывный поток;
- использовать pull-подход;
- улучшать систему постоянно.

## Этапы цикла Lean-улучшения

1. Найти потери в текущем потоке.
1. Сформировать гипотезу улучшения.
1. Внедрить изменение на ограниченном масштабе.
1. Измерить эффект.
1. Закрепить стандарт.

## Артефакты

- value stream map;
- список потерь (waste register);
- improvement backlog;
- метрики потока (lead time, flow efficiency).

## Роли

- Product/Process Owner;
- Delivery Team;
- Lean facilitator/coach;
- аналитик как владелец карты потока и гипотез.

## Плюсы и минусы

Плюсы:

- снижение lead time;
- фокус на результате, а не активности.

Минусы:

- требует системного измерения;
- локальная оптимизация может ухудшить end-to-end процесс.

## Пример подходящего проекта

Оптимизация процесса выпуска релизов в enterprise-команде с длинным согласованием.

## Связь с требованиями и рисками

- требования приоритизируются по value/flow cost;
- риск-идентификация встроена в value-stream review;
- изменения требований оцениваются по влиянию на поток.
