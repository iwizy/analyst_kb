# Анти-паттерны в интеграциях
Страница фиксирует ошибки интеграционного дизайна, которые чаще всего приводят к инцидентам, потере данных и нестабильным релизам.

## Частые анти-паттерны

| Анти-паттерн | Симптом | Последствие | Что делать |
| --- | --- | --- | --- |
| «Синхронно всё» | длинные цепочки request/response | каскадные отказы | разделять критичный sync и фоновый async контур |
| Retry без timeout/backoff | резкий рост повторов при сбое | self-DDoS downstream | задавать timeout + backoff + jitter |
| Нет идемпотентности | дубли операций при повторе | двойные списания/повторные заявки | вводить idempotency key + дедупликацию |
| Ломающие изменения контракта без политики | внезапные падения клиентов | массовые инциденты | semver + deprecation + compatibility checks |
| Shared DB как «скрытый API» | сервисы напрямую читают чужие таблицы | tight coupling и блокировка изменений | перейти к явным контрактам API/events |
| Нет [DLQ](../glossary.md#abbr-030) | «ядовитые» сообщения стопорят поток | потеря пропускной способности | выделять DLQ, retry topics, manual replay process |
| Секреты в коде | ключи/пароли в репозитории | security инциденты | vault + rotation + policy scanning |
| Нет корреляции логов | невозможно собрать end-to-end путь | затянутое расследование | correlation-id и распределенный tracing |

## Мини-кейс: инцидент из-за неконтролируемых retry

| Что было | Что изменили |
| --- | --- |
| 5 сервисов повторяли запросы без ограничений | внедрили timeout 800ms + retry 2 попытки + circuit breaker |
| downstream терял доступность под нагрузкой | добавили rate limit на gateway и отдельные очереди |
| MTTR > 2 часов | MTTR снизился до 25 минут благодаря трассировке |

## Ранние сигналы деградации интеграций

- растет p95 latency при неизменной бизнес-нагрузке;
- увеличивается размер DLQ и возраст сообщений;
- в релизах часто появляются hotfix под совместимость;
- инциденты повторяются в одних и тех же цепочках.

## Быстрый план стабилизации (2-3 недели)

1. Зафиксировать каталог контрактов и версии API/event schema.
2. Включить автоматические проверки совместимости в [CI/CD](../glossary.md#abbr-017).
3. Нормализовать политики retry/timeout/idempotency.
4. Добавить стандарт observability: logs + metrics + traces.
