# Единый стиль и терминология

Документ фиксирует базовые правила оформления, чтобы статьи оставались единообразными.

## 1. Терминология

- Используем русскую форму как основную: `стейкхолдер`, `требование`, `трассировка`, `ограничение`.
- Английский термин добавляем при первом упоминании, если он отраслевой стандарт:
  `stakeholder`, `traceability`, `risk register`.
- Сокращения оформляем через глоссарий:
  [BRD](glossary.md#abbr-010), [SRS](glossary.md#abbr-085), [RTM](glossary.md#abbr-107), [NFR](glossary.md#abbr-053), [KPI](glossary.md#abbr-047).

## 2. Базовая структура статьи

Рекомендуемая последовательность:

1. Определение и назначение.
2. Уровни сложности (`Базовый уровень`, `Средний уровень`, `Продвинутый уровень`).
3. Основной материал (метод/подход/паттерн).
4. Пример или схема.
5. Типичные ошибки.
6. Кросс-ссылки.
7. Источники или стандарты и источники.

## 3. Правила ссылок

- Для терминов и сокращений используем ссылки на [глоссарий](glossary.md).
- Не оборачиваем уже существующие markdown-ссылки дополнительной ссылкой на глоссарий.
- Для внутренних переходов используем относительные пути на страницы базы знаний.
