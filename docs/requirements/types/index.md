# Виды требований

Требования к системе обычно делятся на две ключевые группы: функциональные и нефункциональные. Вместе они формируют полную спецификацию продукта.

## Функциональные требования

Фиксируют, какие действия и бизнес-операции должна выполнять система.

- фокус: поведение и функции;
- артефакты: user flows, use cases, функциональные спецификации;
- проверка: функциональные тест-кейсы.

Подробнее: [Функциональные требования](functional.md)

## Нефункциональные требования

Фиксируют ограничения и характеристики качества работы системы.

- фокус: скорость, надежность, безопасность, масштабируемость, поддерживаемость;
- артефакты: quality attributes, SLO/SLA, архитектурные ограничения;
- проверка: нагрузочные, security, reliability и другие нефункциональные тесты.

Подробнее: [Нефункциональные требования](non-functional.md)

## Почему важны обе группы

Без функциональных требований команда не понимает, что реализовывать. Без нефункциональных требований невозможно гарантировать устойчивую работу решения в реальной эксплуатации.
