# Консоль браузера

Консоль браузера (DevTools Console) помогает анализировать клиентское поведение
приложения: ошибки JavaScript, сетевые запросы, состояние хранилища и работу
интерфейса в реальном времени.

## Когда полезна аналитику

- нужно воспроизвести пользовательскую проблему на фронтенде;
- требуется проверить фактический payload запросов и ответов;
- важно быстро подтвердить гипотезу без привлечения разработки на первом шаге.

## Основные области DevTools

- `Console`: runtime-ошибки, предупреждения, логирование;
- `Network`: HTTP-запросы, заголовки, тела, тайминги;
- `Application`: cookies, localStorage, sessionStorage;
- `Performance`: анализ медленных сценариев интерфейса.

## Практические проверки

1. Повторить пользовательский сценарий и зафиксировать ошибки в Console.
1. В Network проверить endpoint, метод, статус и содержимое ответа.
1. Сверить отправляемые параметры с требованиями/контрактом API.
1. Проверить наличие проблем с кэшированием, токенами и CORS.
1. Зафиксировать технические наблюдения в задаче с шагами воспроизведения.

## Полезные примеры в Console

```javascript
localStorage.getItem("auth_token");
```

```javascript
console.table([{step: "request", status: 200}, {step: "parse", status: "ok"}]);
```

```javascript
JSON.parse('{"id":1001,"status":"NEW"}');
```

## Лучшие практики

- фиксируйте точные шаги и время воспроизведения проблемы;
- сохраняйте HAR/скриншоты при анализе интеграционных ошибок;
- проверяйте проблему в чистой сессии (инкогнито/без расширений);
- отделяйте факты наблюдения от предположений о причине;
- скрывайте персональные данные в артефактах расследования.

## Типичные ошибки

- выводы по одной попытке воспроизведения;
- игнорирование вкладки Network при API-проблемах;
- анализ без учета кэша и состояния сессии;
- публикация чувствительных данных в комментариях задач.

## Чек-лист качества

- Собраны фактические данные запроса и ответа.
- Ошибка воспроизводится по описанным шагам.
- Наблюдения привязаны к конкретному сценарию и версии.
- Передача в разработку содержит технический контекст и доказательства.

## Практическое дополнение

- Определите owner инструмента в команде и минимальный стандарт использования.
- Зафиксируйте шаблон артефакта, который создается в инструменте, и связь с backlog.
- Введите ежемесячную ревизию актуальности артефактов, чтобы не накапливать устаревший контент.
