# Jira

Jira - система управления задачами и процессом поставки, которая помогает
команде вести backlog, контролировать статус работ и прослеживать изменения.

## Когда особенно полезна аналитику

- нужно структурировать требования в backlog и связывать их с релизами;
- важно фиксировать договоренности по изменениям и приоритетам;
- требуется прозрачный контроль статусов задач, дефектов и зависимостей.

## Базовые сущности

- `Issue`: единица работы (story, task, bug и др.);
- `Workflow`: жизненный цикл задачи;
- `Board`: визуализация потока работ;
- `Epic`: крупная функциональная область;
- `Sprint`: ограниченный по времени итерационный план;
- `JQL`: язык поиска и аналитики по задачам.

## Практический сценарий системного аналитика

1. Создать эпик под бизнес-инициативу и зафиксировать цель.
1. Разбить эпик на user story и технические задачи.
1. Для каждой задачи зафиксировать `acceptance criteria` и ссылки на артефакты.
1. Отслеживать блокировки и изменения через workflow и комментарии.
1. Перед релизом сформировать список задач по версии и статусу готовности.

## Полезные JQL-примеры

```jql
project = CORE and issuetype = Story and status in ("In Progress", "Review") order by priority desc
```

```jql
project = CORE and fixVersion = "2026.03" and status != Done
```

```jql
project = CORE and labels = integration and updated >= -7d
```

## Лучшие практики

- единый шаблон задач для требований, дефектов и технического долга;
- обязательные поля: контекст, критерии приемки, зависимости, риски;
- связь задач с документацией, диаграммами и API-контрактами;
- регулярная чистка backlog от дублей и устаревших элементов;
- прозрачные правила переходов по workflow.

## Типичные ошибки

- задачи без проверяемого результата и критериев приемки;
- использование комментариев вместо обновления описания задачи;
- перегруженный workflow с лишними статусами;
- потеря контекста из-за отсутствия связей между задачами.

## Чек-лист качества

- По задаче понятно, что и зачем нужно реализовать.
- Критерии приемки сформулированы однозначно.
- Есть ссылки на связанные требования и документы.
- Статус задачи отражает реальное состояние работ.
