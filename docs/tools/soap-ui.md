# SOAP UI

SOAP UI - инструмент для тестирования SOAP и REST сервисов. Для системного
аналитика он полезен при проверке контрактов, интеграционных сценариев и
граничных условий в системах с тяжелым enterprise-контуром.

## Когда использовать

- нужно проверять SOAP-сервисы по WSDL/XSD;
- требуется построить регрессионные интеграционные сценарии;
- важна проверка сложных XML-сообщений и бизнес-правил ответов.

## Базовые сущности проекта

- `Project`: контейнер интеграционного набора;
- `TestSuite`: группа логически связанных кейсов;
- `TestCase`: отдельный сценарий;
- `TestStep`: шаг сценария (request, assertion, script, datasource);
- `Assertions`: проверки статуса, схемы, значений.

## Пример SOAP-запроса

```xml
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:ord="http://example.com/order">
  <soapenv:Header/>
  <soapenv:Body>
    <ord:GetOrderRequest>
      <ord:OrderId>1001</ord:OrderId>
    </ord:GetOrderRequest>
  </soapenv:Body>
</soapenv:Envelope>
```

## Практический сценарий аналитика

1. Импортировать WSDL и проверить дерево операций.
1. Создать TestSuite по бизнес-процессу (заказ, платеж, возврат).
1. Для каждого TestCase добавить позитивные и негативные TestStep.
1. Настроить Assertions на коды, схему и критичные поля ответа.
1. Зафиксировать набор как базовый regression pack.

## Лучшие практики

- разделять сценарии по доменам и контурам интеграции;
- использовать property-переменные вместо жестко заданных значений;
- проверять не только код ответа, но и XSD/бизнес-ограничения;
- хранить тестовые проекты рядом со спецификациями контрактов;
- использовать data-driven шаги для массовой проверки кейсов.

## Типичные ошибки

- проверки только happy path;
- тесты без схемных и бизнес-assertions;
- копипаст XML без параметризации;
- потеря актуальности тестов после изменения WSDL.

## Чек-лист качества

- Критичные операции покрыты позитивными и негативными кейсами.
- Assertions проверяют структуру и смысл ответа.
- Параметры вынесены в свойства проекта/окружения.
- Набор тестов актуален для текущей версии контракта.
