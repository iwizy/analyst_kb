# Camunda Modeler

Camunda Modeler - инструмент для моделирования BPMN/DMN и подготовки
исполняемых процессов. Для системного аналитика это способ формализовать
процессы, точки принятия решений и интеграционные вызовы.

## Когда использовать

- нужно согласовать процесс между бизнесом и разработкой;
- требуется визуально описать оркестрацию шагов и обработку ошибок;
- важно задать правила решений через DMN-таблицы.

## Ключевые артефакты

- BPMN-диаграмма процесса;
- сервисные и пользовательские задачи;
- события ошибок, таймеры, компенсации;
- DMN-таблицы для бизнес-правил;
- формы/поля для user tasks (в зависимости от платформы исполнения).

## Практический сценарий аналитика

1. Смоделировать верхнеуровневый процесс с основными ветками.
1. Выделить точки интеграций и внешних зависимостей.
1. Добавить исключения: таймауты, ошибки валидации, ручная эскалация.
1. Вынести сложные правила в DMN-таблицу.
1. Согласовать модель с владельцами процесса и технической командой.

## Что фиксировать в каждой задаче процесса

- бизнес-цель шага;
- входные данные и источник;
- ожидаемый результат и выход;
- условия перехода к следующему шагу;
- поведение при ошибке/таймауте.

## Лучшие практики

- разделять бизнес-вид (для обсуждения) и техническую детализацию;
- использовать единые правила именования элементов;
- явно моделировать граничные и негативные сценарии;
- держать процесс на уровне, достаточном для принятия решений;
- фиксировать версию модели и владельца процесса.

## Типичные ошибки

- перегруженная диаграмма без декомпозиции;
- отсутствие обработки ошибок и эскалаций;
- смешение бизнес-ролей и технических компонентов в одном уровне;
- неактуальные модели, расходящиеся с реализацией.

## Чек-лист качества

- По модели понятен путь исполнения и альтернативные ветки.
- Для критичных шагов определены входы/выходы и критерии завершения.
- Ошибки и таймауты имеют явный путь обработки.
- Модель согласована с бизнесом и разработкой.
