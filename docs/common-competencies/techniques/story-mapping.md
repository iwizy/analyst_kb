# Story Mapping

Story Mapping помогает разложить пользовательский путь на шаги и задачи, формируя релизные срезы по ценности.

## Пошагово

1. Определить backbone пользовательского пути.
2. Добавить user activities и tasks.
3. Разбить на релизные срезы (MVP, R1, R2...).
4. Проверить зависимости и риски.
5. Синхронизировать с backlog.

## Шаблон

| Backbone step | Tasks | Release slice |
| --- | --- | --- |
| ... | ... | MVP / R1 / R2 |

## Пример

Checkout:

- выбрать товар -> корзина -> оплата -> подтверждение;
- MVP: карта + подтверждение;
- R1: купоны;
- R2: split-payment.

## Чек-лист

- путь пользователя непрерывен;
- MVP покрывает end-to-end ценность;
- backlog соответствует срезам карты.
