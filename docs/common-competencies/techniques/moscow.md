# MoSCoW

MoSCoW — техника приоритизации требований по четырем категориям: Must, Should, Could, Won't (for now).

## Как применять

1. Сформировать полный список требований.
2. Определить критерии ценности и ограничений.
3. Разнести требования по M/S/C/W.
4. Проверить зависимости и риски.
5. Подтвердить с бизнесом и командой.

## Шаблон

| [Требование](../../glossary.md#term-071) | Категория | Обоснование | Риск при переносе |
| --- | --- | --- | --- |
| ... | Must/Should/Could/Won't | ... | ... |

## Пример

- Must: проведение платежа;
- Should: уведомления по email;
- Could: расширенная фильтрация истории;
- Won't: экспорт PDF в этом релизе.

## Чек-лист

- Must действительно минимально необходимы;
- приоритизация согласована с ограничениями релиза;
- категория Won't задокументирована явно.
## Источники

- Agile prioritization practices.
