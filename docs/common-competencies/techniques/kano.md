# Kano Model

> Глоссарий терминов и сокращений: [Открыть](../../glossary.md)


Kano помогает оценить влияние функциональности на удовлетворенность пользователя: Must-be, Performance, Delighters.

## Категории Kano

- Must-be: обязательные ожидания;
- Performance: чем лучше реализовано, тем выше удовлетворенность;
- Delighters: неожиданные функции, повышающие wow-эффект.

## Пошагово

1. Собрать список функций.
2. Оценить каждую через опрос/интервью.
3. Отнести в категорию Kano.
4. Учесть стоимость реализации.
5. Включить результат в roadmap.
