# SWOT

SWOT помогает оценить инициативу через четыре группы факторов: сильные и слабые стороны (внутренние), возможности и угрозы (внешние).

## Уровни сложности

### Базовый уровень

- заполнять 4 квадранта по конкретной цели;
- отделять факты от предположений;
- формулировать выводы по каждому квадранту.

### Средний уровень

- использовать данные для подтверждения пунктов SWOT;
- переводить SWOT в roadmap/backlog;
- проводить SWOT-review при изменении контекста.

### Продвинутый уровень

- интегрировать SWOT с финансовой и риск-моделью;
- использовать SWOT для стратегической приоритизации портфеля;
- валидировать гипотезы через эксперименты.

## Шаблон SWOT

| Внутренние факторы | Внешние факторы |
| --- | --- |
| Strengths | Opportunities |
| Weaknesses | Threats |

## Пример (e-commerce продукт)

- S: сильная аналитика поведения пользователей;
- W: технический долг checkout-модуля;
- O: рост мобильного трафика;
- T: усиление конкуренции и рост стоимости привлечения.

## Типовые действия по результатам

- S+O: масштабировать сильные стороны на новый сегмент;
- W+T: отдельный план снижения уязвимостей;
- W+O: инвестировать в улучшение базового процесса.

## Связь с другими техниками

- SWOT -> MoSCoW/Kano для приоритизации;
- SWOT -> Risk Register для фиксации угроз;
- SWOT -> финансовая модель (NPV/ROI альтернатив).

## Чек-лист самопроверки

- SWOT привязан к конкретной цели;
- каждый пункт подтвержден фактами;
- по результатам определены конкретные действия.

## Контрольные вопросы

1. Какие пункты SWOT проверены данными, а какие гипотезы?
2. Какие угрозы требуют немедленных действий?
3. Что войдет в backlog по итогам SWOT?
