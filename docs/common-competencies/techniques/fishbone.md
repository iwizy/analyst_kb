# Fishbone (Ishikawa)

Диаграмма Исикавы структурирует возможные причины проблемы по категориям (люди, процесс, технологии, данные, среда).

## Пошагово

1. Сформулировать проблему в "голове" диаграммы.
2. Выбрать категории причин.
3. Добавить возможные причины по каждой ветке.
4. Проверить причины фактами.
5. Выбрать приоритетные меры.

## Пример (низкий NPS)

- Люди: недостаточное обучение поддержки;
- Процесс: долгий маршрут эскалации;
- Технологии: медленный отклик API;
- Данные: неполные статусы заказа.

## Чек-лист

- категории покрывают контекст;
- причины подтверждены данными;
- выбранные меры имеют owner и KPI.

## Контрольные вопросы

1. Какие причины в диаграмме подтверждены фактами, а какие гипотезы?
2. Какие причины дают наибольший вклад в проблему?
3. Какие corrective actions можно запустить уже в этом спринте?
