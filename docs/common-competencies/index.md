# Общие компетенции

Раздел объединяет универсальные практики системного аналитика: оценка трудоемкости, риск-менеджмент, финансовое обоснование, работа со стейкхолдерами и принятие решений на основе данных.

## Что покрыто

- оценка задач и прогнозирование сроков;
- анализ и управление рисками;
- финансовая оценка инициатив (ROI, NPV, TCO);
- планирование и управление временем;
- stakeholder management;
- data-driven подход к принятию решений.

## Оценка трудоемкости: методы

| Метод | Когда использовать | Сильная сторона | Ограничение |
| --- | --- | --- | --- |
| Planning Poker | Agile-команды и story-level оценки | Быстрое выравнивание понимания | Требует синхронного участия команды |
| Wideband Delphi | Сложные и неопределенные задачи | Снижает bias индивидуальных оценок | Затраты на фасилитацию |
| PERT | Проекты с явными рисками сроков | Учитывает optimistic/most likely/pessimistic | Нужна дисциплина по входным оценкам |
| T-shirt sizes | Ранний портфельный уровень | Простота и скорость | Низкая точность для планирования релиза |

## Риск-менеджмент: методы

| Метод | Для чего подходит |
| --- | --- |
| Матрица рисков (Probability x Impact) | Приоритизация риска по влиянию и вероятности |
| FMEA | Анализ отказов и preventive меры |
| SWOT | Стратегическая оценка контекста инициативы |
| Risk burndown | Контроль снижения рисков по итерациям |

## Финансовое обоснование

- ROI: окупаемость инициативы;
- NPV: ценность денежных потоков с учетом дисконтирования;
- TCO: полная стоимость владения;
- payback period: срок возврата инвестиций.

Практика: финансовые метрики проверяются вместе с техническими рисками и зависимостями.

## Data-driven решения

Минимальный стандарт:

- решение опирается на наблюдаемые метрики;
- гипотезы явно отделены от фактов;
- есть период пересмотра решений;
- последствия решений фиксируются в decision log.

## Примеры по доменам

### Финансовый сектор

- акцент: риск-матрица + NPV + регуляторный анализ;
- результат: приоритизация инициатив по снижению операционных потерь.

### E-commerce

- акцент: A/B-метрики, ROI маркетинговых/продуктовых изменений;
- результат: приоритизация бэклога по влиянию на conversion/retention.

### Госуслуги

- акцент: SLA, compliance риски, TCO долгосрочной поддержки;
- результат: обоснованные решения по roadmap и эксплуатационным инвестициям.
