# Общие компетенции

Раздел объединяет практики, которые нужны системному аналитику независимо от домена и технологического стека: оценка, риски, финансы, time management, stakeholder management и data-driven принятие решений.

## Уровни сложности

### Базовый уровень

- понимать базовые методы оценки, риск-анализа и приоритизации;
- использовать шаблоны реестров и чек-листы;
- строить рабочие артефакты для ежедневной работы.

### Средний уровень

- выбирать методику под контекст проекта;
- интегрировать оценки, риски и требования в единый процесс;
- использовать данные для корректировки плана.

### Продвинутый уровень

- проектировать систему управленческих метрик на уровне программы;
- сочетать финансовые и нефинансовые критерии приоритизации;
- внедрять стандарты и улучшать зрелость команды.

## Как использовать раздел на практике

1. Для новой инициативы начните с `Оценка задач и рисков`.
2. Для спорных приоритетов используйте `Финансовую оценку` + `Data-driven`.
3. Для коммуникационных конфликтов используйте `Stakeholder management`.
4. Для процессного улучшения применяйте техники из раздела `Техники`.
