# Практическая рабочая тетрадь аналитика
Страница объединяет ключевые общие компетенции в единый цикл: оценка -> риски -> финансы -> стейкхолдеры -> data-driven решение.

## Уровни сложности

### Базовый уровень

- применять базовые шаблоны оценки и риск-реестра;
- фиксировать допущения и ограничения;
- документировать решения в понятном формате.

### Средний уровень

- связывать оценки, риски и финансовое обоснование;
- управлять коммуникациями со стейкхолдерами;
- корректировать план по фактическим данным.

### Продвинутый уровень

- строить повторяемую управленческую систему для нескольких инициатив;
- внедрять единые метрики эффективности решений;
- повышать зрелость команды через стандартизированные практики.

## Сквозной цикл применения

1. Сформулировать цель инициативы и критерии успеха.
2. Оценить объем и сложность задач.
3. Выявить риски и план реагирования.
4. Рассчитать финансовую целесообразность.
5. Согласовать с ключевыми стейкхолдерами.
6. Запустить и отслеживать фактические метрики.
7. Выполнить переоценку и обновить план.

## Единый шаблон инициативы

| Поле | Что заполнить |
| --- | --- |
| Цель | какой бизнес-результат ожидается |
| Scope | что входит/не входит в инициативу |
| Оценка | метод и диапазон трудоемкости |
| Ключевые риски | top-5 рисков и owners |
| Финансы | [ROI](../glossary.md#abbr-073)/[NPV](../glossary.md#abbr-055)/TCO |
| Стейкхолдеры | карта влияния и план коммуникаций |
| Метрики | leading/lagging показатели |
| Решение | go/no-go/adjust |

## Мини-кейс: e-commerce ускорение checkout

### Шаг 1. Оценка

- метод: Planning Poker + historical velocity;
- итог: 2 спринта при команде 6 человек.

### Шаг 2. Риски

| Риск | Вероятность | Влияние | Действие |
| --- | --- | --- | --- |
| задержка интеграции платежей | medium | high | ранний технический spike |
| неоднозначные критерии приемки | high | medium | joint review BA+QA |

### Шаг 3. Финансовая оценка

- прогноз: +2.5% conversion;
- ориентировочный payback: 3-4 месяца.

### Шаг 4. Data-driven контроль

| Метрика | До | После |
| --- | --- | --- |
| conversion checkout | 41% | 44% |
| среднее время оформления | 4m 20s | 3m 05s |

## Еженедельный ритм управления

| Событие | Цель |
| --- | --- |
| weekly risk review | пересмотр статусов и мер реагирования |
| stakeholder sync | выравнивание ожиданий |
| metrics review | проверка фактической динамики |
| decision log update | фиксация ключевых решений |

## Типичные ошибки

- оценка без фиксации предпосылок;
- риски без владельцев и конкретных действий;
- финансовые расчеты без сценарного анализа;
- метрики, которые не влияют на решения.
