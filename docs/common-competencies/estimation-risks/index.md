# Оценка задач и рисков

Оценка и риски должны работать как единый контур: оценка без рисков создает ложную точность, а риск без оценки не влияет на план и бюджет.

## Уровни сложности

### Базовый уровень

- понимать основные методы оценки и риск-анализа;
- уметь фиксировать допущения;
- вести базовый risk register.

### Средний уровень

- выбирать метод оценки под тип задач;
- рассчитывать резервы и влияние рисков на сроки;
- увязывать оценки с capacity и roadmap.

### Продвинутый уровень

- использовать статистику прошлых проектов и Monte Carlo;
- учитывать финансовый риск (contingency/management reserve);
- управлять прогнозом на уровне программы.
