# Оценка трудоемкости

Оценка трудоемкости нужна для realistic planning и управления ожиданиями. Используйте несколько методов и фиксируйте допущения.

## Методы оценки

## Planning Poker

Команда оценивает задачи картами (обычно Fibonacci), затем обсуждает расхождения.

Когда полезен:

- refinement в Agile-команде;
- нужна коллективная оценка сложности.

## Wideband Delphi

Итеративная экспертная оценка с несколькими раундами и консенсусом.

Когда полезен:

- неопределенные и сложные инициативы;
- несколько экспертов из разных функций.

## PERT (трехточечная оценка)

`E = (O + 4M + P) / 6`

- `O` - optimistic,
- `M` - most likely,
- `P` - pessimistic.

Когда полезен:

- нужно учитывать неопределенность диапазона.

## T-shirt sizes

Категории `XS/S/M/L/XL` для быстрого ранжирования масштаба.

Когда полезен:

- ранний этап портфельного планирования.

## Сравнение методов

| Метод | Скорость | Точность | Лучший контекст |
| --- | --- | --- | --- |
| Planning Poker | средняя | средняя/высокая | sprint planning |
| Wideband Delphi | низкая/средняя | высокая | сложные программы |
| PERT | средняя | высокая при данных | риск-ориентированное планирование |
| T-shirt | высокая | низкая/средняя | high-level roadmap |

## Шаблон таблицы оценки

| Элемент | Метод | Оценка | Допущения | Риски |

| Интеграция с API X | PERT | 8 дн | контракт стабилен | риск изменения API |

## Типичные ошибки

- смешивать effort и календарный срок;
- оценивать без декомпозиции;
- не пересматривать оценку при изменении scope.
