# Оценка трудоемкости

Оценка трудоемкости определяет, сколько усилий потребуется для реализации
задачи с учетом текущих ограничений команды и среды.

## Что влияет на оценку

- полнота и качество требований;
- сложность интеграций и зависимостей;
- зрелость команды и знакомство с доменом;
- технический долг и ограничения архитектуры;
- доступность окружений и тестовых данных.

## Популярные подходы

- экспертная оценка (expert judgment);
- декомпозиция на мелкие задачи (bottom-up);
- аналогия с историческими задачами;
- трехточечная оценка `O/M/P` (PERT).

## Пример PERT

Формула ожидаемой оценки: `(O + 4M + P) / 6`.

| Параметр | Значение |
| --- | --- |
| `O` (optimistic) | 3 дня |
| `M` (most likely) | 5 дней |
| `P` (pessimistic) | 11 дней |
| Ожидаемо | 5.67 дня |

## Лучшие практики

- оценивать после минимальной декомпозиции;
- отдельно фиксировать допущения оценки;
- регулярно калибровать оценки по факту завершенных задач;
- считать буфер на неопределенность для рискованных задач.

## Типичные ошибки

- оценка "в целом" без разбиения на части;
- смешение effort и календарного срока;
- игнорирование внешних зависимостей;
- фиксация оценки как неизменного обязательства.

## Чек-лист качества

- Оценка привязана к конкретному объему и критериям готовности.
- Допущения и ограничения зафиксированы явно.
- Учтены интеграционные и организационные зависимости.
- Есть план пересмотра оценки по мере уточнения требований.
