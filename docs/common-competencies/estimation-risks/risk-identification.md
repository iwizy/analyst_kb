# Выявление рисков

Выявление рисков - это системный поиск событий, которые могут негативно
повлиять на сроки, стоимость, качество или ценность результата.

## Источники рисков

- продуктовые: изменчивость требований, невалидные гипотезы;
- технические: интеграции, производительность, безопасность;
- организационные: ресурсы, коммуникации, приоритеты;
- внешние: регуляторика, поставщики, инфраструктурные ограничения.

## Практический процесс

1. Сформируйте рабочую группу (бизнес, аналитика, разработка, QA).
1. Проведите риск-сессию по этапам и компонентам.
1. Зафиксируйте риск в формате: причина -> событие -> последствие.
1. Оцените вероятность и влияние.
1. Добавьте риск в реестр с владельцем.

## Мини-шаблон записи риска

| Поле | Пример |
| --- | --- |
| Риск | Задержка интеграции с внешним API |
| Причина | Нестабильность контракта поставщика |
| Вероятность | Средняя |
| Влияние | Высокое |
| Мера | Мок-сервис + ранний контрактный тест |
| Владелец | Архитектор интеграций |

## Лучшие практики

- искать риски на ранних этапах, а не перед релизом;
- разделять риск и уже произошедшую проблему;
- работать с вероятностью и влиянием, а не только с "чувством";
- пересматривать реестр при каждом существенном изменении контекста.

## Типичные ошибки

- формальный реестр без владельцев и действий;
- фокус только на технических рисках;
- отсутствие критериев, когда риск считается закрытым.

## Чек-лист качества

- У каждого риска есть описание причины и последствий.
- Назначен владелец и дата пересмотра.
- Для высоких рисков определены превентивные меры.
- Реестр рисков встроен в рабочий процесс команды.
