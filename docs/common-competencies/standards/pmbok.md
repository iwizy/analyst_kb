# PMBOK

PMBOK - свод знаний по управлению проектами, который описывает процессы,
области знаний и практики для достижения целей проекта в ограничениях.

## Когда полезен аналитику

- нужно встроить анализ в проектный контур;
- важно синхронизировать требования со сроками и бюджетом;
- требуется работать с заинтересованными сторонами системно.

## Ключевые области

- Scope, Schedule, Cost, Quality;
- Risk, Communication, Stakeholder;
- Resource, Procurement, Integration.

## Практическое применение

- формализация границ и критериев готовности;
- контроль изменений требований через impact-анализ;
- управление ожиданиями стейкхолдеров.

## Типичные ошибки

- проектный процесс без содержательной работы с требованиями;
- позднее управление изменениями;
- отсутствие связи рисков с планом поставки.

## Чек-лист качества

- Scope проекта согласован и управляется.
- Изменения требований имеют прозрачный процесс.
- Риски и коммуникации встроены в регулярный цикл управления.
