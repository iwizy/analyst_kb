# Стандарты и своды знаний

Стандарты и своды знаний дают системному аналитику общую рамку: какие артефакты
нужны, какие процессы обязательны и по каким критериям оценивается качество
результата.

## Зачем это аналитикам

- единые правила коммуникации с разными командами и аудиторами;
- повышение предсказуемости документации и процесса;
- снижение рисков из-за пропуска критичных шагов;
- аргументированная защита решений перед стейкхолдерами.

## Как выбирать стандарт под задачу

| Контекст | Что обычно используют |
| --- | --- |
| Бизнес-анализ требований | BABOK, CPRE, CBAP |
| Процессы разработки/инжиниринга | ISO 12207, ISO 15288, SEBOK |
| Управление ИТ и сервисами | ITIL, COBIT |
| Проектное управление | PMBOK |
| Управление данными | DAMA DMBOK |
| Формализация документов в РФ | ГОСТ 19, ГОСТ 34 |

## Практический подход

1. Определите цель применения стандарта (аудит, зрелость, унификация, обучение).
1. Выберите минимально достаточный набор практик.
1. Адаптируйте терминологию и шаблоны под контекст команды.
1. Закрепите в процессе: кто, когда и как применяет требования стандарта.
