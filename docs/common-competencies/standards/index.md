# Стандарты и своды знаний

> Глоссарий терминов и сокращений: [Открыть](../../glossary.md)


Стандарты дают аналитикам общий язык, структуру артефактов и проверяемые критерии качества процессов.

## Уровни сложности

### Базовый уровень

- знать назначение ключевых стандартов;
- выбирать стандарт под задачу;
- применять минимальный набор практик без перегруза.

### Средний уровень

- адаптировать требования стандартов под контекст команды;
- строить шаблоны и процессные правила;
- проводить внутренние проверки соответствия.

### Продвинутый уровень

- проектировать framework зрелости на уровне организации;
- сочетать несколько стандартов без дублирования;
- внедрять сертификационные и audit-практики.

## Карта стандартов

| Область | Стандарты / своды знаний | Для чего использовать |
| --- | --- | --- |
| Бизнес-анализ | BABOK, CBAP, CPRE | требования, elicitation, анализ ценности |
| Проектное управление | PMBOK | scope/schedule/cost/risk/stakeholder |
| Системная инженерия | SEBOK, ISO 15288, ISO 12207 | системный жизненный цикл и архитектурные решения |
| Риск-менеджмент | ISO 31000, NIST RMF | framework управления рисками и контролями |
| IT governance / service | COBIT, ITIL | процессы ИТ-управления и эксплуатации |
| Data governance | DAMA DMBOK | качество и управление данными |
| Документирование | IEEE, ГОСТ 19/34 | формализация артефактов |

## Как выбирать стандарт под задачу

1. Определить цель: унификация, аудит, обучение, сертификация.
2. Оценить обязательные внешние требования (регуляторика/контракты).
3. Выбрать минимально достаточный набор практик.
4. Настроить шаблоны и роли.
5. Проверять эффект через метрики процесса и качества.

## Сравнение ключевых стандартов

| Стандарт | Сильная сторона | Ограничение | Когда особенно полезен |
| --- | --- | --- | --- |
| BABOK v3 | глубина BA-практик | не prescriptive-процесс | построение BA-функции |
| PMBOK 7 | универсальный проектный governance | требует адаптации под delivery-подход | крупные программы |
| SEBOK | системное мышление | высокий порог входа | сложные междисциплинарные системы |
| ISO 31000:2018 | универсальный risk framework | не дает детальных процедур по доменам | корпоративный риск-контур |
| NIST RMF | сильная security/risk связка | преимущественно для ИБ-контекста | regulated/security-critical среды |

## Сертификации (рекомендации)

| Роль | Полезные сертификации |
| --- | --- |
| BA/SA middle | CPRE Foundation, ECBA |
| Senior/Lead BA | CBAP, PMI-PBA |
| PM/Program | PMP, PMI-ACP |
| Risk/Compliance | ISO 31000 training, CRISC |

## Примеры применения

- BABOK: построение процесса elicitation и life cycle management требований.
- ISO 31000: внедрение корпоративного risk management framework.
- NIST RMF: оценка и авторизация безопасности системы в гос/критичных проектах.
