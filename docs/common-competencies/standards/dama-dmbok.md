# DAMA DMBOK

DAMA DMBOK - свод знаний по управлению данными (Data Management Body of
Knowledge). Он систематизирует практики качества, архитектуры и governance
данных.

## Когда нужен

- организация строит data-driven процессы;
- требуется унифицировать управление данными между системами;
- критичны качество, происхождение и ответственность за данные.

## Основные домены

- Data Governance;
- Data Architecture;
- Data Quality;
- Master/Reference Data;
- Metadata Management;
- Data Integration and Interoperability;
- Data Security and Privacy.

## Практическая польза

- единые правила владения и качества данных;
- снижение расхождений между отчетами и источниками;
- прозрачность lineage и ответственности за изменения.

## Типичные ошибки

- сведение DMBOK только к BI-отчетности;
- отсутствие data owner/data steward ролей;
- governance "на бумаге" без операционной практики.

## Чек-лист качества

- Для критичных данных определены владельцы.
- Есть правила качества и механизм их контроля.
- Известны источники и поток данных до потребителя.
