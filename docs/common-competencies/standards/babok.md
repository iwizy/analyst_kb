# BABOK v3

BABOK (Business Analysis Body of Knowledge) описывает практики бизнес-анализа: от планирования аналитической работы до оценки решения.

## Уровни сложности

### Базовый уровень

- понимать 6 knowledge areas;
- применять терминологию и базовые техники;
- использовать BABOK как чек-лист полноты анализа.

### Средний уровень

- адаптировать задачи BABOK под SDLC команды;
- выстраивать жизненный цикл требований;
- определять входы/выходы аналитических артефактов.

### Продвинутый уровень

- стандартизировать BA-практики на уровне организации;
- строить систему метрик качества анализа;
- готовить команду к CBAP/PMI-PBA сертификации.

## Структура BABOK

| Knowledge Area | Что покрывает |
| --- | --- |
| BA Planning and Monitoring | план BA-работ, подходы, governance |
| Elicitation and Collaboration | сбор и согласование информации |
| Requirements Life Cycle Management | трассировка, изменения, актуальность |
| Strategy Analysis | оценка потребности и целевого состояния |
| Requirements Analysis and Design Definition | формализация и валидация требований |
| Solution Evaluation | проверка ценности внедренного решения |

## Как применять на проекте

1. Выбрать релевантные задачи BABOK для инициативы.
2. Привязать их к фазам SDLC и ролям.
3. Для каждой задачи определить артефакт и критерий качества.
4. Регулярно проводить review по чек-листу BABOK.

## Пример применения (финтех)

- Elicitation: интервью + анализ регуляторных документов.
- Requirements life cycle: трассировка BR -> SR -> test.
- Solution evaluation: метрики [SLA](../../glossary.md#abbr-079) и доля ошибок транзакций.

## Связь с требованиями и процессом

- BABOK усиливает раздел [Требования](../../requirements/index.md);
- поддерживает управление изменениями и согласование;
- помогает формализовать входы в [процесс разработки](../../development-process/index.md).

## Типичные ошибки

- попытка внедрить BABOK полностью без адаптации;
- фокус на документах вместо результата;
