# NIST Risk Management Framework (RMF)

NIST RMF — framework для управления рисками информационной безопасности и соответствия, широко применяемый в регулируемых и критичных системах.

## Уровни сложности

### Базовый уровень

- понимать шаги RMF;
- связывать controls с системными рисками;
- документировать security risk decisions.

### Средний уровень

- применять категоризацию и выбор контролей;
- вести continuous monitoring;
- интегрировать RMF в SDLC и change management.

### Продвинутый уровень

- управлять авторизацией систем и audit-контуром;
- автоматизировать evidence collection;
- синхронизировать RMF с enterprise governance.

## Шаги RMF (NIST)

1. Prepare.
2. Categorize.
3. Select controls.
4. Implement controls.
5. Assess controls.
6. Authorize.
7. Monitor continuously.

## Пример применения (гос/финтех)

- категоризация системы по уровню воздействия;
- выбор и внедрение security controls;
- независимая оценка контролей;
- решение об авторизации + continuous monitoring.

## Связь с требованиями и процессом

- security требования фиксируются в спецификациях;
- controls включаются в DoD/quality gates;
- инциденты и findings обновляют risk register.

## Типичные ошибки

- формальное «закрытие» контролей без фактической проверки;
- отсутствие непрерывного мониторинга;
- разрыв между security и delivery-командой.
