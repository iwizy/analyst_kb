# Тайм-менеджмент

Тайм-менеджмент для аналитика - это умение управлять потоком задач и фокусом,
чтобы сохранять качество решений в условиях параллельных запросов.

## Где чаще всего теряется время

- частые переключения контекста;
- встречи без результата;
- неструктурированный backlog аналитических задач;
- поздняя приоритизация срочных запросов.

## Практический подход

1. Планировать неделю по целям, а не по списку активности.
1. Выделять фокус-блоки на сложную аналитическую работу.
1. Группировать однотипные задачи (ответы, ревью, документация).
1. Ежедневно обновлять приоритеты на основе новых данных.

## Рабочие практики

- `time blocking` для глубоких задач;
- правило 1-3 приоритетов на день;
- буфер времени на срочные запросы;
- ограничение количества задач "в работе" одновременно.

## Типичные ошибки

- попытка отвечать на все входящие сразу;
- переоценка доступного фокуса в день;
- отсутствие времени на рефлексию и корректировку плана.

## Чек-лист качества

- Критичные задачи закрываются в срок.
- Есть фокус-время без отвлечений.
- План регулярно пересматривается по факту.
