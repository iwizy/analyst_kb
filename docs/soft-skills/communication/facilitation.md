# Фасилитация встреч

> Глоссарий терминов и сокращений: [Открыть](../../glossary.md)


Фасилитация нужна, чтобы встреча завершалась решением, а не обсуждением.

## Уровни сложности

### Базовый уровень

- формулировать цель и результат встречи;
- готовить agenda и таймбоксы;
- фиксировать решения и follow-up.

### Средний уровень

- применять техники генерации и отбора идей;
- управлять участием группы;
- проводить alignment-сессии и review требований.

### Продвинутый уровень

- фасилитировать конфликтные multi-stakeholder встречи;
- управлять групповыми искажениями и доминированием;
- строить decision framework команды.

## Мини-шаблон встречи

| Поле | Пример |
| --- | --- |
| Цель | согласовать scope релиза R12 |
| Формат | requirements review |
| Участники | PO, BA, Tech Lead, QA |
| Agenda | 4 блока по 15 мин |
| Выход | решения + owner + срок |

## Техники фасилитации

| Техника | Когда использовать |
| --- | --- |
| Brainstorming | быстро собрать гипотезы |
| Silent writing | дать голос тихим участникам |
| Dot voting | выбрать приоритет из списка |
| Fishbone | разобрать причины проблемы |
| 6-3-5 | структурированно сгенерировать идеи |

## Принятие решений

- Consensus mapping: оценить уровень согласия по вариантам.
- Decision matrix: сравнить варианты по критериям.
- DACI/RACI: определить, кто принимает решение и кто консультирует.

## Форматы встреч

### Ретроспектива

1. Данные (что произошло).
2. Инсайты (почему).
3. Решения (что меняем).

### Review требований

1. Scope и цель.
2. Риски и зависимости.
3. AC/NFR и open questions.

### Alignment meeting

1. Точки рассогласования.
2. Варианты и trade-off.
3. Финальное решение.

## Типичные ошибки

- отсутствие явной цели встречи;
- доминирование 1-2 участников;
- отсутствие протокола и follow-up;
- обсуждение реализации до согласования цели.
