# Ведение переговоров

> Глоссарий терминов и сокращений: [Открыть](../../glossary.md)


Переговоры аналитика чаще всего про баланс: бизнес-ценность, сроки, технические ограничения и риски качества.

## Уровни сложности

### Базовый уровень

- различать позиции и интересы;
- готовить BATNA/WATNA;
- фиксировать минимально приемлемый результат.

### Средний уровень

- применять Harvard Negotiation и win-win framing;
- использовать принципиальные переговоры и DESC в сложных диалогах;
- оформлять итог соглашением с владельцами и сроками.

### Продвинутый уровень

- вести многосторонние переговоры по приоритетам;
- управлять эскалацией и deadlock;
- выстраивать стратегию уступок без потери цели.

## Подходы и техники

| Подход | Когда применять | Ключевая идея |
| --- | --- | --- |
| Harvard Negotiation | конфликт интересов | отделять людей от проблемы |
| BATNA/WATNA | риск срыва договоренности | готовить лучший и худший альтернативный сценарий |
| Win-win framing | важна долгосрочная кооперация | искать взаимную выгоду, а не победителя |
| Принципиальные переговоры | давление и эмоции | опираться на критерии, а не на статус |
| DESC | эскалация и напряжение | структурировать диалог без обвинений |

## Шаблон подготовки к переговорам

| Поле | Что заполнить |
| --- | --- |
| Цель | какой результат нужен |
| Минимально приемлемый результат | граница, ниже которой нельзя соглашаться |
| BATNA | лучший вариант без сделки |
| WATNA | худший вариант без сделки |
| Аргументы | данные, риски, ограничения |
| Варианты trade-off | минимум 2-3 сценария |
| Agenda | повестка и таймбокс |

## Пример: переговоры по scope и сроку

Ситуация: бизнес просит добавить 5 требований в текущий спринт.

Варианты:

1. Полный scope, сдвиг релиза на 2 недели.
2. МVP из 2 требований в текущий релиз + 3 в следующий.
3. Сохранить срок, убрать 2 менее ценных требования из текущего scope.

Решение: вариант 2, фиксируем KPI и дату пересмотра.

## Шаблон итогового соглашения

| Пункт | Содержание |
| --- | --- |
| Принятое решение | что делаем |
| Scope in/out | что входит и что исключено |
| Срок | дедлайн и контрольная дата |
| Владелец | ответственный за выполнение |
| Риски | что может сорвать договоренность |

## Практика развития

- симуляции переговоров 1 раз в 2 недели;
- дневник уступок: что уступили, что получили, почему;
- post-mortem по сложным переговорам.

## Чек-лист ошибок

- входить в переговоры без BATNA;
- обсуждать позиции вместо интересов;
- давать обещания без оценки зависимости;
- не фиксировать договоренность письменно.
