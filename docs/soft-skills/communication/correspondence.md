# Ведение переписки

Деловая переписка аналитика это юридически и организационно значимый канал фиксации решений, ограничений и обязательств.

## Уровни сложности

### Базовый уровень

- писать кратко и однозначно;
- отделять контекст от запроса;
- указывать владельца и срок.

### Средний уровень

- выбирать канал: email, мессенджер, wiki;
- структурировать сложные письма по шаблону;
- использовать нейтральный деловой стиль.

### Продвинутый уровень

- управлять перепиской в конфликтных ситуациях;
- вести цепочку решений без потери контекста;
- применять шаблоны для массовых согласований.

## Стандарт структуры письма

1. Приветствие.
2. Цель письма (1-2 предложения).
3. Аргументы/контекст.
4. Призыв к действию (что нужно, от кого, к какому сроку).
5. Благодарность и next step.

## Каналы коммуникации

| Канал | Когда использовать | Риск |
| --- | --- | --- |
| Email | официальные решения, согласования, эскалации | длинные треды без решения |
| Мессенджер | быстрые уточнения и синхронизация | потеря контекста/решений |
| Wiki (Confluence) | стабильные артефакты и договоренности | устаревание без ревизии |

## Пример "плохо" и "хорошо"

Плохо:

"Коллеги, по [API](../../glossary.md#abbr-005) вроде есть проблемы, нужно посмотреть побыстрее".

Хорошо:

"Коллеги, по методу `POST /orders` не зафиксированы коды ошибок 409/422. Нужна валидация контракта до 15:00 14.02, owner: Иван. Влияние: без этого [QA](../../glossary.md#abbr-067) не завершит тест-кейсы для релиза R12".

## Чек-лист перед отправкой

- тема отражает суть запроса;
- в письме есть один явный call to action;
- назначен owner и срок;
- указаны ссылки на артефакты;
- тон письма корректный и нейтральный.
