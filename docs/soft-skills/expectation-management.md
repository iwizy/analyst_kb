# Управление ожиданиями заказчика

Управление ожиданиями это постоянная синхронизация между желаемым результатом и реальными ограничениями поставки.

## Уровни сложности

### Базовый уровень

- выявлять и фиксировать ожидания;
- согласовывать критерии успеха;
- прозрачно сообщать об ограничениях.

### Средний уровень

- выстраивать регулярный ритм коммуникации;
- управлять изменениями ожиданий;
- связывать ожидания с roadmap и capacity.

### Продвинутый уровень

- предотвращать конфликт ожиданий между группами стейкхолдеров;
- управлять ожиданиями в условиях высокой неопределенности;
- использовать данные и метрики для переоценки договоренностей.

## Практический алгоритм

1. Выявить ожидания: цели, сроки, качество, границы.
2. Формализовать: scope, AC, NFR, допущения.
3. Согласовать критерии успеха и DoD.
4. Настроить ритм коммуникаций.
5. Управлять изменениями через change process.

## Ритм взаимодействия

| Ритм | Цель | Участники |
| --- | --- | --- |
| Weekly check-in | статус, риски, корректировки | BA, PO, key stakeholders |
| Demo/Review | проверка результата | команда + бизнес |
| Steering committee | решения по отклонениям и приоритетам | менеджмент/владельцы |

## Шаблон плана управления ожиданиями

| Поле | Содержание |
| --- | --- |
| Ожидание | что хочет стейкхолдер |
| Ограничение | что мешает выполнить в полном объеме |
| Способ удовлетворения | какой вариант предлагаем |
| Метрика успеха | как поймем, что ожидание закрыто |
| Owner | кто отвечает |
| Review date | дата пересмотра |

## Таблица "ожидание -> способ -> метрика"

| Ожидание | Способ удовлетворения | Метрика |
| --- | --- | --- |
| релиз в этом квартале | MVP scope + phased rollout | дата релиза и доля покрытого scope |
| "без дефектов" | quality gates + UAT | defect leakage rate |
