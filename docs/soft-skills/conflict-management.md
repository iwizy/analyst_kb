# Управление конфликтами

Конфликт в IT-проектах неизбежен. [Задача](../glossary.md#term-028) аналитика не избежать конфликта, а перевести его в конструктивное решение.

## Уровни сложности

### Базовый уровень

- распознавать тип конфликта;
- отделять интересы от позиций;
- вести диалог без персональных обвинений.

### Средний уровень

- применять DESC, Thomas-Kilmann и interest-based подход;
- фасилитировать обсуждение спорных требований;
- фиксировать договоренности и контрольные точки.

### Продвинутый уровень

- разрешать многослойные межкомандные конфликты;
- работать с эскалациями и power dynamics;
- предотвращать повторные конфликты через изменения процесса.

## Природа и виды конфликтов

| Тип | Пример в проекте | Риск |
| --- | --- | --- |
| Интересов | бизнес хочет scope, команда просит срок | срыв релиза |
| Информационный | разные трактовки требований | rework и дефекты |
| Ролевой | неясно, кто принимает решение | затяжные согласования |
| Ценностный | скорость vs качество | системные компромиссы |
| Структурный | перегрузка команды и зависимости | хроническая эскалация |

## Методы разрешения

- DESC-model.
- Thomas-Kilmann styles (Competing, Collaborating, Compromising, Avoiding, Accommodating).
- Interest-based bargaining.
- Активное слушание и перефразирование.

## Пошаговый план

1. Подготовка: стороны, интересы, BATNA, ограничения.
2. Обсуждение: проблема формулируется без обвинений.
3. Поиск вариантов: brainstorming + оценка trade-off.
4. Решение: выбираем вариант по согласованным критериям.
5. Закрепление: owner, сроки, критерии проверки.

## Пример из IT-проекта

Ситуация: спор о включении нефункционального требования в релиз.

- Бизнес: "важен срок запуска".
- Архитектура/[QA](../glossary.md#abbr-067): "без [NFR](../glossary.md#abbr-053) высокий риск инцидента".

Разрешение:

1. Согласован минимальный NFR-пакет для релиза.
2. Остальные улучшения вынесены в следующий спринт.
3. Назначен post-release review по инцидентам и latency.

## Ролевые упражнения

- симуляция конфликта BA vs Tech Lead по scope;
- разбор кейса "приоритеты vs архитектурный долг";
- тренировка формулировок DESC в паре.
