# Аналитическое мышление

Аналитическое мышление помогает аналитикам структурировать сложные проблемы, строить гипотезы и принимать обоснованные решения.

## Уровни сложности

### Базовый уровень

- декомпозировать проблему;
- отделять факты от предположений;
- формулировать проверяемые вопросы.

### Средний уровень

- строить причинно-следственные связи;
- формировать альтернативы решения;
- выбирать критерии оценки вариантов.

### Продвинутый уровень

- управлять сложными многофакторными решениями;
- находить системные причины, а не симптомы;
- внедрять аналитические практики в команде.

## Как развивать

- mind mapping для декомпозиции;
- 5 Why для поиска корневой причины;
- таблица гипотез: `гипотеза -> данные -> критерий проверки`.

## Упражнение

1. Выбрать сложную проблему из текущего проекта.
2. Построить mind map с минимум 3 уровнями деталей.
3. Сформулировать 3 альтернативных решения.
4. Оценить решения по критериям: ценность, риск, стоимость.

## Оценка прогресса

| [Метрика](../../glossary.md#term-044) | Как считать |
| --- | --- |
| Доля решений с альтернативами | решения с >=2 вариантами / все решения |
| Снижение rework | задачи, возвращенные из-за неполного анализа |
| 360-feedback | оценка структурности анализа от команды |

## Реальный пример

Ситуация: рост времени обработки заявок в госсервисе.

- симптом: [SLA](../../glossary.md#abbr-079) нарушается на 30% кейсов;
- анализ: узкое место не в [UI](../../glossary.md#abbr-091), а в ручной валидации документов;
- решение: автоматизировать pre-check + изменить маршрут сложных кейсов.
