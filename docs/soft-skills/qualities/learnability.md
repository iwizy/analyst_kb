# Обучаемость

Обучаемость позволяет аналитику быстро осваивать новые домены, инструменты и практики, превращая знания в измеримый рабочий эффект.

## Уровни сложности

### Базовый уровень

- видеть собственные skill gap;
- учиться короткими циклами "понял -> применил";
- фиксировать выводы после практики.

### Средний уровень

- строить continuous learning roadmap;
- сочетать обучение, практику и обратную связь;
- оценивать эффект обучения по рабочим метрикам.

### Продвинутый уровень

- управлять развитием компетенций на уровне команды;
- системно внедрять новые практики;
- балансировать ширину и глубину экспертизы.

## Continuous learning roadmap (шаблон)

| Период | Навык | Формат | Практика | Метрика эффекта |
| --- | --- | --- | --- | --- |
| Месяц 1 | API security basics | курс + статьи | review 2 API требований | меньше security-замечаний |
| Месяц 2 | facilitation | воркшоп | 3 фасилитированные встречи | выше доля встреч с решениями |

## Как развивать

- learning backlog: список навыков по приоритету;
- правило 70-20-10: практика/менторинг/теория;
- post-learning review: чему научился и где применил.

## Оценка прогресса

- время адаптации к новому домену;
- доля изученного, примененного в реальных задачах;
- обратная связь от коллег по качеству новых практик.

## Реальный пример

Ситуация: аналитик переходит в проект с event-driven архитектурой.

План:

1. 2 недели базового обучения.
2. Применение на одной интеграционной инициативе.
3. Ретро с архитектором и QA.

Результат: быстрее согласованы требования по надежности и idempotency.
