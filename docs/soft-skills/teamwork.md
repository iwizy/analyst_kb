# Работа в команде

Работа в команде для аналитика - это координация ролей и решений между бизнесом, разработкой, QA и эксплуатацией.

## Базовые принципы

- ясные роли и ответственность;
- прозрачная коммуникация;
- единый источник правды по решениям;
- ранняя эскалация блокеров.

## Практические техники

- RACI для спорных зон ответственности;
- daily blocker check;
- weekly dependency review;
- ретроспективы с фокусом на действия.

## Упражнение

1. Постройте RACI для одного сквозного процесса (например, релиз).
1. Выделите 3 неясные зоны ответственности.
1. Зафиксируйте новые правила handover и проверьте эффект через 2 недели.

## Типичные ошибки

- silo-работа и передача задач без контекста;
- неявные ожидания между ролями;
- отсутствие owner-а у межкомандных проблем.
