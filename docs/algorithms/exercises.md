# Практика и самопроверка

Раздел содержит задания от базовых к продвинутым, чтобы закрепить алгоритмы, структуры данных и инженерные trade-off.

## Уровни сложности

### Базовый

1. Реализуйте линейный и бинарный поиск; сравните число сравнений.
2. Напишите insertion sort и протестируйте на почти отсортированных данных.
3. Реализуйте стек и очередь на массиве.
4. Оцените Big-O для алгоритма нахождения максимума.

### Средний

1. Реализуйте BFS и DFS для одного графа; проверьте компоненты связности.
2. Напишите Dijkstra для графа дорог и сравните с BFS на невзвешенной версии.
3. Реализуйте 0/1 knapsack (DP) и activity selection (greedy), сравните подходы.
4. Напишите quicksort с random pivot и сравните с merge sort на 1 млн элементов.

### Продвинутый

1. Реализуйте KMP и сравните с встроенным поиском строк.
2. Реализуйте Kruskal с DSU и Prim с heap; сравните на плотном и разреженном графе.
3. Реализуйте базовый max-flow (Edmonds-Karp) для сети пропускных способностей.
4. Реализуйте Aho-Corasick или Trie для множественного поиска паттернов.
5. Реализуйте backtracking для N-Queens и добавьте pruning.

## Практические мини-кейсы

### [Кейс](../glossary.md#term-036) 1: сортировка заказов

- Дано: 10 млн заказов, приоритет по дате и [SLA](../glossary.md#abbr-079).
- Выбрать: стабильный алгоритм сортировки.
- Сравнить: время, память, устойчивость.

### Кейс 2: маршрутизация инцидентов

- Дано: поток инцидентов и правила эскалации.
- Выбрать: дерево решений + priority queue.
- Оценить: p95 время принятия решения.

### Кейс 3: логистика

- Дано: граф дорог с весами и ограничениями.
- Выбрать: Dijkstra или A*.
- Сравнить: качество и скорость маршрута.

## Формат самопроверки

Для каждой задачи зафиксируйте:

1. входные данные и ограничения;
2. выбранные алгоритмы/структуры;
3. оценку сложности времени и памяти;
4. результат benchmark;
5. типичные ошибки и как исправили.
