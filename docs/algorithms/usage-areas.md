# Области применения алгоритмов и структур данных

Алгоритмы нужны не только разработке. Аналитик использует их как инструмент выбора решения под бизнес-ограничения.

## Карта применения

| Бизнес-задача | Алгоритмы/структуры | Практический эффект |
| --- | --- | --- |
| Маршрутизация доставки | граф + Dijkstra/A* | минимизация времени и стоимости маршрута |
| Антифрод-проверки | hash table + rule tree + graph analytics | быстрый скоринг и выявление связей |
| Поиск по каталогу | индекс + бинарный поиск + trie | снижение latency поиска |
| Планирование очереди | priority queue / heap | соблюдение SLA по приоритетам |
| Отчеты и ранжирование | сортировки `O(n log n)` | стабильная аналитика на больших объемах |
| ETL и batch | линейные проходы + hash join | ускорение обработки данных |

## Сквозной пример: логистика

Сценарий: нужно распределить заказы между складами и курьерами.

1. Граф дорог и ограничений времени.
1. Dijkstra для расчета кратчайших маршрутов.
1. Priority queue для назначения срочных заказов.
1. Сортировка отгрузок по окнам доставки.

Результат: меньше опозданий, ниже пробег, предсказуемый SLA.

## Практика выбора алгоритма

- Определите цель: минимизировать время, стоимость, риск ошибки.
- Зафиксируйте профиль данных: объем, частота обновления, распределение.
- Оцените нагрузку: средний и пиковый профиль.
- Выберите минимально сложный алгоритм, который закрывает требования.

## Типичные ошибки

- преждевременная оптимизация без подтвержденной проблемы;
- выбор сложной структуры без команды, способной поддерживать решение;
- отсутствие fallback-плана при деградации.
