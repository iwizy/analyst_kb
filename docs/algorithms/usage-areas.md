# Области применения алгоритмов и структур данных

Алгоритмы дают практическую ценность, когда связаны с конкретными сценариями бизнеса и ограничениями системы.

## Уровни сложности

### Базовый

- сопоставлять тип задачи и класс алгоритма;
- видеть связь алгоритма и структуры данных;
- учитывать базовые NFR при выборе.

### Средний

- использовать алгоритмические паттерны в доменных сценариях;
- выбирать компромиссы latency/memory/cost;
- связывать выбор с требованиями и архитектурой.

### Продвинутый

- проектировать алгоритмические pipeline на уровне платформы;
- оценивать влияние решения на интеграции и данные;
- строить roadmap оптимизаций по метрикам.

## Карта применения

| Домен | Типовые алгоритмы/структуры | Сценарий |
| --- | --- | --- |
| Fintech | графы, shortest path, heap, hash-map | antifraud граф связей, приоритизация транзакций |
| E-commerce | сортировки, поиск, trie, recommendation ranking | поиск и ранжирование каталога |
| Госуслуги | DAG, topological sort, очереди | маршрутизация заявок и зависимостей проверок |
| Логистика | Dijkstra/A*, DP | оптимизация маршрутов и загрузки |
| Поддержка и эксплуатация | decision tree, priority queue | triage инцидентов и SLA-эскалации |

## Сквозной кейс: логистика последней мили

1. Граф дорог -> `A*` для маршрута курьера.
2. Приоритетная очередь -> выбор ближайшего SLA-критичного заказа.
3. DP -> оптимизация группировки заказов по слотам.
4. Hash-map -> быстрый lookup статуса и геокодов.

## Плюсы и минусы "алгоритмизации" требований

| Плюсы | Минусы |
| --- | --- |
| прозрачная оценка реализуемости | риск переусложнить решение |
| раннее обнаружение bottleneck | требуется точный профиль данных |
| лучшее обоснование архитектурных решений | нужен регулярный performance review |

## Типичные ошибки

- применение учебного алгоритма без доменной адаптации;
- отсутствие валидации на реальных данных;
- недоучет стоимости эксплуатации;
- выбор "самого умного", а не "достаточно хорошего" решения.
## Задачи для самопроверки

1. Выберите один сервис и составьте карту алгоритмов внутри него.
2. Для каждого алгоритма оцените риски деградации при росте x10.
3. Сформируйте план оптимизации одной проблемной операции.
