# Алгоритмы

Раздел оформлен как практический курс по алгоритмам и структурам данных для системного аналитика. Цель курса — понимать вычислительную сложность решений, обосновывать NFR и говорить с командой разработки на одном языке.

## Зачем аналитику алгоритмы

- оценивать реалистичность SLA/NFR до старта реализации;
- понимать, где проблема в алгоритме, а где в инфраструктуре;
- выбирать структуру данных под бизнес-сценарий;
- снижать риск неверных архитектурных решений при росте нагрузки.

## Что входит в программу

| Блок | Что разбираем | Где применять |
| --- | --- | --- |
| Сортировки | quick sort, merge sort, heap sort | Пакетная обработка, ранжирование, подготовка отчетов |
| Поиск | линейный, бинарный, BFS, DFS | Поиск записей, маршрутов, зависимостей |
| Графы | кратчайшие пути, обходы, связность | Логистика, fraud-цепочки, рекомендации |
| Деревья | бинарные деревья, AVL, B-tree | Индексация, диапазонные запросы, каталоги |
| Хеш-таблицы | map/set, коллизии | Кэш, дедупликация, быстрые lookup |
| Рекурсия и итерация | trade-off по памяти и читаемости | Деревья документов, вычисление иерархий |
| Big-O | асимптотика и сравнение альтернатив | Оценка масштабируемости решений |

## Мини-алгоритм выбора подхода

1. Определите операцию: поиск, сортировка, агрегирование, обход графа.
1. Оцените объем данных и профиль нагрузки.
1. Выберите структуру данных под тип операций.
1. Сравните варианты по Big-O и памяти.
1. Проверьте, подходит ли решение под SLA.

## Примеры по доменам

### Финтех

- задача: быстро находить цепочки транзакций;
- подход: граф + BFS/DFS + индексы по узлам;
- контроль: время построения пути и полнота покрытия.

### E-commerce

- задача: ранжирование каталога и фильтров;
- подход: сортировки + индексные структуры + кеширование;
- контроль: latency поиска и свежесть данных.

### Госуслуги

- задача: обработка и маршрутизация заявлений по этапам;
- подход: деревья состояний + поиск кратчайшего маршрута по регламенту;
- контроль: корректность переходов и время обработки.
