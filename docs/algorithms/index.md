# Алгоритмы

Раздел построен как практический курс для системных аналитиков: от основ оценки сложности до выбора алгоритма под реальные ограничения продукта, архитектуры и данных.

## Уровни сложности

### Базовый уровень

- понимать, что такое алгоритм и структура данных;
- читать и обсуждать псевдокод;
- сравнивать решения по Big-O.

### Средний уровень

- выбирать алгоритм под объем данных и [SLA](../glossary.md#abbr-079);
- оценивать trade-off между временем и памятью;
- объяснять команде, почему выбран конкретный подход.

### Продвинутый уровень

- проектировать решения для высоконагруженных сценариев;
- использовать эвристики, приближенные и распределенные алгоритмы;
- строить системный процесс валидации и benchmark-оценки.

## Зачем это системному аналитику

| [Задача](../glossary.md#term-028) аналитика | Где нужны алгоритмы |
| --- | --- |
| Оценка рисков решения | понимание лучшего/худшего случая сложности |
| Проработка требований [NFR](../glossary.md#abbr-053) | выбор алгоритма под latency и memory budget |
| Согласование с архитекторами/разработкой | аргументация по trade-off и масштабированию |
| [Декомпозиция](../glossary.md#term-019) бизнес-процессов | выбор структуры данных и способа поиска |

## Как работать с разделом

1. Начните с `Сложность и O-нотация` и `Структуры данных`.
2. Пройдите `Сортировки`, `Поиск`, `Графы и деревья`.
3. Изучите `DP`, `Greedy`, `Divide & Conquer`.
4. Перейдите к `Анализу и выбору алгоритмов` и доменным кейсам.
5. Закрепите материал в разделе `Практика`.

## Задачи для самопроверки

1. Возьмите 2 бизнес-сценария из своего проекта и подберите для них алгоритмы/структуры данных.
2. Для каждого сценария выпишите: входные данные, ограничения, target latency.
3. Сравните минимум 2 альтернативных подхода по времени и памяти.
