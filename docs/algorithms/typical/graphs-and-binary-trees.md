# Теория графов и бинарные деревья

Графы и деревья описывают связи между объектами. Графы подходят для сетей и
зависимостей, деревья - для иерархий и поиска в структурированных данных.

## Графы: ключевые понятия

- вершина (node) и ребро (edge);
- ориентированный/неориентированный граф;
- взвешенный граф (вес ребра);
- путь, связность, цикл.

## Полезные алгоритмы для графов

| Задача | Алгоритм | Где применяют |
| --- | --- | --- |
| Обход всей связной области | BFS / DFS | анализ зависимостей, граф ролей |
| Кратчайший путь | Dijkstra, A\* | маршрутизация, логистика |
| Порядок выполнения зависимостей | Topological sort | пайплайны, сборка, workflow |
| Выявление циклов | DFS/Kahn | контроль корректности зависимостей |

## Пример графа маршрутов


## Бинарные деревья

Бинарное дерево - структура, где у узла не более двух потомков (`left`,
`right`). Частный случай - бинарное дерево поиска (BST), где `left < root < right`.

Основные обходы:

- Preorder: `root -> left -> right`;
- Inorder: `left -> root -> right`;
- Postorder: `left -> right -> root`;
- Level-order: по уровням (обычно BFS).

## Пример бинарного дерева

```kroki-plantuml
@startuml
object "8" as N8
object "3" as N3
object "10" as N10
object "1" as N1
object "6" as N6
object "14" as N14

N8 --> N3
N8 --> N10
N3 --> N1
N3 --> N6
N10 --> N14
@enduml
```

## Где это используется в продуктах

- деревья категорий и меню;
- маршруты в графе микросервисов;
- граф зависимостей требований/задач;
- поиск оптимального пути в навигации и доставке.

## Типичные ошибки

- выбор DFS/BFS без понимания целевого результата;
- игнорирование циклов в графе зависимостей;
- несбалансированные деревья без контроля деградации поиска.

## Чек-лист качества

- Понятно, почему выбрана графовая или древовидная модель.
- Для операций определены ожидаемые сложности.
- Обработаны циклы, недостижимые узлы и пустые ветви.
