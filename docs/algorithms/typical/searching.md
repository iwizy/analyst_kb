# Алгоритмы поиска

Поиск отвечает на вопрос: как быстро найти элемент в данных. Для аналитика это важно при оценке latency в пользовательских сценариях и интеграциях.

## Линейный и бинарный поиск

| Алгоритм | Требования к данным | Сложность | Где использовать |
| --- | --- | --- | --- |
| Линейный поиск | не требуется сортировка | `O(n)` | небольшие списки, одноразовый поиск |
| Бинарный поиск | массив должен быть отсортирован | `O(log n)` | частые запросы в статическом/редко меняющемся наборе |

## Псевдокод

### Линейный поиск

```text
linearSearch(A, target):
  for i from 0 to len(A)-1:
    if A[i] == target:
      return i
  return -1
```

### Бинарный поиск

```text
binarySearch(A, target):
  left = 0
  right = len(A) - 1
  while left <= right:
    mid = left + (right - left) / 2
    if A[mid] == target:
      return mid
    if A[mid] < target:
      left = mid + 1
    else:
      right = mid - 1
  return -1
```

## Бизнес-сценарии

- Каталог товаров: бинарный поиск по отсортированному SKU.
- Реестр клиентов: линейный поиск на маленьком наборе кандидатов после фильтра.
- Поиск тарифа: бинарный поиск по диапазонам пороговых значений.

## Практические замечания

- Если данных мало и поиск редкий, сортировка ради бинарного поиска может быть избыточной.
- Если поиск частый, выгодно заранее отсортировать или использовать hash table.

## Типичные ошибки

- запуск бинарного поиска на неотсортированном массиве;
- неверный расчет `mid` и риск бесконечного цикла;
- отсутствие обработки случаев `not found`.
