# UDP

UDP это транспорт без установления соединения и гарантий доставки/порядка, но с минимальной задержкой и накладными расходами.

## Когда использовать

- realtime media;
- телеметрия, где допустима потеря части пакетов;
- протоколы поверх UDP с собственной логикой надежности.

## Ограничения

- нет встроенного подтверждения доставки;
- нет гарантии порядка;
- приложение должно само реализовать retry/ordering/ack при необходимости.

## Пример сценария

Система мониторинга может отправлять частые метрики по UDP, принимая риск потери отдельных пакетов ради throughput.

## Практические рекомендации

- использовать UDP только при явной бизнес-допустимости потерь;
- ограничивать размер datagram (MTU awareness);
- проектировать backpressure и защиту от flood.
