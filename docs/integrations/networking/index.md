# Сетевое взаимодействие

Интеграции зависят от сетевых характеристик: latency, packet loss, jitter, MTU, TLS handshake, connection limits.

## Что важно аналитикам и архитекторам

- выбрать транспорт под профиль нагрузки;
- учесть свойства TCP/UDP;
- понимать влияние сети на retry/timeouts;
- проектировать протоколы приложений с учетом сбоев сети.

## Практический минимум

- модель OSI и соответствие стеку TCP/IP;
- TCP для надежной доставки и упорядоченности;
- UDP для low-latency сценариев;
- sockets как программный интерфейс сети.
