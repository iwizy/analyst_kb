# YAML

YAML чаще используется для конфигураций и спецификаций, чем для runtime API payload.

## Где уместен YAML

- CI/CD конфигурации;
- Kubernetes manifests;
- OpenAPI/RAML спецификации;
- инфраструктурные декларативные файлы.

## Пример

```yaml
service:
  name: orders-api
  port: 8080
security:
  oauth2:
    issuer: https://auth.example.com
rate_limit:
  rps: 100
  burst: 200
```

## Достоинства

- высокая читаемость;
- удобен для ручного редактирования;
- компактнее XML.

## Риски

- чувствительность к отступам;
- сложные конструкции труднее валидировать глазами;
- неоднозначности при преобразовании типов в разных парсерах.

## Практические рекомендации

- использовать schema/lint для YAML;
- фиксировать стиль форматирования в репозитории;
- избегать избыточно сложных YAML-якорей в критичных файлах.
