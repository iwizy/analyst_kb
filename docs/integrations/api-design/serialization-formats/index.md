# Форматы сериализации

Формат сериализации влияет на производительность, совместимость и читаемость интеграций.

## Что выбирать

- JSON: стандарт для web/API-интеграций.
- XML/XSD: строгие enterprise-контракты и legacy/B2B.
- YAML: конфигурации и человекочитаемые спецификации.

## Критерии выбора

- размер payload и latency;
- требования к валидации схемы;
- совместимость со стэком контрагентов;
- требования к человекочитаемости.

## Практические рекомендации

- для runtime API чаще JSON;
- для строго формализованных B2B-договоров XML/XSD;
- YAML использовать для config/spec, но не всегда для runtime payload.
