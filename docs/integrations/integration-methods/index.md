# Способы интеграции

Способ интеграции выбирается по четырем критериям: связность, задержка, гарантии доставки и сложность эксплуатации.

## Сравнение подходов

| Подход | Связь | Задержка | Гарантии | Лучший сценарий |
| --- | --- | --- | --- | --- |
| API (sync) | сильнее | низкая | обычно at-most-once без доп. механик | интерактивные операции |
| Брокер (async) | слабее | средняя | at-least-once / exactly-once* | событийные потоки |
| Shared DB | максимальная | низкая | транзакционная в рамках одной БД | tightly coupled контур |
| File exchange | слабая | высокая | batch + контроль версий файлов | legacy/B2B batch |

`*` зависит от платформы и реализации.
