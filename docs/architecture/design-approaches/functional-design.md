# Функциональное проектирование

Функциональное проектирование описывает систему через функции, входы, выходы и
правила преобразования данных. Подход особенно полезен, когда критична
прозрачность бизнес-процесса и контроль интеграционных потоков.

## Когда применять

- нужно быстро описать процесс от входа до результата;
- есть большое количество интеграций и преобразований данных;
- доменная модель еще нестабильна, но процесс уже понятен.

## Ключевые артефакты

- контекстная схема процесса;
- иерархия функций (декомпозиция);
- спецификация входных/выходных данных;
- правила обработки и бизнес-ограничения.

## Пример: обработка заказа

```kroki-plantuml
@startuml
left to right direction
rectangle "F1 Принять заказ" as F1
rectangle "F2 Проверить оплату" as F2
rectangle "F3 Резервировать товар" as F3
rectangle "F4 Сформировать отгрузку" as F4

F1 --> F2 : заказ + данные оплаты
F2 --> F3 : подтвержденный платеж
F3 --> F4 : подтвержденный резерв
@enduml
```

## Практика внедрения

1. Зафиксируйте верхнеуровневую функцию и границы системы.
1. Выполните декомпозицию до уровня, где можно писать требования и контракты.
1. Для каждой функции определите вход, выход и критерий завершения.
1. Проверьте, что между уровнями декомпозиции сохраняется баланс входов/выходов.

## Типичные ошибки

- смешение функциональной модели с организационной структурой;
- разная глубина декомпозиции в соседних ветках;
- отсутствие явных правил обработки исключений.

## Чек-лист качества

- У каждой функции есть измеримый результат.
- Для потоков данных определены источники и получатели.
- В модели отражены основные альтернативные и ошибочные сценарии.
