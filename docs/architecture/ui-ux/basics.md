# Основы UI/UX

UI отвечает за визуальный интерфейс, UX - за удобство и результативность
взаимодействия пользователя с продуктом. Сильный UX снижает количество ошибок,
ускоряет выполнение сценариев и повышает конверсию.

## Базовые принципы

- ясная информационная иерархия;
- предсказуемые элементы управления;
- минимизация когнитивной нагрузки;
- обратная связь о состоянии системы.

## Мини-сценарий: форма заявки

```kroki-plantuml
@startuml
start
:Открыть форму;
:Заполнить обязательные поля;
if (Ошибки валидации?) then (Да)
  :Показать подсказки рядом с полями;
  :Исправить данные;
endif
:Отправить заявку;
:Показать статус принятия;
stop
@enduml
```

## Практические правила

1. Сначала проектируйте пользовательский сценарий, затем экран.
1. Для каждой формы фиксируйте правила валидации и тексты ошибок.
1. Проверяйте ключевые сценарии на реальных пользователях или внутренних экспертных сессиях.

## Типичные ошибки

- перегруженные экраны без приоритизации действий;
- скрытые ограничения без явной обратной связи;
- проектирование только под "идеальный" путь без учета исключений.

## Чек-лист качества

- У ключевых сценариев измеряются время выполнения и доля ошибок.
- Тексты интерфейса соответствуют терминологии предметной области.
- UX-решения согласованы с доступностью и мобильным контуром.
