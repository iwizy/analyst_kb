# Customer Journey Map

Customer Journey Map (CJM) показывает путь пользователя по этапам взаимодействия
с продуктом: от первого контакта до достижения результата и поддержки.

## Из чего состоит CJM

- этапы пути (stages);
- действия пользователя;
- точки контакта (touchpoints);
- эмоции и pain points;
- возможности для улучшения.

## Пример этапов для цифрового онбординга

```kroki-plantuml
@startuml
left to right direction
rectangle "Осведомленность" as S1
rectangle "Регистрация" as S2
rectangle "Верификация" as S3
rectangle "Первое целевое действие" as S4
rectangle "Поддержка" as S5

S1 --> S2
S2 --> S3
S3 --> S4
S4 --> S5
@enduml
```

Типичные pain points в таком пути:

- длинная форма регистрации;
- неочевидные причины отказа в верификации;
- разрыв между web и mobile каналами.

## Как работать с CJM

1. Выберите конкретный сегмент пользователей и цель сценария.
1. Зафиксируйте фактический путь по данным и интервью.
1. Выделите узкие места по метрикам (drop-off, время шага, обращения в поддержку).
1. Сформулируйте backlog UX-улучшений и критерии эффекта.

## Чек-лист качества

- Для каждого этапа определены метрика и ответственный.
- Pain points подтверждены данными, а не только мнениями.
- CJM связан с roadmap продукта и архитектурными ограничениями.

## Практическое дополнение

1. Перед применением подхода зафиксируйте измеримую цель (SLA, lead time, error rate, time-to-market).
1. На архитектурном review проверяйте соответствие решения ограничениям безопасности, интеграций и эксплуатации.
1. Добавляйте в backlog минимум одну задачу на снижение архитектурного риска для каждой high-impact зоны.
