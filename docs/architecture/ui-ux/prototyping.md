# Прототипирование

Прототипирование позволяет быстро проверить гипотезы интерфейса до дорогой
реализации. Это инструмент снижения риска неверных решений в UX и требованиях.

## Уровни детализации

- Low-fidelity: быстрые схемы экранов и пользовательских потоков.
- Mid-fidelity: кликабельные макеты с базовой структурой контента.
- High-fidelity: максимально близко к финальному UI для UX-тестов и передачи в разработку.

## Когда и что использовать

- ранняя стадия: low-fi для проверки структуры сценария;
- уточнение требований: mid-fi для согласования экранов и переходов;
- перед реализацией: high-fi для проверки микровзаимодействий и контента.

## Пример процесса

```kroki-plantuml
@startuml
start
:Сформулировать UX-гипотезу;
:Собрать low-fi прототип;
:Провести быстрый пользовательский тест;
if (Гипотеза подтверждена?) then (Да)
  :Подготовить high-fi и спецификацию;
  :Передать в разработку;
else (Нет)
  :Изменить сценарий и повторить;
endif
stop
@enduml
```

## Типичные ошибки

- попытка сразу делать high-fi без проверки сценария;
- отсутствие критериев успешности UX-теста;
- несоответствие прототипа реальным ограничениям backend/API.

## Чек-лист качества

- Прототип покрывает ключевые и негативные сценарии.
- Результаты тестов зафиксированы и конвертированы в требования.
- Решения по прототипу синхронизированы с командой разработки.
