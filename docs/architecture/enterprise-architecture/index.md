# Архитектура предприятия

Архитектура предприятия (Enterprise Architecture, EA) связывает стратегию,
бизнес-процессы, приложения и технологический ландшафт в единую систему
управления изменениями.

## Ценность для аналитика

- помогает обосновывать инициативы через стратегические цели;
- показывает зависимости между бизнес-возможностями и ИТ-активами;
- снижает риск локальных решений, конфликтующих с целевой архитектурой.

## Популярные фреймворки

| Фреймворк | Сильная сторона | Где чаще применяют |
| --- | --- | --- |
| TOGAF | процесс трансформации (ADM) | крупные корпоративные программы |
| Zachman | структурирование архитектурных артефактов | аудит и полнота описания |
| FEAF | унификация госархитектуры и reference models | госорганизации США |
| DoDAF | формализация view для сложных систем | оборонные и критичные системы |

## Как использовать EA на проекте

1. Определите стратегическую цель и целевые capability.
1. Зафиксируйте текущий и целевой ландшафт (as-is / to-be).
1. Сформируйте дорожную карту перехода с контрольными вехами.
1. Проверяйте архитектурные решения проекта на соответствие целевой модели.

## Практическое дополнение

1. Перед применением подхода зафиксируйте измеримую цель (SLA, lead time, error rate, time-to-market).
1. На архитектурном review проверяйте соответствие решения ограничениям безопасности, интеграций и эксплуатации.
1. Добавляйте в backlog минимум одну задачу на снижение архитектурного риска для каждой high-impact зоны.
