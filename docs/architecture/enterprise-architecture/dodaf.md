# DODAF

> Глоссарий терминов и сокращений: [Открыть](../../glossary.md)


DoDAF (Department of Defense Architecture Framework) - фреймворк для описания
сложных систем через набор стандартизированных viewpoints и моделей.

## Когда применяют

- большие программы с множеством участников и подрядчиков;
- критичные системы с высокими требованиями к прослеживаемости решений;
- проекты, где важна совместимость систем и формальная проверка соответствия.

## Основные viewpoint-группы

- All Viewpoint (AV): контекст и словарь архитектуры.
- Capability (CV): способности и их развитие.
- Operational (OV): операционные процессы и взаимодействия.
- Systems/Services (SV/SvcV): системные и сервисные реализации.
- Standards (StdV): стандарты и ограничения.

## Пример архитектурной связки

```kroki-plantuml
@startuml
left to right direction
rectangle "CV-1: Целевые capability" as CV
rectangle "OV-5: Операционный процесс" as OV
rectangle "SV-1: Системный контекст" as SV
rectangle "StdV-1: Стандарты" as ST

CV --> OV : capability реализуется
OV --> SV : процесс поддерживается системами
SV --> ST : соблюдает стандарты
@enduml
```

## Типичные ошибки

- формальное заполнение view без связи с управленческими решениями;
- избыточный объем моделей без приоритизации;
- отсутствие единого глоссария между командами.

## Чек-лист качества

- Для ключевых решений построены связанные view разных уровней.
- Термины и сущности согласованы между view.
- Модели поддерживают анализ влияния изменений.

## Практическое дополнение

1. Перед применением подхода зафиксируйте измеримую цель (SLA, lead time, error rate, time-to-market).
1. На архитектурном review проверяйте соответствие решения ограничениям безопасности, интеграций и эксплуатации.
1. Добавляйте в backlog минимум одну задачу на снижение архитектурного риска для каждой high-impact зоны.
