# Оркестрация и хореография

Это два подхода к управлению межсервисными процессами.

## Отличие

- Оркестрация: центральный оркестратор управляет шагами процесса.
- Хореография: сервисы реагируют на события без центрального управляющего.

## Когда что лучше

- Оркестрация: сложный процесс с явной последовательностью и rollback-логикой.
- Хореография: независимые домены с event-driven коммуникацией.

## Риски

- Оркестрация: риск "умного центра" и усложнения оркестратора.
- Хореография: неочевидный поток процесса, сложнее observability.

## Чек-лист качества

- Явно определена ответственность за бизнес-процесс.
- Для ошибок описан compensation path.
- Сквозная трассировка покрывает все шаги процесса.
