# Архитектурные паттерны и антипаттерны

Паттерны помогают решать типовые архитектурные проблемы, антипаттерны -
подсвечивают решения, которые дают краткосрочную выгоду, но ломают систему в
долгую.

## Часто используемые паттерны

- Circuit Breaker;
- Retry with Backoff;
- CQRS;
- Event Sourcing;
- Strangler Fig (эволюция legacy).

## Распространенные антипаттерны

- Distributed Monolith;
- Shared Database как главный способ интеграции;
- God Service;
- Golden Hammer (один инструмент для всех задач).

## Как анализировать архитектурное решение

1. Какая проблема решается и как измеряется эффект?
1. Какие новые риски создает паттерн?
1. Какие операционные требования нужны для безопасной эксплуатации?

## Чек-лист качества

- Решение обосновано контекстом, а не трендом.
- Известны trade-off и ограничения паттерна.
- Есть план отката/эволюции при изменении условий.
