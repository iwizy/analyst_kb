# Многослойная архитектура

Многослойная архитектура разделяет систему по уровням ответственности:
presentation, application, domain, data access.

## Когда полезна

- корпоративные системы с четкими бизнес-процессами;
- важна поддерживаемость и читаемость кода;
- команда работает с классическим backend-монолитом.

## Базовые слои

- UI/API слой: входные контракты и валидация.
- Application слой: orchestration use cases.
- Domain слой: бизнес-правила.
- Infrastructure слой: БД, очереди, внешние интеграции.

## Типичные ошибки

- "утечки" инфраструктуры в domain;
- бизнес-логика в контроллерах;
- слишком глубокая слойность без ценности.

## Чек-лист качества

- У каждого слоя четкая зона ответственности.
- Направление зависимостей контролируется архитектурно.
- Изменения в UI/БД не ломают domain-логику.
