# Монолитная архитектура

Монолит - приложение, где все функциональные части работают в одном процессе и
развертываются как единый артефакт.

## Когда использовать

- ранняя стадия продукта и небольшая команда;
- доменная модель еще формируется;
- нужен быстрый старт без distributed complexity.

## Сильные стороны

- простая сборка, деплой и отладка;
- низкие эксплуатационные издержки на старте;
- проще поддерживать транзакционную целостность.

## Ограничения

- сложнее масштабировать отдельные части;
- рост связанности модулей со временем;
- релиз одной части тянет релиз всего приложения.

## Типичные ошибки

- отсутствие модульных границ внутри монолита;
- смешение domain logic и инфраструктуры;
- позднее выделение bounded contexts.

## Чек-лист качества

- Монолит разделен на модули с явными интерфейсами.
- Есть архитектурные правила импорта/зависимостей.
- Подготовлен сценарий эволюции при росте нагрузки.
