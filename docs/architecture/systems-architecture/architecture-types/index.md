# Виды архитектур

Выбор архитектурного стиля определяет скорость изменений, стоимость поддержки и
устойчивость системы к росту нагрузки.

## Сравнение подходов

| Стиль | Сильные стороны | Основные риски |
| --- | --- | --- |
| Монолит | простота запуска и сопровождения на старте | сложность масштабирования по компонентам |
| Модульный монолит | структурность без distributed complexity | нужен строгий контроль границ модулей |
| Микросервисы | независимые релизы и масштабирование | высокая стоимость интеграции и эксплуатации |
| SOA | повторное использование сервисов | governance и интеграционный overhead |
| Клиент-сервер | понятная модель взаимодействия | узкое место на стороне сервера |
| P2P | отказоустойчивость и децентрализация | сложность консистентности и безопасности |

## Как выбирать

1. Оцените уровень зрелости команды и эксплуатации.
1. Зафиксируйте NFR (latency, availability, throughput).
1. Определите частоту изменений и границы доменов.
1. Выберите минимально сложную архитектуру, которая закрывает риски.
