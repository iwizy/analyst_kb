# MVC, MVP, MVVM

Это паттерны разделения UI-логики, которые помогают держать интерфейс
поддерживаемым и тестируемым.

## Кратко о моделях

- MVC: контроллер обрабатывает ввод и обновляет модель/представление.
- MVP: presenter управляет view через интерфейс.
- MVVM: view model связывает представление с моделью через binding.

## Когда что выбирать

| Паттерн | Где чаще используется | Сильная сторона |
| --- | --- | --- |
| MVC | веб-серверные приложения | простота старта |
| MVP | приложения с акцентом на testability | явная изоляция view |
| MVVM | SPA/mobile с data binding | удобная реактивная модель |

## Чек-лист качества

- Бизнес-логика не размазана по view-компонентам.
- Слой представления тестируется отдельно.
- Выбранный паттерн соответствует UI-технологии команды.
