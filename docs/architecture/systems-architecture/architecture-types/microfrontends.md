# Микрофронтенды

Микрофронтенды делят UI на независимые части, которые могут разрабатываться и
поставляться разными командами.

## Когда оправдано

- крупный frontend с несколькими доменными командами;
- разные части интерфейса имеют разный цикл релизов;
- важно автономное развитие продуктовых зон.

## Плюсы и риски

- Плюсы: независимые релизы, ownership по доменам.
- Риски: разрозненность UX, дублирование зависимостей, рост bundle size.

## Критичные практики

- единая design system;
- стандарты маршрутизации и shared state;
- централизованный контроль производительности.

## Чек-лист качества

- UX-консистентность подтверждена общими гайдлайнами.
- Есть стратегия интеграции (runtime/build-time).
- Метрики фронтенда собираются на уровне всего продукта.
