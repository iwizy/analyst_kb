# Клиент-серверная архитектура

Клиент-серверная архитектура разделяет систему на потребителя интерфейса
(клиент) и сервер, который хранит данные и исполняет бизнес-логику.

## Когда подходит

- центральное хранение данных и единые правила доступа;
- относительно предсказуемая нагрузка;
- команда строит веб/мобильный продукт с API-слоем.

## Базовая схема

```kroki-plantuml
@startuml
left to right direction
actor "Клиент" as C
rectangle "API-сервер" as S
database "База данных" as D
C --> S : HTTP/gRPC
S --> D : SQL/NoSQL
D --> S
S --> C
@enduml
```

## Плюсы и ограничения

- Плюсы: централизованная безопасность, простота администрирования.
- Ограничения: сервер может стать bottleneck, нужен careful scaling.

## Чек-лист качества

- Явно определены границы API-контрактов.
- Реализованы механизмы горизонтального масштабирования сервера.
- Отказ БД и API покрыт планом деградации.
