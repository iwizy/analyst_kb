# Микросервисная архитектура

Микросервисная архитектура делит систему на автономные сервисы с независимой
поставкой и собственным жизненным циклом.

## Когда оправдана

- крупный продукт с несколькими командами;
- разные домены развиваются с разной скоростью;
- требуется независимое масштабирование компонентов.

## Ключевые требования к зрелости

- развитые CI/CD и observability;
- стандарты API и контрактного тестирования;
- зрелая практика incident/problem management.

## Преимущества и риски

- Плюсы: независимые релизы, изоляция отказов, гибкое масштабирование.
- Риски: распределенные транзакции, сложность диагностики, сетевые ошибки.

## Чек-лист качества

- Границы сервисов соответствуют bounded contexts.
- Для межсервисных вызовов определены retry, timeout, idempotency.
- Метрики и трассировка работают end-to-end.
