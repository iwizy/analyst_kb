# Масштабирование сервисов

Масштабирование - это настройка системы под рост нагрузки без деградации SLA.

## Виды масштабирования

- Вертикальное: больше ресурсов на один инстанс.
- Горизонтальное: больше инстансов сервиса.
- Функциональное: выделение нагрузки по доменам/сервисам.

## Практический подход

1. Найдите bottleneck по метрикам (CPU, DB locks, queue lag).
1. Устраните алгоритмические и инфраструктурные узкие места.
1. Выберите стратегию scaling с учетом стоимости.
1. Проверьте результат на нагрузочном профиле, близком к production.

## Чек-лист качества

- Для каждого критичного сервиса есть scaling policy.
- Пороговые метрики и алерты настроены.
- Проведены load/stress-тесты и зафиксированы лимиты.
