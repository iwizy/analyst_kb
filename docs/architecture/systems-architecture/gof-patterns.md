# GoF-паттерны в системном проектировании

GoF-паттерны - фундаментальные шаблоны объектного проектирования. Для аналитика важны не детали кода, а влияние паттерна на расширяемость и поддерживаемость системы.

## Singleton

### Идея

Гарантирует единственный экземпляр объекта и глобальную точку доступа.

### Где полезен

- кэш конфигурации приложения;
- менеджер соединений (с осторожностью);
- централизованный feature-flag provider.

### Риски

- скрытые глобальные зависимости;
- сложность тестирования;
- риск проблем с потокобезопасностью.

## Factory (Factory Method / Abstract Factory)

### Идея

Отделяет создание объекта от его использования.

### Где полезен

- выбор провайдера интеграции по конфигурации;
- создание клиентов разных API (REST/SOAP) по единому интерфейсу;
- переключение реализаций для разных контуров.

### Риски

- избыточная абстракция для простых сценариев;
- усложнение навигации по коду.

## Observer

### Идея

Сущность подписывается на события и реагирует на изменения источника.

### Где полезен

- событийная обработка в UI и backend;
- интеграции publish/subscribe;
- реакция на изменения статуса заказа.

### Риски

- неявные зависимости подписчиков;
- сложность отладки последовательности реакций.

## Чек-лист выбора GoF-паттерна

- есть повторяемая проблема, а не разовый кейс;
- выгода от шаблона выше стоимости усложнения;
- команда понимает правила применения и ограничения.
