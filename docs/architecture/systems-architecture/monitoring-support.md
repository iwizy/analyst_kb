# Мониторинг и поддержка

Наблюдаемость (logs, metrics, traces) и процессы поддержки определяют
способность системы быстро обнаруживать и устранять инциденты.

## Минимальный стек наблюдаемости

- метрики сервиса и инфраструктуры;
- структурированные логи;
- distributed tracing;
- алерты по SLI/SLO.

## Operational flow

```kroki-plantuml
@startuml
start
:Сработал алерт;
:Дежурный анализирует метрики/логи/трейсы;
if (Причина ясна?) then (Да)
  :Устранить и проверить восстановление;
else (Нет)
  :Эскалация и war-room;
endif
:Postmortem и действия по предотвращению;
stop
@enduml
```

## Чек-лист качества

- Для критичных сервисов определены SLO/SLI.
- Runbook покрывает частые инциденты.
- Постмортемы превращаются в план улучшений.
