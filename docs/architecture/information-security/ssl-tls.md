# SSL/TLS

TLS защищает транспортный канал между клиентом и сервером: обеспечивает
конфиденциальность, целостность и проверку подлинности сервера.

## Что важно в современных системах

- использовать TLS 1.2/1.3;
- отключать устаревшие протоколы и слабые шифры;
- контролировать срок жизни сертификатов и цепочку доверия.

## Упрощенный handshake

```kroki-plantuml
@startuml
participant Client
participant Server

Client -> Server : ClientHello
Server -> Client : ServerHello + Certificate
Client -> Server : KeyExchange + Finished
Server -> Client : Finished
Client <-> Server : Шифрованный трафик приложения
@enduml
```

## Практические рекомендации

- включать HSTS для публичных веб-приложений;
- автоматизировать выпуск и ротацию сертификатов;
- применять mTLS на внутренних сервисных контурах с повышенными требованиями.

## Типичные ошибки

- self-signed сертификаты в продуктивном контуре без доверенной цепочки;
- отсутствие проверки hostname и срока действия сертификата;
- смешение HTTP и HTTPS в одном пользовательском сценарии.

## Чек-лист качества

- Сертификаты выпускаются и обновляются автоматически.
- TLS-настройки проходят регулярный security-review.
- Для критичных сервисов включен мониторинг срока действия сертификатов.
