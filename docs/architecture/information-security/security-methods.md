# Способы обеспечения ИБ

Безопасность достигается комбинацией организационных, процессных и технических
контролей. Один инструмент не закрывает весь профиль угроз.

## Классы мер

- Организационные: политики доступа, сегрегация ролей, обучение персонала.
- Процессные: secure SDLC, threat modeling, управление уязвимостями.
- Технические: IAM, шифрование, WAF, EDR, сегментация сети, резервирование.

## Практический минимум для продукта

1. Утвердите модель угроз по критичным сценариям.
1. Внедрите обязательные security-gates в [CI/CD](../../glossary.md#abbr-017).
1. Настройте мониторинг инцидентов и процесс реагирования.
1. Регулярно проводите ревизию прав и тестирование уязвимостей.

## Типичные ошибки

- безопасность добавляется в конце проекта;
- reliance на периметр без защиты на уровне приложения;
- отсутствие владельца security-контролей.

## Чек-лист качества

- Для критичных данных определены классы защиты.
- У каждого контроля есть владелец и период проверки.
- Результаты security-проверок влияют на release decision.

## Практическое дополнение

1. Перед применением подхода зафиксируйте измеримую цель ([SLA](../../glossary.md#abbr-079), lead time, error rate, time-to-market).
1. На архитектурном review проверяйте соответствие решения ограничениям безопасности, интеграций и эксплуатации.
1. Добавляйте в backlog минимум одну задачу на снижение архитектурного риска для каждой high-impact зоны.
