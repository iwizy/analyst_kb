# Основы криптографии

Криптография дает технические механизмы защиты данных и доверия в цифровых
взаимодействиях. Для аналитика важно понимать, какую задачу решает каждый
криптографический примитив.

## Основные примитивы

- симметричное шифрование: один общий ключ для шифрования и расшифрования;
- асимметричное шифрование: пара открытого и закрытого ключа;
- хеш-функции: контроль целостности;
- цифровая подпись: подтверждение авторства и неизменности.

## Управление ключами

- генерация ключей в доверенной среде;
- безопасное хранение в KMS/HSM;
- ротация и отзыв ключей;
- контроль доступа к операциям подписи/дешифрования.

## Пример гибридной схемы

```kroki-plantuml
@startuml
actor "Клиент" as C
participant "Сервис" as S

C -> S : Публичный ключ сервиса
C -> S : Сессионный ключ (зашифрован RSA)
C -> S : Данные (зашифрованы AES)
S --> C : Ответ (AES)
@enduml
```

## Типичные ошибки

- смешение задач шифрования и подписи;
- недостаточное качество случайных чисел;
- отсутствие политики ротации ключей.

## Чек-лист качества

- Для каждого кейса выбраны примитивы под задачу, а не "по привычке".
- Управление ключами автоматизировано и аудируется.
- Криптография встроена в архитектуру, а не добавлена как точечный патч.
