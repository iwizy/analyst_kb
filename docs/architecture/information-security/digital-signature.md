# Электронная подпись

Электронная подпись подтверждает авторство и целостность документа или
сообщения. Для аналитика это ключевой механизм юридически значимых сценариев:
договоры, акты, заявки, финансовые операции.

## Как работает подпись

1. От документа вычисляется хеш.
1. Хеш подписывается закрытым ключом владельца.
1. Получатель проверяет подпись открытым ключом.
1. При совпадении хеша подтверждаются неизменность и авторство.

## Пример потока подписания

```kroki-plantuml
@startuml
actor "Подписант" as U
participant "Сервис подписи" as S
participant "Проверяющий" as V

U -> S : Документ + команда подписать
S --> U : Подпись + сертификат
U -> V : Документ + подпись + сертификат
V --> U : Результат проверки
@enduml
```

## Где применяется

- подписание договоров и заявлений;
- юридически значимый ЭДО;
- подтверждение целостности системных артефактов.

## Типичные ошибки

- хранение закрытого ключа без аппаратной или защищенной среды;
- отсутствие проверки статуса сертификата (отзыв, срок действия);
- подписание неканонизированных данных.

## Чек-лист качества

- Тип подписи соответствует регуляторным требованиям домена.
- Верификация подписи встроена в бизнес-процесс, а не выполняется вручную.
- Журнал подписания и проверки хранится с неизменяемым аудитом.
