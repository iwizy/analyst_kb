# Анализ данных
Data literacy для системного аналитика это умение формулировать вопросы к данным, проверять гипотезы и превращать результаты в управленческие решения.

## Уровни сложности

### Базовый уровень

- понимать типы данных: структурированные, полу-структурированные, неструктурированные;
- владеть операциями `filter`, `group`, `pivot`;
- читать базовые [BI](../../glossary.md#abbr-008)-дашборды и видеть аномалии.

### Средний уровень

- писать [SQL](../../glossary.md#abbr-084)-запросы для сегментации, когорт и воронок;
- формулировать корректные метрики и гипотезы;
- участвовать в дизайне [A/B](../../glossary.md#abbr-001)-тестов и интерпретации результатов.

### Продвинутый уровень

- работать с [DWH](../../glossary.md#abbr-029)/[ETL](../../glossary.md#abbr-033) на уровне требований;
- определять data contracts и проверки качества данных;
- использовать продвинутую аналитику для roadmap и приоритизации.

## Практический цикл аналитика

1. Сформулировать бизнес-вопрос и гипотезу.
2. Определить метрику успеха и источники данных.
3. Проверить качество данных (полнота, свежесть, дубликаты).
4. Выполнить анализ и визуализацию.
5. Подготовить решение и ожидаемый эффект.
6. Проверить эффект после внедрения.

## Примеры по доменам

| [Домен](../../glossary.md#term-023) | Вопрос | [Метрика](../../glossary.md#term-044) | Решение |
| --- | --- | --- | --- |
| Финтех | Почему растет отказ в оплате? | payment success rate | доработка маршрутизации платежей |
| E-commerce | Где теряется конверсия? | checkout conversion | упрощение шага доставки |
| Госуслуги | Что замедляет обработку заявок? | cycle time | переразметка [SLA](../../glossary.md#abbr-079) и очередей |

## Типовые ошибки

- выбор vanity-метрик вместо actionable-метрик;
- сравнение несопоставимых периодов;
- выводы без проверки качества данных;
- принятие решений без учета статистической значимости.
