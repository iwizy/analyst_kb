# Статистика и A/B-тесты

> Глоссарий терминов и сокращений: [Открыть](../../glossary.md)


Статистика нужна аналитику, чтобы отделять реальные эффекты от случайного шума.

## Уровни сложности

### Базовый уровень

- среднее, медиана, дисперсия, стандартное отклонение;
- доверительный интервал;
- p-value и статистическая значимость.

### Средний уровень

- ошибки I и II рода;
- power analysis и расчет выборки;
- корректная постановка гипотез для A/B.

### Продвинутый уровень

- множественные сравнения и поправки;
- cohort analysis и causal impact;
- интерпретация результата в бизнес-контексте.

## Минимальный словарь

| Термин | Что означает |
| --- | --- |
| p-value | вероятность получить не менее экстремальный результат при верной H0 |
| Confidence Interval | диапазон значений параметра с заданной надежностью |
| Power | вероятность обнаружить реальный эффект |
| Effect size | размер практического эффекта, а не только его значимость |

## Процесс A/B-теста

1. Сформулировать гипотезу и primary metric.
2. Определить MDE (minimum detectable effect).
3. Рассчитать выборку и длительность теста.
4. Провести рандомизацию и контролировать смещения.
5. Проанализировать результат и принять решение.

## Пример

Гипотеза: новый экран оплаты увеличит конверсию с 4.0% до 4.6%.

- Базовая конверсия: 4.0%.
- MDE: +0.6 п.п.
- Уровень значимости: 0.05.
- Power: 0.8.

Результат: p-value = 0.02, uplift = +0.5 п.п.

Интерпретация:

- статистически значимо;
- эффект близок к MDE и требует проверки на удержании и выручке;
- rollout поэтапный через canary.

## Типичные ошибки

- ранняя остановка теста;
- переключение primary metric в середине эксперимента;
- игнорирование сезонности и внешних кампаний;
- решение только по p-value без анализа эффекта и стоимости.

## Инструменты

- SQL + Python (pandas/scipy) для расчета;
- BI-дашборды для мониторинга;
- experimentation платформы продукта.
