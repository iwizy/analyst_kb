# OLAP

OLAP (Online Analytical Processing) поддерживает многомерный анализ исторических данных для управленческих решений.

## Уровни сложности

### Базовый уровень

- понимать, что OLAP ориентирован на чтение и агрегации;
- различать факты и измерения;
- формулировать вопросы для аналитических витрин.

### Средний уровень

- строить срезы по времени, сегментам и каналам;
- использовать drill-down/roll-up;
- задавать правила консистентности метрик.

### Продвинутый уровень

- проектировать метрики на уровне enterprise;
- учитывать latency обновления и data freshness;
- управлять аналитической моделью при изменении источников.

## Типовые операции OLAP

| Операция | Назначение |
| --- | --- |
| Slice | выбрать одно измерение |
| Dice | выбрать подмножество по нескольким измерениям |
| Drill-down | перейти к детализации |
| Roll-up | агрегировать на более высокий уровень |

## Кейc: аналитика воронки в e-commerce

- Измерения: дата, канал, устройство, сегмент клиента.
- Метрики: визиты, корзины, заказы, выручка.
- Применение: поиск этапа с максимальным drop-off.

## Типичные ошибки

- разные формулы одной метрики в разных отчетах;
- дашборд без владельца и SLA обновления;
- отсутствие контролей качества на источниках.
