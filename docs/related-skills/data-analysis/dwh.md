# DWH

DWH (Data Warehouse) - корпоративное хранилище, которое интегрирует данные из
разных источников для аналитики и отчетности.

## Когда использовать

- данные распределены по нескольким операционным системам;
- нужны единые метрики и историчность изменений;
- требуется стабильная база для BI-отчетов и аналитики.

## Типовая структура

- `Staging`: прием сырых данных;
- `Core/ODS`: очищенные и согласованные данные;
- `Data Marts`: витрины под конкретные бизнес-задачи.

## Практический сценарий

Для розничной компании DWH объединяет CRM, биллинг и e-commerce:

1. Загрузка данных из источников.
1. Очистка и унификация справочников.
1. Формирование витрины продаж.
1. Построение отчетов по выручке и марже.

## Лучшие практики

- явная модель качества и lineage данных;
- единый глоссарий бизнес-терминов;
- прозрачные SLA обновления витрин;
- контроль изменений схем источников.

## Типичные ошибки

- отсутствие владельцев данных и метрик;
- попытка собрать "универсальную" витрину под все задачи;
- игнорирование влияния изменений источников.

## Чек-лист качества

- Витрины привязаны к конкретным аналитическим вопросам.
- Данные имеют описанные правила качества и проверки.
- Процессы загрузки и обновления мониторятся.
