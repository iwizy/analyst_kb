# UX/UI основы

UX/UI-компетенции помогают аналитику проектировать не только реализуемые, но и удобные решения, которые реально используются пользователями.

## Уровни сложности

### Базовый уровень

- понимать пользовательскую цель, контекст и ограничения;
- строить user flow и простые wireframes;
- применять базовые эвристики Нильсена.

### Средний уровень

- проводить usability-тесты и фиксировать findings;
- учитывать информационную архитектуру и доступность (WCAG);
- переводить UX-наблюдения в требования и acceptance criteria.

### Продвинутый уровень

- организовывать end-to-end discovery с дизайнерами и продуктом;
- применять метрики UX в приоритизации backlog;
- стандартизировать UX-практики на уровне команды/продукта.

## Базовые концепции

- UCD (User-Centered Design): проектирование от задач пользователя.
- Information Architecture: структура контента и навигации.
- Usability heuristics (Nielsen): видимость статуса, контроль пользователя, предотвращение ошибок.
- Accessibility (WCAG): контраст, фокус, keyboard navigation, читаемость.

## Роли в UX-контуре

| Роль | Зона ответственности | Что важно аналитику |
| --- | --- | --- |
| UX-исследователь | интервью, тесты, инсайты | валидировать гипотезы, не подменять их мнением |
| UX/UI-дизайнер | wireframe/prototype, визуальная система | давать полный сценарный контекст |
| Системный аналитик | требования, ограничения, интеграции | связать UX-решение с системой и данными |

## Практический процесс

1. Определить задачу пользователя и критерий успеха.
2. Построить user flow (`as is` -> `to be`).
3. Создать lo-fi wireframe, затем hi-fi прототип.
4. Провести usability-тест (5-7 пользователей).
5. Обновить требования и AC по findings.

## Шаблон user flow

| Шаг | Цель пользователя | Точка риска | Системная реакция |
| --- | --- | --- | --- |
| 1 | Начать регистрацию | не понимает, сколько шагов | показать прогресс 1/3 |
| 2 | Подтвердить телефон | не пришел OTP | повторная отправка + fallback |
| 3 | Завершить профиль | слишком много полей | progressive disclosure |

## Кейс: улучшение формы регистрации

| Пользовательская задача | UX/UI-изменение | Метрика |
| --- | --- | --- |
| Быстро начать регистрацию | сократить поля на первом шаге | completion rate |
| Понять прогресс | добавить индикатор шагов | drop-off by step |
| Исправить ошибку | валидация inline + понятный текст | error recovery time |

Результат пилота: drop-off на шаге OTP снизился с 30% до 18%.

## Типичные ошибки

- описание интерфейса без сценария и цели;
- перегруженный flow без приоритета основных задач;
- игнорирование доступности;
- отсутствие post-release проверки UX-гипотез.

## Инструменты

- Figma, FigJam;
- Draw.io/Miro для user flow;
- Maze/Useberry для удаленного usability testing.
