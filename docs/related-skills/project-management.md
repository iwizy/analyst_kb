# Управление проектами

Проектное управление для аналитика это способность доводить изменения до результата в ограничениях сроков, ресурсов и рисков.

## Уровни сложности

### Базовый уровень

- понимать scope, срок, стоимость, риск и качество;
- участвовать в планировании и коммуникациях;
- фиксировать решения и изменения.

### Средний уровень

- управлять зависимостями и критическим путем;
- использовать WBS, milestones, burn-up/down;
- поддерживать риск-реестр и план коммуникаций.

### Продвинутый уровень

- выбирать между predictive и adaptive подходами;
- вести hybrid delivery модели;
- управлять портфелем инициатив и межкомандной координацией.

## Основные функции PM-контура

| Функция | Что нужно аналитику |
| --- | --- |
| Планирование | декомпозиция scope и критерии готовности |
| Управление сроками | зависимости, вехи, буферы |
| Управление ресурсами | capacity и роли |
| Управление рисками | выявление, mitigation, эскалация |
| Коммуникации | регулярный статус и решение конфликтов |

## Классические и гибкие подходы

| Подход | Плюсы | Ограничения |
| --- | --- | --- |
| PMBOK/PRINCE2 | предсказуемость, сильный governance | медленная адаптация |
| Scrum/Kanban | быстрая обратная связь | выше требования к зрелости команды |
| Hybrid | баланс контроля и гибкости | сложнее внедрение и роли |

## Инструменты

- Gantt/CPM: MS Project, Smartsheet;
- Agile boards: Jira, Azure DevOps;
- Коммуникации: Confluence, Teams/Slack;
- Риски: risk register и review cadence.

## Шаблоны

### Project Charter (минимум)

| Поле | Содержание |
| --- | --- |
| Цель проекта | ожидаемый бизнес-эффект |
| Scope | что входит/не входит |
| Критерии успеха | KPI и acceptance metrics |
| Команда и роли | RACI |
| Риски | топ-5 рисков и владельцы |

### Kick-off agenda

1. Цель и контекст проекта.
2. Scope и границы.
3. План этапов и вех.
4. Роли и коммуникации.
5. Риски и договоренности по эскалации.

## Типичные ошибки

- старт без согласованного scope;
- отсутствует связь между изменением требований и планом;
- риск-реестр не обновляется;
- статус-отчеты без управленческого решения.
