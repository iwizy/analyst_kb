# TCO

TCO (Total Cost of Ownership) помогает увидеть полную стоимость решения за жизненный цикл, а не только стоимость внедрения.

## Уровни сложности

### Базовый уровень

- различать CAPEX и OPEX;
- учитывать прямые и косвенные затраты;
- считать TCO на горизонте 3-5 лет.

### Средний уровень

- сравнивать альтернативы `build vs buy`, `cloud vs on-prem`;
- учитывать стоимость изменений и поддержки;
- добавлять стоимость рисков и простоев.

### Продвинутый уровень

- строить сценарную TCO-модель;
- учитывать инфляцию и рост нагрузки;
- связывать TCO с NPV/ROI.

## Компоненты TCO

| Компонент | Примеры |
| --- | --- |
| Внедрение | разработка, интеграция, миграция |
| Лицензии | подписка, support plan, расширения |
| Инфраструктура | compute, storage, network, backup |
| Эксплуатация | SRE/администраторы, мониторинг, инциденты |
| Развитие | change requests, масштабирование |
| Риски | простой, штрафы, регуляторные издержки |

## Пример сравнения: on-prem vs managed cloud

| Параметр (5 лет) | On-prem | Managed cloud |
| --- | --- | --- |
| Внедрение | выше | ниже |
| Эксплуатация | выше | средняя |
| Гибкость масштабирования | ниже | выше |
| Риск vendor lock-in | ниже | выше |
| Итоговый TCO | зависит от нагрузки и SLA | зависит от тарифов и egress |

## Практические рекомендации

- фиксировать объем нагрузки (RPS, объем данных, рост);
- учитывать обучение команды и change management;
- считать стоимость downtime в деньгах;
- пересматривать TCO при существенных изменениях архитектуры.

## Типичные ошибки

- считать только лицензию/инфраструктуру;
- игнорировать стоимость миграций и интеграций;
- не учитывать рост нагрузки;
- не учитывать ограничение по компетенциям команды.

## Чек-лист самопроверки

- определен горизонт расчета и драйверы затрат;
- есть 2-3 сравниваемые альтернативы;
- оценены прямые и косвенные затраты;
- вывод пригоден для decision board.
