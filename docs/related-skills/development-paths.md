# Треки развития смежных навыков
Страница помогает выбрать персональный трек развития смежных компетенций в зависимости от роли аналитика, домена и карьерной цели.

## Уровни сложности

### Базовый уровень

- выбрать 1 основной и 1 поддерживающий трек;
- задать цели развития на 90 дней;
- связать обучение с текущими рабочими задачами.

### Средний уровень

- комбинировать навыки из нескольких треков;
- измерять эффект на качестве требований и delivery;
- формировать портфолио практических кейсов.

### Продвинутый уровень

- развивать доменную специализацию;
- выступать внутренним экспертом и ментором;
- формировать стандарты команды по смежным практикам.

## Матрица треков

| Трек | Фокус | Ключевой результат |
| --- | --- | --- |
| Data & BI | SQL, BI, статистика, A/B | data-driven решения и прозрачные KPI |
| UX + Testing | UX research, прототипирование, тест-дизайн | меньше дефектов и выше usability |
| Product | discovery, roadmap, приоритизация | связь backlog с бизнес-ценностью |
| Security | authz/authn, threat modeling, compliance | снижение security-рисков в требованиях |
| ML/DS | постановка ML-задач, метрики моделей | корректные ML-требования и эксперименты |
| Finance | unit economics, ROI/NPV/TCO | обоснованная экономическая приоритизация |
| Prompt engineering | автоматизация аналитических рутин | ускорение подготовки артефактов |

## Рекомендованные дорожки

### Дорожка A. Системный аналитик в продуктовой команде

1. UX/UI основы.
2. Основы тестирования.
3. SQL для аналитиков + BI.
4. Product management.

### Дорожка B. Аналитик интеграционных платформ

1. Security basics.
2. Data analysis (SQL + DWH).
3. Project management.
4. Prompt engineering для анализа контрактов и логов.

### Дорожка C. Доменный аналитик (fintech/gov)

1. Экономика и финансы.
2. Security and compliance.
3. Продвинутые специализации по домену.
4. ML/DS basics для риск-моделей и аналитики.

## План развития на 90 дней

| Период | Фокус |
| --- | --- |
| 1-30 день | базовые материалы и шаблоны |
| 31-60 день | применение на реальных задачах |
| 61-90 день | измерение эффекта и корректировка трека |

## Шаблон персонального плана

| Поле | Пример |
| --- | --- |
| Цель | повысить качество acceptance criteria |
| Выбранный трек | UX + Testing |
| Практика | 1 usability review + 1 test design review в неделю |
| Метрика эффекта | снижение дефектов на приемке на 20% |
| Переоценка | в конце каждого месяца |

## Типовые ошибки

- изучение материалов без применения в текущем проекте;
- попытка идти сразу по всем трекам одновременно;
- отсутствие измеримых критериев прогресса;
- фокус на инструментах без понимания методик.
